# 1.5 — Before you start

This is a **medium–advanced, full-stack** course. It does *not* re-teach the basics — it starts at "you can build a simple app" and climbs from there. So before Chapter 2, make sure you're comfortable with the few things below. These are the **floor the course stands on** — deliberately *easier* than what you'll be doing, and only the things you genuinely need on day one. Anything the course itself teaches is **not** listed here.

## What you should already be comfortable with

| You should be comfortable with… | Why you need it | Status |
|---|---|---|
| Building a basic **CRUD REST API** on a Node framework | The whole backend starts here and climbs into auth, feeds, and real-time | **Must have** |
| **Basic React** — components, props, `useState`/`useEffect`, fetching data | The front end is React; the course builds *features*, not React fundamentals | **Must have** |
| **SQL basics** — tables, `SELECT`/`INSERT`/`UPDATE`, joins, foreign keys | You'll model communities, posts, comment trees, and votes, and query them well | **Must have** |
| **`async`/`await`** in JavaScript/TypeScript | Every API call, query, and background job uses it | **Must have** |
| Everyday **Git** — commit, branch, `.gitignore` | You'll commit daily and keep secrets out of history | **Revise if rusty** |
| The shape of an **HTTP request/response** (methods, status codes, headers) | Every endpoint and the auth flow assume it | **Revise if rusty** |

## Helpful, but the course teaches it anyway

Don't worry if you've never used these — you'll learn them here, in context. They're listed only so nobody is scared off:

- **TypeScript** — used throughout; if you only know JavaScript, you'll pick up what you need.
- **Docker** — for running PostgreSQL and Redis locally; introduced from scratch in Chapter 2.
- **Redis** — caching and queues; taught where it's used.
- **WebSockets** — for real-time; introduced in Phase 4.
- **A React state library** (Redux Toolkit, Zustand, or Context) — you'll choose and learn one when the app needs shared state.

## A quick self-check

If you can, right now, sketch a tiny Express (or similar) route that reads a row from a database and returns it as JSON — and a small React component that fetches and displays it — you're ready. If that feels shaky, spend a day on a CRUD-plus-React refresher first; this course will be far more rewarding for it.
