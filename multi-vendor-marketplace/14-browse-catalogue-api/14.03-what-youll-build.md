# 14.3 — What you'll build

By the end of Chapter 14, customers (and your Chapter 15 shop UI) can fetch a **paginated, filterable catalogue** of published in-stock products with store name included — **without N+1 queries**. Single product detail via `GET /products/:id`.

Blueprint for sub-chapters 14.8–14.14. Specs only — no handler implementations here.

---

## Read this if you're new to public APIs

Until now, product reads required a **vendor token** — "show me **my** inventory." Chapter 14 adds routes anyone can call (no login) that return only **published, in-stock** items from **all** vendors.

Think of two doors into the same warehouse:

| Door | Who walks in | What they see |
|---|---|---|
| Vendor door (`/vendor/products`) | Logged-in vendor | Only their stall's goods, including drafts |
| Public door (`/products`) | Anyone | Every stall's **ready-to-sell** goods on the floor |

Same MongoDB collection. Different **filter** and **auth** rules.

## `GET /products` contract

| Field | Value |
|---|---|
| Method | GET |
| Path | `/products` (under API prefix) |
| Auth | **None** required for browse |
| Default filter | `isPublished: true`, `stockQty > 0` |

### Query parameters

| Param | Type | Default | Behaviour |
|---|---|---|---|
| `page` | integer | `1` | 1-based page index |
| `limit` | integer | `20` | Max items per page; cap e.g. at 100 |
| `category` | string | — | Exact or case-insensitive match — document choice |
| `storeId` | ObjectId string | — | Filter products from one store |

**Future (Ch 20):** `q` search — not required in Ch 14 gate unless you finish early.

### Success 200 body

```json
{
  "items": [
    {
      "id": "...",
      "name": "Organic Tomatoes",
      "price": 3.5,
      "unit": "kg",
      "weightOrQuantity": "500g pack",
      "category": "vegetables",
      "imageUrl": "https://res.cloudinary.com/...",
      "stockQty": 12,
      "store": {
        "id": "...",
        "name": "Green Valley Grocer"
      }
    }
  ],
  "page": 1,
  "limit": 20,
  "total": 47,
  "totalPages": 3
}
```

**Public DTO — exclude:**

- Vendor user ids  
- Unpublished/draft flags on hidden products  
- Internal Cloudinary `imagePublicId` if you added it  
- Cost/margin fields if any

## `GET /products/:id` contract

| Field | Value |
|---|---|
| Auth | None |
| Filter | Published + in-stock **or** return 404 for unpublished/OOS — document choice |

**Recommended:** 404 for unpublished — don't leak draft existence.

**Success 200:** Single product object — same shape as list item, optionally more description fields.

**404:** Invalid id or not publicly visible.

---

## Pagination spec

| Rule | Value |
|---|---|
| Default limit | 20 |
| Max limit | 100 — clamp if client sends 9999 |
| Skip | `(page - 1) * limit` |
| Total count | `countDocuments` with **same filter** as find |
| Empty page | `items: []`, valid `total` — not 404 |

Invalid `page` or `limit` (non-numeric, <1) → 400 with message.

---

## Populate spec

| Ref field | Populate path | Select fields |
|---|---|---|
| `storeId` | `storeId` → Store | `name` (+ `slug` optional) |

Response maps to nested `store: { id, name }` — rename in serializer if populate returns `storeId` object.

**Query count gate:** ≤2 queries for list page (typical populate) or 1 with aggregation.

---

## Filter combination

All filters AND together:

```
isPublished: true
stockQty: { $gt: 0 }
category: param (if present)
storeId: param (if present)
```

Seed must include products matching filter tests — multiple categories, two stores.

## Error matrix

| Condition | Status |
|---|---|
| Invalid page/limit | 400 |
| Invalid storeId format | 400 |
| Success empty catalogue | 200 + empty items |
| Unpublished product by id | 404 |

---

## Verification story (14.14)

1. `GET /products` → 200 + items from multiple stores  
2. `?category=vegetables` → subset  
3. `?storeId=...` → one store only  
4. `?page=2&limit=5` → correct slice + total  
5. Mongoose debug: query count **not** proportional to page size for store names  
6. `GET /products/:id` → detail with store name  

---

## ✅ Do / ❌ Don't

**✅ Do:** Share filter builder between count and find — same match object.

**✅ Do:** Index `isPublished`, `category`, `storeId` if slow — note in learning log.

**❌ Don't:** Reuse vendor list handler without visibility filter — drafts would leak.

**❌ Don't:** Return all products when limit omitted — use default 20.
