# 14.16 — Checklist and learning log

**Why this step matters:** The checklist **gates Chapter 15**. Honest ticks prevent building a shop UI on an API that still leaks drafts, skips pagination, or runs 21 queries per page.

Gate for Chapter 14. Learning log: **`learning-log/14-browse-catalogue-api.md`**.

---

## Definition of Done

### Public routes

- [ ] `GET /products` — no auth required
- [ ] `GET /products/:id` — no auth required
- [ ] Only `isPublished: true` and `stockQty > 0` in public results
- [ ] Draft/unpublished products absent from list and detail (404 on detail)
- [ ] Vendor `GET /vendor/products` unchanged and still auth-protected

### Filters

- [ ] `?category=` filters correctly
- [ ] `?storeId=` filters correctly; invalid id → 400
- [ ] Filters combine with base public filter (AND)

### Pagination

- [ ] Default `page=1`, `limit=20`
- [ ] `limit` capped at 100
- [ ] Response includes `items`, `page`, `limit`, `total`, `totalPages`
- [ ] Invalid page/limit → 400
- [ ] Empty catalogue → 200 with empty items

### Populate & N+1

- [ ] Each item includes `store: { id, name }` (or documented shape)
- [ ] Store owner email / sensitive fields not exposed
- [ ] Query count logged for `limit=20` — **not** 21+ store queries
- [ ] Same mapper used for list and detail

### Public DTO

- [ ] Includes name, price, unit, category, imageUrl
- [ ] Excludes internal fields (imagePublicId, vendor-only data)
- [ ] `imageUrl` from Cloudinary — not embedded bytes

### Testing & docs

- [ ] curl test matrix from 14.14 completed
- [ ] Products from ≥2 stores in default list (seed permitting)
- [ ] README documents public catalogue endpoints
- [ ] CORS allows GET from client dev origin

### Prior gates

- [ ] Chapter 13 complete
- [ ] Chapter 11 vendor writes still work

## Learning log questions

1. **Public filter:** What MongoDB conditions define a "public" product? Why exclude zero stock?

2. **N+1 evidence:** Paste query count from Mongoose debug for one request with limit=20. How many queries? What would naive loop cost?

3. **Pagination:** Walk through computing `skip` and `totalPages` for page 3, limit 10, total 47.

4. **Populate:** How does your service shape `store` in JSON? What store fields are intentionally omitted?

5. **Filters:** Test one category and one storeId filter — describe results vs unfiltered total.

6. **Detail vs list:** Same DTO? What happens for draft product id?

7. **Chapter 15 preview:** Which query params will Shop.jsx pass on first load?

8. **Performance:** Would cursor pagination help this app today? When would you switch?
