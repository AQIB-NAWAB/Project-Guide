# 14.9 — Public products route

**Why this step matters:** This is the **first endpoint customers will use** — the foundation of the whole shop. Getting the visibility filter right here (`published` + in-stock only) prevents draft products leaking to the public before you add filters and pagination.

**Where the project is now:** Only vendor-scoped product reads exist.

**After this step:** `GET /products` returns published in-stock products — pagination and populate added in 14.11–14.12; filters in 14.10.

---

## Route contract

| Field | Value |
|---|---|
| Method | GET |
| Path | `/products` |
| Auth | None |
| Handler | `listPublicProducts(req.query)` |

---

## Public visibility filter (base)

Every public product query **must** include:

```javascript
// Spec — exact syntax yours
{
  isPublished: true,
  stockQty: { $gt: 0 }
}
```

| Product state | In public list? |
|---|---|
| Published, stock > 0 | ✅ |
| Published, stock 0 | ❌ |
| Draft / unpublished | ❌ |

**You implement** in `buildPublicFilter()` — extend with query params in 14.10.
