# 14.11 — Pagination params

**Why this step matters:** Without page/limit, every browse request loads the **entire** catalogue — fine with seed data, broken at scale. Chapter 15's shop will call this on every page click; metadata (`total`, `totalPages`) drives the UI controls.

**Where the project is now:** Filtered public list may return entire catalogue in one response.

**After this step:** `?page=` and `?limit=` slice results; response includes `total`, `totalPages`, `page`, `limit`.

---

## Query parameters

| Param | Default | Max | Validation |
|---|---|---|---|
| `page` | 1 | — | integer ≥ 1 |
| `limit` | 20 | 100 | integer ≥ 1 |

Parse from `req.query` — strings to numbers; reject NaN with 400.

---

## Service logic spec

```javascript
// Pseudocode
const page = parsePage(req.query.page)      // default 1
const limit = parseLimit(req.query.limit)   // default 20, max 100
const skip = (page - 1) * limit
const filter = buildPublicFilter(req.query)

const total = await Product.countDocuments(filter)
const items = await Product.find(filter)
  // populate added in 14.12
  .sort({ createdAt: -1 })  // or your documented default
  .skip(skip)
  .limit(limit)

return {
  items,
  page,
  limit,
  total,
  totalPages: Math.ceil(total / limit) || 0
}
```

Run `countDocuments` and `find` with **identical** filter.

---

## Response contract

```json
{
  "items": [],
  "page": 2,
  "limit": 20,
  "total": 47,
  "totalPages": 3
}
```

When `total === 0`: `totalPages: 0`, `items: []`, `page: 1` — valid.

When `page > totalPages` and total > 0: `items: []` — valid (user bookmarked old page).

---

## Edge cases

| Case | Behaviour |
|---|---|
| `limit=1000` | Clamp to 100 |
| `page=0` | 400 |
| No params | page 1, limit 20 |
| Single item total | totalPages 1 |

---

## Performance

`countDocuments` on every request — acceptable for MVP. Chapter 20 may cache catalogue; note in learning log.

Consider index `{ isPublished: 1, category: 1, storeId: 1 }` if explain shows collection scan.


**❌ Don't:** Omit `total` — Chapter 15 page UI needs it.
