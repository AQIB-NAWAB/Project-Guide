# 14.10 — Filter query params

**Why this step matters:** Real shoppers narrow by category or vendor — "vegetables only" or "this farm stall." Filters belong in the **API** so pagination stays honest; client-side filtering of one page would lie about what exists in the catalogue.

**Where the project is now:** Public list returns all published in-stock products.

**After this step:** Clients narrow catalogue with `?category=` and `?storeId=` — combined with AND logic.

---

## Query parameters

| Param | Type | Example | Filter |
|---|---|---|---|
| `category` | string | `?category=vegetables` | Match product.category |
| `storeId` | ObjectId string | `?storeId=674a...` | Match product.storeId |

Both optional. Omit = no extra constraint.

---

## Category matching spec

Pick one — document in README:

| Strategy | Behaviour |
|---|---|
| Exact match | `category === 'vegetables'` |
| Case-insensitive | `category` lowercased comparison |

Seed categories must match strings used in filters — consistent casing in seed script.

**Invalid category** (empty string) → ignore or 400 — document.

---

## storeId validation

| Input | Response |
|---|---|
| Valid ObjectId | Filter applied |
| Invalid format | 400 `{ message: "Invalid storeId" }` |
| Valid id, no products | 200 + `items: []` |
| Valid id, store exists, all OOS | 200 + empty |

Do not 404 on empty filter result — empty shelf is valid.

---

## Combined filters

```
GET /products?category=fruit&storeId=674a...
```

Equivalent to:

```javascript
{
  isPublished: true,
  stockQty: { $gt: 0 },
  category: 'fruit',
  storeId: ObjectId('674a...')
}
```
