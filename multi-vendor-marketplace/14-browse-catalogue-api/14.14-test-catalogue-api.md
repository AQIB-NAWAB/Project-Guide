# 14.14 — Test catalogue API

**Why this step matters:** curl proof and **query-count logging** are your gate evidence — not "it looks fine in Compass." This step catches N+1 and filter bugs before you build React on top of a broken API.

**Where the project is now:** Public list and detail routes implemented.

**After this step:** Documented curl tests pass; N+1 query count recorded; ready for Chapter 15 UI.

---

## Test matrix

Run unauthenticated unless noted.

| # | Request | Expected |
|---|---|---|
| 1 | `GET /products` | 200; items only published+in-stock |
| 2 | `GET /products?limit=5` | ≤5 items; total ≥ items length |
| 3 | `GET /products?category=X` | All items category X |
| 4 | `GET /products?storeId=Y` | All items from store Y |
| 5 | `GET /products?page=2&limit=5` | Correct page metadata |
| 6 | `GET /products/:publishedId` | 200; store populated |
| 7 | `GET /products/:draftId` | 404 |
| 8 | Compare to vendor list | Draft visible vendor-only, not public |

Save example JSON snippets in learning log (redact ids if sharing publicly).

---

## Multi-vendor check

Response must include products from **≥2 different** `store.id` values when seed supports it — proves marketplace not single-vendor accidentally.

---

## N+1 query count — gate evidence

1. Enable Mongoose debug (dev only)  
2. `GET /products?limit=20`  
3. Count store-related queries in log  
4. Write in learning log: **"N products → M queries"**

**Pass:** M is small constant (typically 2), not 21.

If fail: find loop in service — fix before 14.16 gate.

---

## Response size check

Network tab or curl `-w '%size_download'`:

- JSON should be kilobytes for page of 20 with URLs — not megabytes  
- `imageUrl` present; no embedded image data
