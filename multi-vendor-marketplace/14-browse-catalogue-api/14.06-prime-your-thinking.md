# 14.6 — Prime your thinking

Before implementing pagination in 14.11, read how **offset** and **cursor** pagination differ — you implement offset for this chapter; cursor is the scale-up path mentors may ask about in viva.

---

## Why read before you build

You are about to write **four related features** in sequence: filters → pagination → populate → detail route. Ten minutes of reading now prevents:

- Applying populate to the **whole collection** instead of the **current page** (slow + wrong mental model)
- Different filters on `countDocuments` vs `find` (pagination shows wrong total pages)
- Skipping query-count verification and shipping N+1 to Chapter 15's shop

This sub-chapter is **not** optional fluff — it is the pause before the build cluster where mistakes compound.

---

## Mandatory reads

1. **Mongoose — Query populate**  
   Focus: `select`, nested populate, lean queries.

2. **MongoDB — Pagination** (official docs or trusted blog)  
   Focus: skip/limit vs range queries on `_id` or `createdAt`.

3. **Your Chapter 11 product list** — if vendor list exists, compare filters; public list is wider scope.

Skim: MongoDB compound indexes for `{ isPublished: 1, category: 1 }` — optional optimization note in learning log.

---

## Offset vs cursor — decision table

| | Offset (`page` + `limit`) | Cursor (`after` + `limit`) |
|---|---|---|
| UX | Page numbers — familiar | Infinite scroll — common on mobile |
| Deep pages | Slow (large skip) | Stable speed |
| Implementation | Simple | Needs encoded cursor token |
| Jump to page 5 | Easy | Hard |
| **This chapter** | **✅ mandated** | Interesting read only |

## Questions for learning log draft

1. Why does `skip(10000)` get expensive at scale?  
2. What cursor field would you use for products — `_id` or `createdAt`?  
3. How does populate affect pagination — populate before or after limit? (**After** — limit products first, then populate page only.)  
4. Should `total` count run on every request — cache in Ch 20?  
5. What happens when a product is unpublished between page 1 and page 2 load — acceptable inconsistency for MVP?
