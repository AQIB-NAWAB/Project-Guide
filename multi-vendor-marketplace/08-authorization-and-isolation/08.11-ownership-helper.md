# 8.11 — Ownership helper

## Why this step matters

Role checks are not enough — two vendors share the same role. Ownership helpers load the **object** and compare its owner field to `req.user.id`. This is the core IDOR defense. You'll clone the pattern for products in Chapter 11.

---

**Where the project is now:** requireRole blocks wrong roles; no reusable store ownership check.

**After this step:** `assertStoreOwner(storeId, userId)` in shared module — loads Store, compares userId, throws or returns error for 403. Optional Express wrapper middleware for `:id` param routes.

---

## Core helper contract

**File:** `server/src/shared/ownership.js` (or `stores/stores.authorization.js`)

```
assertStoreOwner(storeId, userId) → Promise<Store>
```

**Behaviour:**

1. Validate storeId is valid ObjectId format — 400 if malformed optional  
2. `store = await Store.findById(storeId)`  
3. If !store → throw NotFoundError (404) or ForbiddenError (403) — document choice  
4. If store.userId.toString() !== userId.toString() → throw ForbiddenError  
5. Return store for handler use — avoids double fetch  

---

## Express middleware wrapper (optional)

```
assertStoreOwnerFromParams(req, res, next)
  → reads req.params.id
  → calls assertStoreOwner(req.params.id, req.user.id)
  → attaches req.store = store
  → next() or 403
```

Reuse on all `/stores/:id` mutating routes.

## Product ownership preview (Ch 11)

Add later:

```
assertProductOwner(productId, userId)
  → load Product → assertStoreOwner(product.storeId, userId)
```

Chain reuse — don't duplicate store.userId compare in product handlers.
