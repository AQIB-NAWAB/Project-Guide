# 8.8 — Authorization matrix

## Why this step matters

Writing middleware before you know the rules produces inconsistent guards — one route checks role, another forgets ownership. The matrix is your **spec**: every action, every role, every check type, committed before code. Chapters 9–18 will add rows; Chapter 8 sets the pattern.

---

**Where the project is now:** Auth works; no `docs/AUTHORIZATION.md`; no role/ownership middleware.

**After this step:** **`docs/AUTHORIZATION.md`** committed — action × role × check type matrix driving all later chapter guards.

## Required sections

### 1. Purpose

One paragraph: authn (JWT) vs authz (role + ownership). Link to ownership chain in erd.md.

### 2. Roles

| Role | Capabilities summary |
|---|---|
| vendor | Own store, products, vendor orders |
| customer | Cart, checkout, own orders |
| anonymous | Public catalogue only (Ch 14) |

### 3. Master matrix table

Minimum rows — extend as course progresses:

| Action | Route (current/planned) | Authn | Role | Ownership / scope | Deny status |
|---|---|---|---|---|---|
| Health | GET /health | none | — | — | — |
| Register | POST /api/auth/register | none | — | — | — |
| Me | GET /api/auth/me | JWT | any | self | 401 |
| Patch store | PATCH /api/stores/:id | JWT | vendor | store.userId = req.user.id | 403 |
| Create store | POST /api/stores | JWT | vendor | one store per user (Ch 9) | 403 |
| Create product | POST /api/products | JWT | vendor | product.storeId = own store | 403 |
| Patch product | PATCH /api/products/:id | JWT | vendor | via store chain | 403 |
| Public catalogue | GET /api/products | optional | — | published only | — |
| Get cart | GET /api/cart | JWT | customer | cart.userId | 403 |
| Checkout | POST /api/checkout | JWT | customer | own cart | 403 |
| Vendor orders | GET /api/vendor/orders | JWT | vendor | order.storeId → own store | 403 |
| Customer orders | GET /api/orders/my | JWT | customer | order.customerId | 403 |

Mark rows "planned" for future chapters — honest status column OK.

### 4. Check types glossary

| Type | Meaning |
|---|---|
| none | Public |
| JWT | requireAuth |
| role | requireRole(...) |
| ownership | DB lookup chain |
| scope | query filter { userId: req.user.id } |
