# 8.14 — Recap and what's next

Chapter 8 closes the security gap between **authenticated** and **authorized**. Vendor B cannot mutate Vendor A's store — proven with curl and documented in `docs/AUTHORIZATION.md`.

## What you built

**docs/AUTHORIZATION.md:** Matrix of action × role × check type; IDOR checklist; 401 vs 403 rules.

**requireRole:** Function-level RBAC — vendor vs customer routes.

**assertStoreOwner:** Object-level ownership via Store.userId.

**PATCH /api/stores/:id:** Full middleware stack + minimal update — pattern for all vendor mutations.

**IDOR test:** Vendor B → 403 on Vendor A store; data unchanged.

## Authorization stack (reference)

```
Request

  → handler (400 validation / 200 success)
```
