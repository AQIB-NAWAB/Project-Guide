# 8.5 — RBAC and ownership

FreshMarket uses **two-layer authorization**: role-based access control (RBAC) at the route, then **ownership chain lookup** at the object.

---

## Roles (RBAC)

| Role | Description |
|---|---|
| vendor | Operates one store, products, vendor orders |
| customer | Browses, cart, checkout, own orders |

No admin role in MVP.

**requireRole('vendor')** — rejects customer token with 403 before handler runs.

---

## Ownership chain lookup

For mutating Store identified by `:id`:

```
1. store = await Store.findById(storeId)
2. if !store → 404 Not Found
3. if store.userId.toString() !== req.user.id → 403 Forbidden
4. proceed
```

For Product (Ch 11 preview):

```
1. product = await Product.findById(productId)
2. store = await Store.findById(product.storeId)
3. if store.userId !== req.user.id → 403
```

**Never trust client-sent userId or storeId in body** for authorization — derive from token + DB.

---

## Helper function contract

```
assertStoreOwner(storeId, userId) → void | throws ForbiddenError
```

Centralize in `shared/ownership.js` — used by store routes, product routes, order status updates.

Throws or returns `{ ok: false }` — consistent error type for route handlers.

---

## Combined stack example

**PATCH /api/stores/:id**

| Step | Middleware / logic |
|---|---|
| 1 | requireAuth |
| 2 | requireRole('vendor') |
| 3 | assertStoreOwner(req.params.id, req.user.id) |
| 4 | storesService.updateStore(...) |

Customer never reaches step 3 with vendor-only route — blocked at step 2.

Vendor B reaches step 3 — blocked with 403.

---

## Customer-scoped resources

Cart and orders use **direct userId match**:

```
CartItem.find({ userId: req.user.id })
Order.find({ customerId: req.user.id })
```

No store chain — different ownership model.

---

## Matrix preview (full doc in 8.8)

| Action | Role | Ownership check |
|---|---|---|
| PATCH /stores/:id | vendor | store.userId |
| POST /products | vendor | caller's store exists |
| PATCH /products/:id | vendor | product → store → userId |
| GET /cart | customer | cart.userId |
| GET /vendor/orders | vendor | order.storeId → store.userId |

---

## Performance note

Ownership check adds one DB read per mutating request — acceptable for MVP. Cache storeId on req in handler if multiple checks — optional.

---

## ✅ Do / ❌ Don't

**✅ Do:** Compare ObjectIds with `.toString()` or `equals()`.

**✅ Do:** 404 when store id invalid — optional vs 403 hide existence.

**❌ Don't:** Skip store lookup — trusting :id from URL without DB verify.
