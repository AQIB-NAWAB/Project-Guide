# 8.2 — IDOR and marketplaces

**Insecure Direct Object Reference (IDOR)** — also called **BOLA** (Broken Object Level Authorization) in OWASP API Top 10 — is the #1 marketplace breach pattern: change an id in the URL, access someone else's data.

---

## Classic attack

Vendor A is authenticated. Legitimate request:

```
PATCH /api/stores/STORE_A_ID
Authorization: Bearer <vendor A token>
```

Attacker changes id:

```
PATCH /api/stores/STORE_B_ID
Authorization: Bearer <vendor A token>
```

**Vulnerable API:** Updates Store B — Vendor A renamed competitor's shop.

**Secure API:** **403 Forbidden** — ownership check failed.

---

## Why marketplaces are high risk

| Resource | IDOR impact |
|---|---|
| Store | Rebrand competitor, change address |
| Product | Change price to £0.01, delete inventory |
| Order | View customer PII, mark shipped falsely |
| Cart | Merge or view another user's cart |

One missing check on PATCH becomes front-page incident.

---

## Auth without authorization fails

```
✅ requireAuth — proves token valid
❌ stops here — any vendor hits any storeId
✅ requireRole('vendor') — proves role
❌ stops here — Vendor A still hits Store B
✅ assertStoreOwner(storeId, req.user.id) — proves object ownership
```

All three layers for mutating vendor resources.

---

## Real-world pattern

API returns 200 with generic message — attacker enumerates ids. Use **403** consistently; optional 404 to hide existence — course uses **403** for clarity in tests.

---

## FreshMarket test scenario (8.13)

| Step | Actor | Action | Expected |
|---|---|---|---|
| 1 | Vendor A | Login → get token A | 200 |
| 2 | Vendor A | GET own store id from seed/Compass | — |
| 3 | Vendor B | Login → token B | 200 |
| 4 | Vendor B | PATCH Store A id with token B | **403** |

Document store ids in test notes — not secrets.

---

## Client is not trusted

Hidden form fields, React props, disabled buttons — attacker uses curl. **Every mutating route validates server-side.**

---

## ✅ Do / ❌ Don't

**✅ Do:** Test IDOR with two seed vendors every chapter adding PATCH/DELETE.

**✅ Do:** Log ownership failures in dev — not in production response body.

**❌ Don't:** Rely on UI hiding edit buttons for security.

**❌ Don't:** Return 401 for wrong owner — user IS authenticated → **403**.
