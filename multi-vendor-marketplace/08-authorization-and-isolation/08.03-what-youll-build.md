# 8.3 — What you'll build

Chapter 8 adds **authorization infrastructure** and proves it with a stub store route and two-vendor IDOR test.

| Deliverable | Location |
|---|---|

## Stub route scope

**Not** full store CRUD — Chapter 9 adds POST, GET /me, full PATCH body.

Chapter 8 PATCH may accept minimal body `{ name: "..." }` — enough to prove ownership check runs **before** service mutates document.

---

## Middleware stack order

```
requireAuth → requireRole('vendor') → assertStoreOwnerParam → handler
```

Order matters: auth before role before ownership.

---

## docs/AUTHORIZATION.md structure

1. Introduction — authn vs authz  
2. Roles: vendor, customer  
3. Matrix table — action × role × check type  
4. Ownership rules — store, product, cart, order  
5. HTTP status conventions — 401 vs 403  
6. Test checklist — IDOR scenarios  

Build matrix in **8.8** before middleware — design before code.

---

## Status code contract

| Situation | Status |
|---|---|
| No/invalid token | 401 |
| Valid token, wrong role | 403 |
| Valid token, wrong owner | 403 |
| Valid token, owner, bad body | 400 |
