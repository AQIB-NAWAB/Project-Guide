# 2.8 — Create the repo

**Why this step matters:** Everything you build for FreshMarket lives in a repo you own — not in the course guides folder. This step creates that root with Git, ignore rules, and a learning log so every later sub-chapter has a committed home and secrets stay out of history from day one.

**Where the project is now:** You have no application repository — only the course materials you read. After this sub-chapter you'll have a Git root with ignore rules and a place for the learning log.

---

## What you must do

Create a **new directory** for your marketplace application — **not** inside the course guides folder. Initialize Git, add a root `.gitignore`, stub `README.md`, and create `learning-log/` with your Chapter 1 answers (or add them now if missing).

---

## Step 1 — Create the project directory

Choose a name matching your product — examples: `greenbasket`, `freshmarket-app`, `local-harvest`.

```bash
mkdir greenbasket
cd greenbasket
```

This directory is the **monorepo root**. `server/` and `client/` will be created in the next sub-chapters.

---

## Step 2 — Initialize Git

```bash
git init
```

Use the default branch name your Git version provides (`main` preferred). Do not commit secrets in this or any future commit.

---

## Step 3 — Root `.gitignore`

Create `.gitignore` at the **repository root** with at least:

```gitignore
# Dependencies — reinstall from package.json
node_modules/

# Environment files — secrets never in Git
.env
.env.local
.env.*.local

# Logs
npm-debug.log*
*.log

# OS / editor
.DS_Store
Thumbs.db
.idea/
.vscode/*
!.vscode/extensions.json

# Build output
dist/
build/

# Optional: coverage
coverage/
```

**✅ Do:** Verify ignore works *before* first commit:

```bash
touch .env
git status    # .env should NOT appear as untracked if ignored — 
              # if it appears, fix .gitignore first
rm .env       # you'll recreate server/.env in Chapter 4
```

**❌ Don't:** `git add -f .env` "just once" — ever.

---

## Step 4 — Stub `README.md`

Create a minimal root README you'll expand through the course:

```markdown
# GreenBasket

Multi-vendor local grocery marketplace — MERN stack.

## Structure

- `server/` — Express API
- `client/` — React (Vite) frontend
- `learning-log/` — chapter notes and viva prep

## Run locally

(Completed in Chapter 2 — fill in after sub-chapter 2.16)

### API
```bash
cd server && npm install && npm run dev
```

### Client
```bash
cd client && npm install && npm run dev
```

## Author

Your name — course project
```

Replace **GreenBasket** with your product name.

---

## Step 5 — Learning log folder

```bash
mkdir -p learning-log
```

Ensure `learning-log/01-introduction.md` exists with your Chapter 1 answers. If you wrote it elsewhere, move it here.

You'll add `learning-log/02-project-skeleton.md` at the chapter gate (2.19).

---

## Step 6 — Optional `docs/` placeholder

```bash
mkdir -p docs
```

Empty is fine. Chapter 3 adds ERD draft; Chapter 8 adds authorization matrix.

---

## Step 7 — First commit

```bash
git add .
git status    # review: no .env, no node_modules
git commit -m "chore: initialize repository with gitignore and learning log"
```

---

## Verify

| Check | Expected |
|---|---|
| `git log` | One commit |
| `git status` | clean working tree |
| `learning-log/` exists | yes |
| `.gitignore` lists `.env` | yes |

## ✅ Do / ❌ Don't

**✅ Do:** Commit after each major sub-chapter through Chapter 2 (2.11, 2.15, 2.16).

**❌ Don't:** Clone someone else's marketplace repo and erase their code — start empty; the course builds it with you.

**❌ Don't:** Put `server/` or `client/` outside this repo root — one product, one history.

---

## Hint

If `git status` shows thousands of files, you accidentally initialized Git in your home directory or parent folder. `cd` to the intended project folder only, remove mistaken `.git` if needed, and re-init **inside** `greenbasket/`.
