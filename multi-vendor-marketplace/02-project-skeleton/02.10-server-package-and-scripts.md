# 2.10 — Server package and scripts

**Why this step matters:** `npm run dev` is how you (and every reviewer) start the API. Installing Express and wiring scripts turns a folder tree into a **runnable server process** — the foundation for the health route in the next step.

**Where the project is now:** Server folder tree with stub `app.js` and `server.js`. No dependencies installed; `npm run dev` doesn't exist yet.

**After this step:** `server/package.json` with Express installed; dev script starts the API with file watching.

---

## Step 1 — Initialize package.json

```bash
cd server
npm init -y
```

Edit `server/package.json` to include at minimum:

```json
{
  "name": "greenbasket-server",
  "version": "1.0.0",
  "private": true,
  "type": "module",
  "scripts": {
    "dev": "node --watch src/server.js",
    "start": "node src/server.js"
  }
}
```

Replace `greenbasket-server` with your project name.

**About `"type": "module"`:** Enables `import`/`export` syntax in `.js` files. If you chose CommonJS, omit this line and use `require` throughout.

**About `node --watch`:** Built into Node 18+ — restarts server when files change. Alternative: install `nodemon` and use `"dev": "nodemon src/server.js"`.

---

## Step 2 — Install Express

```bash
npm install express
```

Optional now, required before 2.15 (browser calls):

```bash
npm install cors
```

You may install `cors` in 2.15 instead — but installing now is fine.

---

## Step 3 — Implement minimal `app.js` and `server.js`

Now wire stubs into a **bootable** server — still without health route if you prefer one step at a time, but the process must stay up.

**`app.js` must:**

- Create Express app
- Use `express.json()`
- Export `app`

**`server.js` must:**

- Import app
- Listen on `process.env.PORT || 5000`
- Log `API listening on port XXXX`

**You write the code.** If the server crashes on import, read the stack trace — usually typo in import path or missing `"type": "module"`.

---

## Step 4 — Run dev script

```bash
npm run dev
```

**Expected:** Process stays running; terminal shows listen log; no unhandled crash.

**Test:** Open browser or curl to `http://localhost:5000/` — likely 404 until health route exists. **404 from Express proves the server runs** — different from connection refused.

Stop with Ctrl+C when verified.

## Troubleshooting

| Problem | Likely fix |
|---|---|
| `Cannot use import statement outside a module` | Add `"type": "module"` to package.json |
| `ERR_MODULE_NOT_FOUND` | Check import paths include `.js` extension in ESM |
| Port already in use | Kill other process on 5000 or set `PORT=5001` |
| Server exits immediately | `listen` not called; uncaught exception — read stack trace |

---

## ✅ Do / ❌ Don't

**✅ Do:** Commit `package-lock.json` — reproducible installs for mentors and deploy.

**❌ Don't:** Commit `server/node_modules/` — must be in `.gitignore` (root ignore covers nested node_modules).

**❌ Don't:** Hardcode port without `process.env.PORT` fallback — deploy hosts set PORT dynamically.
