## What you built — narrative recap

**Repository & habits:** Git initialized; `.gitignore` protects secrets; `learning-log/` established; README stub documents run commands.

**Structural decision:** Module-based server layout mandated — `modules/auth`, `stores`, `products`, `cart`, `orders` plus `shared/` for cross-cutting code.

**Server:** Express app separated into `app.js` (configure) and `server.js` (listen). `GET /health` returns JSON `{ "status": "ok" }`. CORS enables browser access from Vite dev server.

**Client:** Vite React with `pages/`, `components/`, `api/`, `hooks/`. React Router serves Home at `/`. `api/client.js` centralizes calls to `VITE_API_URL`. Home fetches and displays real health status.

**Proof:** curl and browser both confirm the API; stopping server surfaces error UI — not a mock.

## Decisions you can defend in a viva

Prepare short answers:

1. **Why module-based folders?** — Cohesion per domain; marketplace has many domains; course paths align.

2. **Why two entry files on server?** — Separate app wiring from listen for testability.

3. **Why CORS?** — Browser same-origin policy; client and API on different ports.

4. **Why `api/client.js`?** — Single place for base URL and future auth headers.

5. **Why health route first?** — Proves HTTP pipeline before MongoDB complexity.
