# 2.16 — Verify end-to-end

**Why this step matters:** Informal "it works on my machine" is not a gate. This verification pass documents curl output, browser screenshot, and folder tree — evidence that Week 1's first demo brick is real before you open the database chapters.

**Where the project is now:** Home fetches `/health`; server and client both run. This sub-chapter is a **formal verification pass** — treat it like a mini QA checklist before you claim Chapter 2 done.

---

## Full demo script

Perform these steps **in order** while recording optional screen capture for your portfolio:

### 1. Clean start

```bash
# Stop any running dev servers
# From repo root
cd server && npm run dev
# New terminal
cd client && npm run dev
```

### 2. API alone

```bash
curl -s http://localhost:5000/health | jq .
```

If `jq` not installed, use raw curl:

```bash
curl -i http://localhost:5000/health
```

**Pass criteria:** HTTP 200; body contains `"status":"ok"`.

### 3. Client alone

Open `http://localhost:5173`.

**Pass criteria:**

- Home loads without white screen  
- No uncaught errors in browser console (warnings OK)  
- API status area shows success with server running  

### 4. Failure mode

Stop server terminal. Refresh browser.

**Pass criteria:** Home shows **error state** — not blank, not fake ok.

Restart server. Refresh.

**Pass criteria:** Success returns.

### 5. Git hygiene

```bash
git status
```

**Pass criteria:** No `.env` staged; no `node_modules`; working tree clean or only intentional changes.

```bash
git log --oneline -5
```

**Pass criteria:** Multiple commits through Chapter 2 — not one giant commit.

## Ports and env audit

| Setting | Expected value |
|---|---|
| Server `PORT` | 5000 unless you changed consistently |
| `VITE_API_URL` | Matches server origin exactly |
| CORS origin | Matches Vite URL (5173 default) |

If you changed any port, **grep** repo for old port references:

```bash
grep -r "5000" --include="*.md" --include="*.env*" --include="*.js" --include="*.jsx" .
```

Consistency prevents mysterious failures in Chapter 7.

## Common failures at this stage

| Failure | Diagnosis |
|---|---|
| CORS error | Server cors origin mismatch |
| `status ok` in UI but curl fails | You hardcoded UI — fix 2.15 |
| curl works, UI errors | Wrong `VITE_API_URL`; restart Vite |
| Both fail | Server not running or wrong port |
| `import` errors in server | ESM path missing `.js` extension |

Fix before proceeding — Chapter 4 adds MongoDB on top of this skeleton.
