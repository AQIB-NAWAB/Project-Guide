# 2.9 — Scaffold server folders

**Why this step matters:** Empty module folders are not busywork — they are **labeled shelves** for auth, stores, products, cart, and orders. Creating them now means Chapter 5–18 instructions always point at paths that already exist in your tree.

**Where the project is now:** Git repo with `learning-log/`, `docs/`, `.gitignore`, `README.md`. No `server/` yet.

**After this step:** Server directory tree exists with empty module placeholders and entry file stubs (may be empty exports until 2.10–2.11).

---

## Step 1 — Create directories

From repository root:

```bash
mkdir -p server/src/shared
mkdir -p server/src/modules/auth
mkdir -p server/src/modules/stores
mkdir -p server/src/modules/products
mkdir -p server/src/modules/cart
mkdir -p server/src/modules/orders
```

Git doesn't track empty folders — add `.gitkeep` (empty file) in each empty directory:

```bash
touch server/src/shared/.gitkeep
touch server/src/modules/auth/.gitkeep
touch server/src/modules/stores/.gitkeep
touch server/src/modules/products/.gitkeep
touch server/src/modules/cart/.gitkeep
touch server/src/modules/orders/.gitkeep
```

---

## Step 2 — Create `app.js` stub

Create `server/src/app.js`. **Your job:** export a function or object that **creates and configures** an Express application.

**Requirements (implement yourself):**

- Import Express and create `const app = express()`
- Register `express.json()` middleware so future POST bodies parse as JSON
- Export `app` as default export (ESM) or `module.exports` (CommonJS) — pick one style and use it consistently; below assumes **ESM**

**Do not** call `app.listen()` in this file.

**Placeholder comment block** you should replace as you build:

```javascript
// server/src/app.js — YOU implement
// - create express app
// - app.use(express.json())
// - mount routes (health in 2.11)
// - export app
```

---

## Step 3 — Create `server.js` stub

Create `server/src/server.js`. **Your job:**

- Import `app` from `./app.js`
- Read `PORT` from `process.env.PORT || 5000`
- Call `app.listen(PORT, callback)` that logs a clear message including the port

**Do not** put route definitions here — only startup.

---

## Step 4 — Commit structure

```bash
git add server/
git commit -m "chore(server): scaffold module folders and entry stubs"
```

## ✅ Do / ❌ Don't

**✅ Do:** Keep module names **plural** (`products`, `orders`) — matches REST resource naming.

**✅ Do:** Split `app.js` (configure) and `server.js` (listen) as explained in 2.5.

**❌ Don't:** Create `controllers/` and `services/` top-level folders alongside `modules/` — pick module-based only.

**❌ Don't:** Implement health route yet — 2.11 after Express installs.
