# 2.13 — Client shell and routing

**Why this step matters:** A single Home route is enough for Chapter 2, but you need a **router shell** so Chapter 7 adds `/login` and `/register` without restructuring the app. This step replaces the Vite default with your marketplace entry point.

**Where the project is now:** Vite React app runs default template. Course folders exist but Home page not wired.

**After this step:** React Router sends `/` to a **Home** page component — minimal placeholder text acceptable before health fetch.

---

## Step 1 — Install React Router

```bash
cd client
npm install react-router-dom
```

Use React Router v6 API (`BrowserRouter`, `Routes`, `Route`).

---

## Step 2 — Create Home page

Create `client/src/pages/Home.jsx`.

**Requirements:**

- Export a functional component `Home`
- Render a heading with your product name (e.g. "GreenBasket")
- Render a short subtitle — e.g. "Local grocery marketplace"
- Include a **placeholder** element where API status will appear later — empty `<p>` or `<div id="api-status">` with text "Checking API…" or similar

**Do not hardcode `{ status: 'ok' }` as fake success** — 2.15 fetches real data.

**You write the JSX and styling.** Clean editorial style: simple typography, generous spacing — no design system required.

---

## Step 3 — Wire router in `App.jsx`

**Requirements for `App.jsx`:**

- Wrap routes in `BrowserRouter`
- Define route `/` → `Home` component
- Remove Vite boilerplate (logos, counter demo) — this is your app now

**Optional:** Add a minimal layout wrapper (header with product name) — keeps future nav consistent.

---

## Step 4 — Clean `main.jsx`

Ensure `main.jsx` renders `<App />` inside `StrictMode` as Vite template does — no change needed unless you removed it.

---

## Step 5 — Run and verify

```bash
npm run dev
```

Open `http://localhost:5173/`.

**Expected:**

- Home page shows your product heading  
- No router errors in browser console  
- Placeholder for API status visible  

Server does **not** need to run for this sub-chapter verification — only client routing.

## Editorial styling note (minimal)

Clean editorial style for Home means:

- One strong heading (product name)  
- One muted subtitle (tagline)  
- Neutral background — white or off-white  
- System font stack or one Google Font — avoid loading five weights  

You are not graded on design — but avoid default Vite purple-on-dark demo styling if it fights "local grocery." A five-minute CSS pass is enough.
