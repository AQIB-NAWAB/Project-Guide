# 2.17 — Practice (optional stretch)

This sub-chapter is **optional**. Clear the Chapter 2 gate in **2.19** without it. Complete it if you want extra practice wiring **server route → client display** before Chapter 3 adds concepts without new UI.

---

## Stretch goal

Add a second public endpoint on the API and surface it on Home (or a footnote section).

---

## Server contract to implement

| Method | Path | Auth | Success |
|---|---|---|---|
| GET | `/version` | None | `200` JSON |

Response body **minimum:**

```json
{
  "version": "0.1.0",
  "name": "GreenBasket API"
}
```

Use your product name and a semver string you choose. **You implement the route** — same pattern as `/health`.

Mount at root `/version` or under `/api/version` — if you prefix, update client helper path to match.

---

## Client task

1. Add `getVersion()` to `api/client.js` (or extend generic GET helper)
2. On Home, fetch version after health succeeds — or in parallel with `Promise.all`
3. Display version string under API status

**Handle errors** — version fetch failing shouldn't crash page if health succeeded (decouple state).

---

## Verify

```bash
curl -s http://localhost:5000/version
```

Browser shows version string matching API.

---

## Why this exercise helps

Repeats the Chapter 2 loop without new concepts:

```
define contract → implement route → add client helper → render in UI → curl + browser verify
```

Chapter 6 adds ten routes — muscle memory from this stretch reduces friction.
