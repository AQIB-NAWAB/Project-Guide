## Definition of Done

### Repository & structure

- [ ] Git repository exists at your marketplace project root (separate from course content folder)
- [ ] Root `.gitignore` excludes `node_modules/`, `.env`, and build output
- [ ] `git log --all -- "**/.env"` returns **empty** (no env file ever committed)
- [ ] `learning-log/` exists with `01-introduction.md` and `02-project-skeleton.md`
- [ ] Root `README.md` documents how to run server and client in two terminals

### Server

- [ ] `server/package.json` has `dev` and `start` scripts
- [ ] `server/src/app.js` creates Express app, uses JSON middleware, exports app without listening
- [ ] `server/src/server.js` starts server on `process.env.PORT || 5000`
- [ ] Module folders exist: `auth`, `stores`, `products`, `cart`, `orders`
- [ ] `server/src/shared/` exists
- [ ] `npm run dev` in `server/` runs without crash
- [ ] `curl -i http://localhost:5000/health` returns **200** with JSON containing `"status":"ok"`

### Client

- [ ] Vite React app in `client/` with `pages/`, `components/`, `api/`, `hooks/`
- [ ] React Router renders Home at `/`
- [ ] `client/.env.example` documents `VITE_API_URL`
- [ ] `client/src/api/client.js` reads base URL from env — no hardcoded localhost in components
- [ ] `npm run dev` in `client/` runs without crash
- [ ] Home page displays **live** health status from API when server is running
- [ ] Home page displays **error state** when server is stopped (not hardcoded success)

### Integration

- [ ] CORS configured so browser can call API from Vite origin
- [ ] Browser Network tab shows successful `GET` to `/health` with 200
- [ ] End-to-end verification from **2.16** completed

### Understanding (self-check — explain aloud)

- [ ] I can explain **module-based** vs layer-based structure and why this course chose modules
- [ ] I can explain what runs in **browser** vs **Node server**
- [ ] I can explain why **`app.js` and `server.js` are separate**
- [ ] I can point to where **auth** and **products** code will live in my tree
