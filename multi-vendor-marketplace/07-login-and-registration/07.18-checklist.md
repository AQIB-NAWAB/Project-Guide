## Definition of Done

### Pages & routing

- [ ] `/register`, `/login`, `/verify-otp` routes work
- [ ] Register → verify → login flow completes in browser
- [ ] Role-based redirect after login (vendor vs customer)
- [ ] Login shows error for wrong password and unverified user

### Auth state

- [ ] useAuth or AuthProvider manages user + access token in memory
- [ ] Boot refresh restores session on page reload (or documented limitation fixed)
- [ ] Logout clears state and blocks protected routes

### API client

- [ ] apiFetch attaches Authorization Bearer when authenticated
- [ ] 401 triggers refresh once then retry (tested or short-expiry verified)
- [ ] credentials: 'include' on refresh request
- [ ] Access token **not** in localStorage or sessionStorage

### Protected routes

- [ ] ProtectedRoute redirects unauthenticated users to login
- [ ] Vendor stub page reachable only when logged in as vendor (if role prop used)
- [ ] isLoading prevents redirect flash during boot

### Integration

- [ ] Seed vendor-a and customer-a login via UI
- [ ] CORS allows credentials from Vite origin
- [ ] No new server auth endpoints required beyond Ch 6 (cookie Set-Cookie OK)

### Prior gates

- [ ] Ch 6 auth API still passes curl tests
- [ ] Ch 5 models/seed intact
