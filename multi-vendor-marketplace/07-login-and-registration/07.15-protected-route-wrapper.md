# 7.15 — Protected route wrapper

**Why this step matters:** Public Home and auth pages stay open; vendor stubs must reject strangers. `ProtectedRoute` is the client-side door check — authorization by role deepens in Chapter 8, but unauthenticated users should never see vendor screens.

**Where the project is now:** Auth state exists; vendor routes publicly accessible.

**After this step:** `ProtectedRoute` component redirects unauthenticated users to `/login`, preserves intended URL optional, waits for auth boot loading.

---

## File

```
client/src/components/ProtectedRoute.jsx
```

---

## Component contract

**Props:**

| Prop | Type | Purpose |
|---|---|---|
| children | node | protected page |
| roles | string[] optional | if set, require role match — basic check; full RBAC Ch 8 |

**Behaviour:**

```
if isLoading → return loading UI or null
if !isAuthenticated → <Navigate to="/login" replace state={{ from: location }} />
if roles && !roles.includes(user.role) → <Navigate to="/" /> or 403 page stub
return children
```

Chapter 8 adds proper 403 UX — here optional role prop preview.

---

## Usage in router

```
<Route path="/vendor" element={
  <ProtectedRoute roles={['vendor']}>
    <VendorHomeStub />
  </ProtectedRoute>
} />
```

Customer-only route example with `roles={['customer']}` optional.
