# 7.10 — Auth context or hook

**Why this step matters:** Login state shared across pages needs one source of truth — `useAuth` or Context — so Header, protected routes, and the API client all agree whether someone is signed in without prop drilling.

**Where the project is now:** Auth pages scaffolded; no shared auth state.

**After this step:** `useAuth` hook (via AuthProvider) manages user, access token in memory, login/logout/register/verify methods, boot refresh, and `isLoading` gate.

---

## File

```
client/src/hooks/useAuth.js
```

Or split:

```
client/src/context/AuthContext.jsx
client/src/hooks/useAuth.js   ← re-exports useContext
```

---

## Context value contract

| Property / method | Type | Purpose |
|---|---|---|
| user | object \| null | id, email, role, displayName |
| isAuthenticated | boolean | derived |
| isLoading | boolean | boot refresh in progress |
| login(email, password) | async | POST login, set tokens |
| register(data) | async | POST register |
| verifyOtp(email, otp) | async | POST verify |
| logout() | async | clear state, call logout API |
| getAccessToken() | fn | for api client — internal |

Do not expose setAccessToken publicly.

---

## Provider placement

Wrap router in `main.jsx`:

```
<AuthProvider>
  <App />
</AuthProvider>
```

---

## Boot refresh (on mount)

```
useEffect once:
  try POST /auth/refresh credentials include
  if ok: set access, optionally GET /me
  catch: leave logged out
  finally: isLoading false
```

---

## Access token storage

Use `useRef` for accessToken string — update when login/refresh succeeds.

Pass getter to api client module:

```
setAuthTokenGetter(() => getAccessToken())
```

Pattern avoids stale closure in interceptor.

---

## User state after login

Option A: login response includes user — set directly

Option B: login then GET /me — richer profile

Either acceptable — document choice.

## ✅ Do / ❌ Don't

**✅ Do:** Set isLoading false even on refresh failure.

**❌ Don't:** Persist access token to localStorage.

**❌ Don't:** Call useAuth outside provider — guard with error in hook.
