# 7.9 — Scaffold auth pages

**Why this step matters:** Empty page components and router entries give Register, Login, and Verify OTP **named homes** before you wire forms. Adding routes now prevents a single mega-component auth file later.

**Where the project is now:** No auth pages in client.

**After this step:** `Login.jsx`, `Register.jsx`, `VerifyOtp.jsx` exist with placeholder forms; React Router paths registered; navigation links from Home optional.

---

## Files to create

```
client/src/pages/Login.jsx
client/src/pages/Register.jsx
client/src/pages/VerifyOtp.jsx
```

## Placeholder UI spec

Each page minimum:

- `<h1>` title  
- Form with labeled inputs (no submit logic yet OK)  
- Link to sibling pages ("Already have account? Login")  
- Basic layout — match Home editorial style  

**Register fields:** email, password, confirm password (client-side), role select vendor/customer, displayName optional

**Login fields:** email, password

**VerifyOtp fields:** email (pre-filled from state), otp 6 digits
