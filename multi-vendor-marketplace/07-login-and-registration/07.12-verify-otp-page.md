# 7.12 — Verify OTP page

**Why this step matters:** Without this screen, `isVerified` stays false and login always fails — the registration story feels broken. This page closes the loop between register API and usable accounts.

**Where the project is now:** Register navigates here with email; verify not wired.

**After this step:** VerifyOtp submits to `POST /api/auth/verify-otp`, success navigates to `/login` with success message optional.

---

## Form contract

| Field | Source |
|---|---|
| email | router location.state.email or input fallback |
| otp | 6-digit input — text or number |

If email missing from state, show email field editable.

---

## Submit flow

```
1. await verifyOtp(email, otp)
2. On 200: navigate('/login', { state: { message: 'Verified — please login' } })
3. On 400: show invalid/expired OTP
```

---

## UX notes

- Auto-focus OTP input  
- Login page displays state.message if present  
- Link back to register if wrong email
