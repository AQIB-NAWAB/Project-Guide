# 4.11 — DB module

**Why this step matters:** Centralizing `connectDB()` means the API, Chapter 5 seed script, and future tests share one connection path. Inline `mongoose.connect` in `server.js` would duplicate URI logic and invite inconsistent behaviour.

**Where the project is now:** Config module validates env. Mongoose not installed; no database connection.

**After this step:** `server/src/shared/db.js` exports `connectDB()` (and optionally `disconnectDB()`); Mongoose added to dependencies.

---

## Step 1 — Install Mongoose

From `server/`:

```bash
npm install mongoose
```

Verify `package.json` lists `mongoose` in dependencies.

---

## Step 2 — Create `server/src/shared/db.js`

**Exports required:**

| Function | Contract |
|---|---|
| `connectDB()` | Async. Uses `mongoUri` from `./config.js`. Calls Mongoose connect. Resolves when connection established. On failure: throw or reject with useful error — do not swallow. |
| `disconnectDB()` | Async. Closes Mongoose connection. Optional for gate; recommended for graceful shutdown. |

**Implementation notes — you write the code:**

- Import `mongoose` and your config object  
- Import config from `./config.js` (same `shared/` folder)  
- Log success once — e.g. `"MongoDB connected"` without printing password  
- Do not call `connectDB()` at module top level (import side effect) — **`server.js` calls it** in 4.12  

Reference Mongoose docs from 4.6 — use Promise/async pattern, not deprecated callbacks.

## Optional connect options

You may pass second argument to `connect()`:

```javascript
// Illustration only — options are your choice
{
  serverSelectionTimeoutMS: 5000,
}
```

Helpful on Atlas when IP whitelist wrong — fails in 5s with clearer message than hanging.

Not required for chapter gate.

## What db.js must NOT do

- Define schemas or models — Chapter 5  
- Start HTTP server  
- Load dotenv — entry file handles that before config  
- Log full `MONGODB_URI`

## Step 3 — Manual smoke test (temporary)

Before wiring `server.js`, optional quick test script or temporary call:

With MongoDB running and valid `.env`:

- Call `await connectDB()`  
- Observe success log  
- Open Compass — database name from URI visible (empty collections OK)  

Remove temporary test code or move to proper boot in 4.12.

With MongoDB **stopped** (local):

- `await connectDB()` should reject/fail — not hang forever  

Restore MongoDB before continuing.

## disconnectDB and shutdown (recommended)

In `server.js` (4.12), optional pattern:

```
SIGINT / SIGTERM handler
  → await disconnectDB()
  → process.exit(0)
```

Prevents orphaned connections during dev restarts. Stretch goal if time short — document in learning log if skipped.

## ✅ Do / ❌ Don't

**✅ Do:** Single connect path reused by API and seed script (Chapter 5).

**✅ Do:** Read URI only from config — never duplicate env read.

**❌ Don't:** Register Mongoose models in `db.js` — models live in modules.

**❌ Don't:** Call `connect()` multiple times on every request — once at boot.

## Troubleshooting

| Error | Likely fix |
|---|---|
| `MongooseServerSelectionError` | MongoDB not running; wrong URI; Atlas IP whitelist |
| `Authentication failed` | Wrong password; URL-encode special chars in Atlas password |
| `Cannot find module '../config.js'` | ESM path — include `.js` extension |
| Connect succeeds but wrong DB in Compass | Database name segment in URI path |
