# 4.15 — Verify connection

**Why this step matters:** Happy-path demos lie by accident unless you test failure — missing env, stopped MongoDB, wrong URI. This step produces evidence for the learning log that your app **fails honestly**, not silently.

**Where the project is now:** Happy path works — server boots with MongoDB, curl and browser show `db: connected`.

**After this step:** You have **evidence** the system fails honestly when configuration or MongoDB is wrong — not only when everything is green.

This sub-chapter is your Chapter 4 demo script. Run each test; note results in learning log or screenshot folder.

---

## Test matrix

| # | Scenario | What you do | Expected behaviour |
|---|---|---|---|
| 1 | Happy path | MongoDB up, valid `.env`, both apps running | curl + Home show `db: connected`; server logs connect before listen |
| 2 | Missing config | Remove or empty `MONGODB_URI` in `.env` | Server **refuses to start**; clear error naming missing var |
| 3 | MongoDB down | Server was running; stop MongoDB OR use invalid URI | Health shows disconnected **or** server exits; **not** silent fake connected |
| 4 | Git secrets | `git status` + `git log --all -- "**/.env"` | No `.env` tracked; history clean |
| 5 | Recovery | Fix config / restart MongoDB / restart server | Returns to happy path without code changes |

Document which strategy you chose for test 3 if DB drops at runtime (503 vs 200 with disconnected) — from 4.13.

## Test 1 — Happy path (confirm baseline)

**Server:**

```bash
cd server && npm run dev
```

**curl:**

```bash

```

**Expected:** JSON with `"status":"ok"` and `"db":"connected"`.

**Client:** Home shows API and database connected.

**Compass:** Connect with same URI — database visible (empty collections until Chapter 5).

## Test 2 — Missing configuration

1. Stop server  
2. Edit `server/.env` — comment out or delete `MONGODB_URI` line  
3. Start server  

**Expected:**

- Process exits or throws **before** listening  
- Error message identifies missing `MONGODB_URI` — not generic Mongoose stack only  

4. Restore valid `MONGODB_URI`  
5. Confirm server starts again  

**Learning log:** One sentence on why fail-fast beats running without DB.

---

## Test 3 — MongoDB unavailable

Pick **at least one** approach:

### Option A — Local: stop MongoDB service

While server running (if your boot allows) or before start:

```bash
# platform-specific stop mongod
```

Restart server or wait for reconnect behaviour.

**Expected:** One of:

- Startup fails at `connectDB()` — no listen  
- Health returns `db: disconnected` or HTTP 503  
- Home shows disconnected / error  

**Not expected:** `db: connected` while Compass cannot connect.

### Option B — Invalid URI

Temporarily set nonsense host in `.env`:

```env
MONGODB_URI=mongodb://127.0.0.1:59999/nonexistent
```

Start server.

**Expected:** Connection failure at boot with clear error.

Restore valid URI after test.

### Option C — Atlas

Wrong password or paused cluster — same expectation: failure visible.

## Test 4 — Secrets not in Git

```bash
git status
```

**Expected:** No `.env` files staged.

```bash
git check-ignore -v server/.env client/.env
```

**Expected:** Both ignored.

```bash
git log --all --full-history -- "**/.env"
```

**Expected:** Empty — if not, stop and remediate before gate (rotate Atlas password).

## Test 5 — Recovery

After failure tests:

1. Valid `server/.env`  
2. MongoDB running  
3. `npm run dev` server + client  
4. curl and Home green again  

No code edits required for recovery — proves problem was config/runtime, not broken implementation.
