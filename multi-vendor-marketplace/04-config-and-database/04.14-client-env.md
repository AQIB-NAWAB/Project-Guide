# 4.14 — Client env

**Why this step matters:** Re-verifying `VITE_API_URL` while adding server secrets reinforces the client/server boundary — the browser still never sees `MONGODB_URI`, but it must display the new health field from your API.

**Where the project is now:** Server health returns `db: connected`. Client Home page from Chapter 2 likely shows only `status`.

**After this step:** Client env verified; Home page displays **database status** from live API response; no MongoDB secrets on client.

---

## Step 1 — Verify client environment files

**`client/.env`** (local):

```env
VITE_API_URL=http://localhost:5000
```

**`client/.env.example`** (committed):

```env
VITE_API_URL=http://localhost:5000
```

If server uses non-default `PORT` from `server/.env`, update client URL to match.

Restart Vite after any `.env` change:

```bash
cd client
npm run dev
```

Console check (temporary):

```javascript
console.log(import.meta.env.VITE_API_URL)
```

**Expected:** Full base URL — not `undefined`.

---

## Step 2 — Confirm `api/client.js` unchanged in contract

`getHealth()` (or `apiGet('/health')`) should already:

- GET `{VITE_API_URL}/health`  
- Parse JSON  
- Return object to caller  

**No change required** if helper returns full parsed body — Home will read new `db` field.

If helper strips fields, update to pass through full response object.

**Do not** add MongoDB URI to client env or helper.

---

## Step 3 — Update Home page UI

**Requirements — you implement JSX and styling:**

Display at minimum:

| UI element | Data source |
|---|---|
| API status | `data.status` from health response |
| Database status | `data.db` from health response |

Example structure (not copy-paste solution):

```
FreshMarket — System status
API: ok
Database: connected
```

**States to handle (from Chapter 2 patterns):**

| State | UI behaviour |
|---|---|
| Loading | Show loading indicator while fetch in flight |
| Success | Show `status` and `db` |
| Network error / server down | Show error message — not fake success |
| JSON missing `db` | Show sensible fallback — e.g. "unknown" — and note in learning log |

When server returns `db: disconnected` (4.15 test), Home should show that — not hide behind generic ok.

## Troubleshooting

| Problem | Fix |
|---|---|
| CORS error | Server cors origin must include Vite URL |
| UI shows old status only | Home not reading `db`; cache — hard refresh |
| `undefined` API URL | `VITE_` prefix; restart Vite |
| 404 on health | Wrong base URL path — no double `/health/health` |
