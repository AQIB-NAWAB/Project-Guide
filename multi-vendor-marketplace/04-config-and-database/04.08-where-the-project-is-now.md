# 4.8 — Where the project is now

**Why this step matters:** The build cluster assumes Chapter 2's skeleton and Chapter 3's ERD still hold. This recap prevents wiring MongoDB into a repo that never passed the health demo or skipped the design gate — fixing foundation bugs now is cheaper than debugging `MONGODB_URI` on a broken tree.

**Build cluster starts here.** Before creating `server/.env` or `shared/config.js`, confirm your repo matches what Chapters 2 and 3 left behind — and that gates are cleared.
