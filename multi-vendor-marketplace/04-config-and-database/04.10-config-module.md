# 4.10 ‚Äî Config module

**Why this step matters:** One validated config object stops `process.env` sprawl across dozens of files. Fail-fast on missing `MONGODB_URI` here turns mysterious Mongoose errors into a clear message the first time you forget to copy `.env.example`.

**Where the project is now:** `server/.env` exists locally; no code reads `MONGODB_URI` yet.

**After this step:** `server/src/shared/config.js` validates required environment variables at import/startup and exports a config object for the rest of the server.

---

## File location

```
server/src/shared/config.js
```

**Why `shared/`:** Config is cross-cutting ‚Äî used by `server.js`, `db.js`, later auth and seed scripts ‚Äî not owned by one domain module.

---

## Behaviour contract ‚Äî you implement

### Required exports

Export a **config object** (named export or default ‚Äî pick one, use consistently) with at minimum:

| Property | Source | Notes |
|---|---|---|
| `mongoUri` | `process.env.MONGODB_URI` | **Required** ‚Äî no fallback |
| `port` | `process.env.PORT` | Default `5000` if unset |
| `nodeEnv` | `process.env.NODE_ENV` | Default `development` if unset |

Naming is yours (`mongodbUri` vs `mongoUri`) ‚Äî keep stable for `db.js` imports.

### Validation rules

1. On module load (or explicit `validateConfig()` you call from `server.js` ‚Äî either pattern OK), check `MONGODB_URI`:
   - If **missing, undefined, or empty string** ‚Üí throw `Error` with message naming the variable **or** log and `process.exit(1)`
2. Do **not** log the full URI ‚Äî optional dev log: `"Config loaded for DB host: ..."` parsing hostname only

3. Optional URI shape check:
   - Must start with `mongodb://` or `mongodb+srv://`
   - If not ‚Üí fail with clear message

### What config.js must NOT do

- Connect to MongoDB ‚Äî that's `db.js`  
- Import Express  
- Read client `VITE_*` variables

## Loading `.env` ‚Äî coordinate with entry file

`config.js` reads `process.env`. Something must load `.env` **before** config runs:

**Pattern A ‚Äî dotenv in `server.js` first line:**

```javascript
// server.js ‚Äî first import
import 'dotenv/config'
import { config } from './shared/config.js'
```

Install: `npm install dotenv` if using this pattern.

**Pattern B ‚Äî Node `--env-file` in package.json** (no dotenv package):

```json
"dev": "node --env-file=.env --watch src/server.js"
```

Then `config.js` assumes env is already populated when Node starts.

**Pick one.** Document in README. Sub-chapter 4.12 wires entry ‚Äî config module stays focused on validation + export.

---

## Suggested structure (not the solution)

Many teams use:

```
shared/config.js
  - define requiredKeys array
  - loop and assert present
  - export frozen config object
```

Or:

```
shared/config.js
  - export function loadConfig() returning object
  - server.js calls loadConfig() at startup
```

Both valid ‚Äî gate is **fail-fast on missing MONGODB_URI**.

```javascript
// Pseudocode ‚Äî YOU implement db.js

```

**`server.js` (4.12)** will import config for port:

```javascript

// import { config } from './shared/config.js'
// app.listen(config.port)
```

No other file should read `process.env.MONGODB_URI` directly ‚Äî **single source of truth**.

## Test validation before Mongoose

Temporarily rename `MONGODB_URI` in `.env` to empty or comment it out.

Run server (or import config in isolation):

**Expected:** Process exits or throws with message mentioning `MONGODB_URI` ‚Äî not Mongoose "Invalid scheme".

Restore `.env` after test.

## ‚úÖ Do / ‚ùå Don't

**‚úÖ Do:** Export plain values ‚Äî avoid mutating config object after export.

**‚úÖ Do:** Add new required env vars here when Chapter 6 adds JWT ‚Äî same validation pattern.

**‚ùå Don't:** Hardcode fallback MongoDB URI "for convenience" ‚Äî hides misconfiguration.

**‚ùå Don't:** Export `process.env` wholesale ‚Äî explicit fields only.

## Troubleshooting

| Problem | Likely cause |
|---|---|
| `MONGODB_URI` undefined but present in `.env` | dotenv not loaded; wrong `.env` path; dev script missing `--env-file` |
| Config throws in tests later | Chapter 5 seed must load env same way ‚Äî reuse entry pattern |
| Port still 5000 when `.env` says 5001 | config not used in `listen` yet ‚Äî fixed in 4.12 |
