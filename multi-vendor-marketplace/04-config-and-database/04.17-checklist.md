### Questions

1. **Environment setup:** Walk through how configuration flows from `server/.env` into running code — include env loading mechanism (dotenv or `--env-file`), `config.js`, and `db.js`. What happens if `MONGODB_URI` is missing?

2. **Secrets boundary:** Why must `MONGODB_URI` never appear in `client/.env` or React components? What is safe in `VITE_API_URL` and why?

3. **Boot order:** Why does this chapter require MongoDB connection before `app.listen`? What bug appears if you reverse the order?

4. **Health contract:** Paste your actual `/health` JSON from curl (redact nothing — it should contain no secrets). Explain what `db: connected` checks and what it does **not** prove about your data.

5. **Failure test:** Describe one test you ran when MongoDB or config was wrong (4.15). What did the server do? What did the client show?

6. **Local vs Atlas:** Which did you use for development? One advantage and one disadvantage for your choice on this project.

7. **Chapter 5 preview:** How will `scripts/seed.js` reuse `connectDB` and the same `MONGODB_URI`? What collections from your ERD draft will get documents first?

8. **Git verification:** What commands did you run to confirm `.env` was never committed? What would you do if you accidentally committed a URI?
