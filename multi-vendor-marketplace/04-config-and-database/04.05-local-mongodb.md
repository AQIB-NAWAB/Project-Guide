# 4.5 — Local MongoDB vs Atlas

Before sub-chapter 4.9 asks you to paste `MONGODB_URI` into `.env`, you need a **running MongoDB instance** and a connection string that works from your machine.

This course supports **either** local MongoDB **or** MongoDB Atlas (cloud free tier). Pick one for development; stick with it through Week 1 unless you hit a blocker.

## Why this matters for absolute beginners

Nothing in the build cluster works until **mongosh or Compass** connects with the same URI you will paste into `.env`. Debugging Node before the database runs wastes hours blaming Mongoose for network problems. Verify the wire with official tools first; wire Express second.

## Common beginner questions

**Q: Atlas free tier — do I need a credit card?**  
A: MongoDB's policy changes — follow Atlas signup flow; M0 free clusters are common for learning.

**Q: Special characters in Atlas password break my URI — why?**  
A: URLs parse `@` and `#` specially — URL-encode the password segment.

**Q: How do I test "DB down" on Atlas?**  
A: Wrong password at boot, paused cluster, or temporary bad URI — goal is honest failure, not only stopping local `mongod`.

**Q: What database name should I use?**  
A: A project slug like `freshmarket-dev` — not generic `test`.

## Option A — Local MongoDB

**Best if:** You want offline dev, fast iteration, no cloud signup, Compass pointing at localhost.

**Install paths (pick your OS docs):**

| Platform | Typical approach |
|---|---|
| Linux | Official MongoDB packages or Docker `mongo:7` |
| macOS | Homebrew `brew install mongodb-community` or Docker |
| Windows | MongoDB installer or Docker |

**Verify mongod is running:**

```bash
mongosh --eval "db.runCommand({ ping: 1 })"
```

**Expected:** `{ ok: 1 }` or successful connection message — not connection refused.

**Connection string for `.env`:**

```env
MONGODB_URI=mongodb://127.0.0.1:27017/your-app-db-name
```

Replace `your-app-db-name` with your product slug, e.g. `greenbasket-dev`. MongoDB creates the database on first write — no manual "create database" required.

**Tools:** [MongoDB Compass](https://www.mongodb.com/products/compass) — connect to same URI; you'll use it heavily in Chapter 5 seed inspection.

---

## Option B — MongoDB Atlas (free tier)

**Best if:** Local install is painful on your machine, you want identical setup to many deploy guides, or you develop on multiple machines.

**Setup outline:**

1. Create account at [mongodb.com/cloud/atlas](https://www.mongodb.com/cloud/atlas)  
2. Create a **free M0 cluster**  
3. Create **database user** (username + password) — not your Atlas login password  
4. **Network access:** allow your IP (or `0.0.0.0/0` for dev only — tighten before production)  
5. **Connect** → Drivers → copy connection string  
6. Replace `<password>` and set database name in path  

**Connection string shape:**

```env
MONGODB_URI=mongodb+srv://dbuser:YOUR_PASSWORD@cluster0.xxxxx.mongodb.net/your-app-db?retryWrites=true&w=majority
```

**Special characters in password:** URL-encode `@`, `#`, `%`, etc. — raw password in URI breaks parsing.

## Decision table — which should you pick?

| Factor | Local | Atlas |
|---|---|---|
| Setup time | Varies by OS | ~15 min first time |
| Offline | ✓ | ✗ needs internet |
| Matches Chapter 22 deploy | Optional | Often yes |
| Chapter 4.15 "stop MongoDB" test | Stop local `mongod` service | Pause cluster or block IP / wrong URI |
| Cost | Free | Free tier limits |
| Data visibility | Compass → localhost | Compass → Atlas connection |

**Course gate does not require Atlas** — only that Mongoose connects and health reports status.

## Naming your database

Use one logical database name for the whole project, e.g.:

- `freshmarket-dev`  
- `greenbasket-dev`  

Avoid `test` — ambiguous when you run Jest later. Production will use a different name or cluster in Chapter 22 — same code, different `MONGODB_URI` on host.

---

## Security baseline (both options)

- **Never commit** URI with password  
- **Atlas:** dedicated app user, not org owner  
- **Local:** bind to localhost in dev; don't expose `27017` to public internet without auth  
- **Rotate** password if leaked

## Troubleshooting connection before Chapter 4 build

| Symptom | Local fix | Atlas fix |
|---|---|---|
| `ECONNREFUSED 127.0.0.1:27017` | Start `mongod` | N/A — use srv URI |
| Authentication failed | N/A unless auth enabled | Wrong user/password; check URL encoding |
| IP not whitelisted | N/A | Add current IP in Network Access |
| `querySrv ENOTFOUND` | Wrong host in URI | Copy fresh string from Atlas UI |
| Timeout | Firewall | IP whitelist; cluster paused |

Test with **mongosh** or Compass **before** wiring Node — isolates network vs code bugs.

## Chapter 4.15 note for Atlas users

"Stop MongoDB" test is harder on Atlas than stopping local `mongod`. Acceptable alternatives:

- Temporarily set wrong password in `.env` and confirm server **fails to start** with clear error  
- Disconnect network and observe connection failure  
- Pause cluster in Atlas UI if available  

Learning goal: **the app must not pretend DB is fine when it isn't** — exact mechanical test is flexible.
