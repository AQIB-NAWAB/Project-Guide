# 6.11 — Wire auth routes

**Why this step matters:** Unmounted routers are dead code — Express must expose `/api/auth/*` before any handler runs. One mount line in `app.js` is how every future module (stores, products) attaches too.

**Where the project is now:** Auth router exists but not mounted on Express app.

**After this step:** `app.js` mounts auth routes at `/api/auth`. Unregistered paths under prefix return 404; health route unchanged.

---

## Mount specification

In `server/src/app.js`:

```
import authRoutes from './modules/auth/auth.routes.js'

app.use('/api/auth', authRoutes)
```

**Order:**

1. `express.json()` middleware  
2. CORS (if configured from Ch 2)  
3. `/health` route  
4. `/api/auth` router  
5. 404 handler optional  

---

## Path resolution

Full paths become:

| Router path | Full URL |
|---|---|
| POST `/register` | POST `/api/auth/register` |
| GET `/me` | GET `/api/auth/me` |

curl must use full path including `/api`.

---

## Smoke test — routes reachable

Before handlers complete, stub may return 501:

```bash
curl -s -o /dev/null -w "%{http_code}" -X POST http://localhost:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{}'
```

Expect **400** (validation) or **501** (stub) — not **404**.

404 means mount path wrong.
