# 6.10 — Scaffold auth module

**Why this step matters:** Auth HTTP code lives in `modules/auth/` while User models stay in `modules/users/` — separation keeps login routes from bloating user schema files. Empty route and service files give Chapter 6 build steps named targets.

**Where the project is now:** Empty or missing `server/src/modules/auth/` files.

**After this step:** Four auth module files exist with exported stubs — route paths declared, service functions named, middleware placeholder, schema validators sketched. No full handler bodies required yet — contracts defined.

---

## Target files

```
server/src/modules/auth/
├── auth.routes.js
├── auth.service.js
├── auth.middleware.js
└── auth.schema.js
```

---

## auth.routes.js — route table (wire handlers you implement)

| Method | Path | Handler | Middleware |
|---|---|---|---|
| POST | `/register` | registerHandler | validateRegister |
| POST | `/verify-otp` | verifyOtpHandler | validateVerifyOtp |
| POST | `/login` | loginHandler | validateLogin |
| POST | `/refresh` | refreshHandler | — |
| POST | `/logout` | logoutHandler | requireAuth (optional) |
| GET | `/me` | meHandler | requireAuth |

Export Express Router — mounted at `/api/auth` in 6.11.

---

## auth.service.js — function contracts

Export async functions — **implement in later sub-chapters**:

| Function | Returns | Throws / errors |
|---|---|---|
| `registerUser({ email, password, role, displayName })` | `{ userId, otp? }` | 409 duplicate email |
| `verifyOtp({ email, otp })` | `{ verified: true }` | 400 invalid/expired OTP |
| `loginUser({ email, password })` | `{ accessToken, refreshToken, user }` | 401 bad creds; 403 unverified |
| `refreshAccessToken(refreshToken)` | `{ accessToken, refreshToken? }` | 401 invalid refresh |
| `logoutUser(userId)` | void | clears refresh hash |
| `getMe(userId)` | `{ user, profile }` | 404 |

Service imports User, Profile models — not routes.

---

## auth.middleware.js

| Export | Behaviour |
|---|---|
| `requireAuth` | Bearer JWT verify → req.user |
| `validateRegister` etc. | delegate to auth.schema |

---

## auth.schema.js

Validate req.body — return 400 with `{ errors: [...] }` or `{ message }`.

**register:** email format, password min 8, role in vendor/customer

**login:** email, password required

**verify-otp:** email, otp (6 digits typical)

You choose validator library or manual checks.
