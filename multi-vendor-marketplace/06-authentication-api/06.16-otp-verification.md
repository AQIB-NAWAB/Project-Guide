# 6.16 — OTP verification

**Why this step matters:** `isVerified` separates real accounts from spam registrations — vendors should not list products until email is confirmed. Chapter 7's Verify OTP page is useless without this endpoint returning clear success and failure.

**Where the project is now:** Register generates OTP; login blocks unverified users. Verify endpoint missing or stub.

**After this step:** `POST /api/auth/verify-otp` validates OTP against stored value/hash, sets `isVerified: true`, clears OTP fields. User can login afterward.

---

## Request contract

**POST** `/api/auth/verify-otp`

**Body:**

| Field | Required |
|---|---|
| email | yes |
| otp | yes — typically 6 digits |

---

## Success response — 200

```json
{
  "message": "Email verified successfully",
  "verified": true
}
```

Optional: return tokens on verify (auto-login) — document if implemented; not required for gate.

---

## Error responses

| Status | Condition |
|---|---|
| 400 | missing fields / invalid format |
| 400 | wrong OTP |
| 400 | expired OTP |
| 404 | email not found |

Use 400 for wrong OTP — not 401 (not authenticated yet).

---

## Service behaviour

1. Find User by email  
2. If !user → 404  
3. If user.isVerified → 200 already verified (idempotent OK)  
4. Check OTP matches stored value (or bcrypt.compare if hashed)  
5. Check otpExpiresAt > now  
6. Set isVerified = true; clear otp fields  
7. Save user  

---

## OTP generation recap (register)

- 6-digit numeric  
- Expiry 10 minutes typical  
- Dev: log to console on register  

---

## curl flow

```bash
# 1. Register
curl -s -X POST http://localhost:5000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"verifyme@test.local","password":"Password123!","role":"customer"}'

# 2. Note OTP from response or server log

# 3. Verify
curl -s -X POST http://localhost:5000/api/auth/verify-otp \
  -H "Content-Type: application/json" \
  -d '{"email":"verifyme@test.local","otp":"123456"}'

# 4. Login — now 200
curl -s -X POST http://localhost:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"verifyme@test.local","password":"Password123!"}'
```

## ✅ Do / ❌ Don't

**✅ Do:** Expire OTP — don't accept week-old codes.

**✅ Do:** Clear OTP after success — prevent reuse.

**❌ Don't:** Skip verify for production — even if dev shortcut tempting.

**❌ Don't:** Email OTP via real SMTP required — console dev OK.
