# 6.15 — Refresh token

**Why this step matters:** Refresh tokens let clients renew access without re-entering passwords — but only if stored hashed and revocable on logout. This step completes the session story Chapter 7's silent refresh depends on.

**Where the project is now:** Access JWT works; refresh may be duplicate long JWT or missing persistence.

**After this step:** Refresh token issued on login, **hashed and stored** on User document, validated on `POST /api/auth/refresh`, rotatable on use. Logout clears hash.

---

## Refresh token specification

| Property | Value |
|---|---|
| Format | Opaque random string **or** separate JWT — document choice |
| Lifetime | `config.jwtRefreshExpiry` — default `7d` |
| Storage (client) | JSON body Ch 6; httpOnly cookie Ch 7 |
| Storage (server) | `User.refreshTokenHash` — bcrypt or sha256 hash |

**Never store raw refresh token in DB.**

---

## Issue on login

After access token signed:

1. Generate refresh token (crypto.randomBytes or jwt.sign with longer exp)  
2. `hash = await bcrypt.hash(refreshToken, 10)` or sha256  
3. Save hash to User.refreshTokenHash  
4. Return refreshToken in login response body (Ch 6 curl)

---

## POST /api/auth/refresh

**Body (Ch 6 mode):**

```json
{ "refreshToken": "..." }
```

**Success 200:**

```json
{
  "accessToken": "new-access...",
  "refreshToken": "new-refresh..."
}
```

**Rotation (recommended):** Issue new refresh, update hash, invalidate old refresh.

---

## Error responses

| Status | Condition |
|---|---|
| 400 | missing refreshToken |
| 401 | invalid or revoked refresh |
| 401 | user not found |

Compare presented refresh with stored hash — same pattern as password.

---

## Logout (optional for gate)

**POST** `/api/auth/logout` with requireAuth:

- Set User.refreshTokenHash = null  
- Return 204  

Client discards access token in Ch 7.

---

## curl verify

```bash
# After login — save REFRESH from response
curl -s -X POST http://localhost:5000/api/auth/refresh \
  -H "Content-Type: application/json" \
  -d "{\"refreshToken\": \"$REFRESH\"}"
```

Expect new accessToken. Use new access on GET /me.

## Revocation test

Login → logout (or manually clear refreshTokenHash in Compass) → refresh → **401**.

## ✅ Do / ❌ Don't

**✅ Do:** Hash refresh before DB — like passwords.

**✅ Do:** Rotate on refresh — stolen old refresh dies on next legitimate refresh.

**❌ Don't:** Store refresh in localStorage long-term — Ch 7 uses httpOnly cookie.

**❌ Don't:** Accept refresh as Bearer on protected routes — access only.
