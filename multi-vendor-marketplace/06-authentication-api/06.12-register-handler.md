# 6.12 — Register handler

**Why this step matters:** Register is the front door — it creates User + Profile, hashes nothing yet incorrectly, and starts the OTP verification story. Chapter 7's Register page calls exactly what you implement here.

**Where the project is now:** Auth routes mounted; register endpoint returns stub or 501.

**After this step:** `POST /api/auth/register` creates User + Profile, hashes password with bcrypt, generates OTP, returns appropriate JSON. Duplicate email returns 409.

---

## Request contract

**POST** `/api/auth/register`

**Headers:** `Content-Type: application/json`

**Body:**

| Field | Type | Required | Rules |
|---|---|---|---|
| email | string | yes | valid email, lowercase |
| password | string | yes | min 8 chars |
| role | string | yes | `vendor` \| `customer` |
| displayName | string | no | defaults from email local part |

---

## Success response — 201

```json
{
  "message": "Registration successful. Verify your email with OTP.",
  "userId": "664a...",
  "otp": "123456"
}
```

**otp field:** Include only when `NODE_ENV=development` OR always log to server console in dev — document choice. Production response must not leak OTP in JSON.

---

## Error responses

| Status | Condition | Body example |
|---|---|---|
| 400 | validation fail | `{ "message": "Invalid email" }` |
| 409 | email exists | `{ "message": "Email already registered" }` |
| 500 | server error | `{ "message": "Internal server error" }` |

---

## Service behaviour (implement in auth.service)

1. Normalize email to lowercase trim  
2. `User.findOne({ email })` — if exists throw 409  
3. `passwordHash = await bcrypt.hash(password, 10)`  
4. Create User: email, passwordHash, role, isVerified: false  
5. Create Profile: userId, displayName  
6. Generate 6-digit OTP + expiry (e.g. 10 minutes) — store on User  
7. Log OTP in dev  
8. Return userId (no passwordHash)

**Transaction optional:** Sequential saves acceptable for MVP; unique index catches duplicate email race.

## ✅ Do / ❌ Don't

**✅ Do:** Return 201 on success.

**✅ Do:** Strip password from any log of req.body.

**❌ Don't:** Auto-set isVerified true — breaks OTP flow.

**❌ Don't:** Return passwordHash in response.
