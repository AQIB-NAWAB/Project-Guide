# 6.18 — Test with curl

**Why this step matters:** Browser bugs hide API bugs and vice versa. A full curl script — register, verify, login, me, refresh — is your Chapter 6 gate evidence and the contract Chapter 7 must not break.

**Where the project is now:** All auth endpoints implemented individually.

**After this step:** You have run the **complete auth flow** via curl and documented results — proof for gate and Chapter 7 client integration.

---

## Full flow script

Run in order; replace values from your responses.

```bash
BASE=http://localhost:5000

# --- 1. Register ---
curl -s -X POST $BASE/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "curltest@test.local",
    "password": "Password123!",
    "role": "customer",
    "displayName": "Curl Tester"
  }'
# Save OTP from response or server log

# --- 2. Verify OTP ---
curl -s -X POST $BASE/api/auth/verify-otp \
  -H "Content-Type: application/json" \
  -d '{"email":"curltest@test.local","otp":"REPLACE_OTP"}'

# --- 3. Login ---
LOGIN=$(curl -s -X POST $BASE/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"curltest@test.local","password":"Password123!"}')
echo $LOGIN
# Extract accessToken and refreshToken — use jq if available:
# ACCESS=$(echo $LOGIN | jq -r .accessToken)
# REFRESH=$(echo $LOGIN | jq -r .refreshToken)

# --- 4. Protected route ---
curl -s $BASE/api/auth/me \
  -H "Authorization: Bearer REPLACE_ACCESS"

# --- 5. Refresh ---
curl -s -X POST $BASE/api/auth/refresh \
  -H "Content-Type: application/json" \
  -d '{"refreshToken":"REPLACE_REFRESH"}'

# --- 6. Seed vendor login ---
curl -s -X POST $BASE/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"vendor-a@test.local","password":"Password123!"}'
```

## Negative tests checklist

| # | Test | Expected |
|---|---|---|
| 1 | Login unverified new user (skip step 2) | 403 |
| 2 | Login wrong password | 401 |
| 3 | /me without token | 401 |
| 4 | /me with garbage token | 401 |
| 5 | Register duplicate email | 409 |
| 6 | Refresh with invalid token | 401 |
| 7 | Register role admin | 400 |
