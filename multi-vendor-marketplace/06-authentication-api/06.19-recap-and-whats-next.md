# 6.19 — Recap and what's next

Chapter 6 turned an anonymous API into an **authenticated** one. Every protected route from here forward builds on `requireAuth` and the token contracts you tested with curl.

---

## What you built

**Register:** bcrypt password hash, User + Profile creation, OTP generation, duplicate email rejection.

**Verify OTP:** isVerified gate before login.

**Login:** Credential check, 403 for unverified, access + refresh tokens issued.

**Access JWT:** Short-lived, sub + role, Bearer header.

**Refresh:** Hashed in DB, rotation endpoint, revocation on logout.

**requireAuth:** 401 on bad/missing/expired token; req.user for downstream handlers.

**GET /me:** Protected smoke test returning user + profile without secrets.
