# 12.9 — Publish toggle

## Why this step matters

Draft vs published is the line between back office and shop window. The toggle PATCHes `isPublished` — the same field Chapter 14 will filter on for customers. Vendors control when items go live without deleting drafts.

---

**After this step:** Vendor can **publish or unpublish** products from edit page (or table) via `PATCH` with `isPublished` — status visible in product list.

---

## UI placement

| Option | Spec |
|---|---|
| **Edit page toggle** | Switch labeled "Published" — recommended for gate |
| Table quick action | Stretch — PATCH on toggle without full edit |

Primary: **edit page** checkbox or switch bound to `isPublished`.

---

## Toggle flow

```
onToggle(newValue):
  1. setPublishing true
  2. updateProduct(id, { isPublished: newValue })
  3. 200 → update local state; show Published/Draft label
  4. error → revert toggle; show banner
```

**Optimistic toggle** acceptable for publish only — revert on failure.

Alternatively: "Save" button saves all fields including publish — single PATCH. Toggle may batch with other edits — both OK if list reflects status after save.

---

## List page status column

| isPublished | Badge |
|---|---|
| false | Draft — muted styling |
| true | Published — accent styling |

Refetch list when returning from edit — badge updates.

---

## Business meaning

| State | Vendor list | Public catalogue (Ch 14) |
|---|---|---|
| Draft | Visible | Hidden |
| Published | Visible | Visible if in stock |

Helper text on edit: "Published products appear in the marketplace when catalogue launches."

---

## PATCH contract

```json
{ "isPublished": true }
```

Can combine with other fields in one PATCH on save — server supports partial update.

---

## Edge cases

| Case | UX |
|---|---|
| Publish with zero stock | Allow — Ch 14 may hide OOS |
| Unpublish | PATCH false — removes from future public list |
