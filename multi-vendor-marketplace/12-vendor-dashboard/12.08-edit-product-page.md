# 12.8 — Edit product page

## Why this step matters

Vendors fix prices and stock daily. Pre-filled edit forms test your PATCH contract and prove the vendor can round-trip data — load, change, save, see update in the list. Chapter 13's image upload attaches to this page.

---

**After this step:** `/vendor/products/:id/edit` **pre-fills form** from product data and saves via `PATCH /api/products/:id`.

---

## Loading product data

| Strategy | Pros |
|---|---|
| A — from list via route state | No extra request |
| B — refetch list and find id | Simple; works on direct URL |
| C — GET single product (if added) | Best deep links |

**MVP:** Strategy B — on mount call `listVendorProducts()`, find `params.id`, or 404 if missing.

---

## Page: EditProduct.jsx

Same fields as create — all editable except id.

### Submit flow

```
onSubmit:
  1. updateProduct(id, dirtyFields) → PATCH
  2. 200 → success toast/banner + stay or navigate list
  3. 400 → field errors
  4. 403/404 → banner per 12.4
```

Recommend navigate back to list on success — table shows updates.
