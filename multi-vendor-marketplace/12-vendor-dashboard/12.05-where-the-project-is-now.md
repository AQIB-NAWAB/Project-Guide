### Product API contracts (reference)

| Method | Path | Role | Success | Key failure |
|---|---|---|---|---|
| `POST` | `/api/products` | vendor | 201 | 400, 404 no store |
| `GET` | `/api/vendor/products` | vendor | 200 array | 403 customer |
| `PATCH` | `/api/products/:id` | vendor | 200 | 403 cross-vendor |
| `DELETE` | `/api/products/:id` | vendor | 200/204 | 403 cross-vendor |

**Body rule:** Never send `storeId` — server resolves from vendor's store.

### Pre-build curl gate

```bash
curl -s http://localhost:5000/api/vendor/products -H "Authorization: Bearer <vendor-token>"
curl -s -X POST http://localhost:5000/api/products \
  -H "Authorization: Bearer <vendor-token>" \
  -H "Content-Type: application/json" \
  -d '{"name":"Organic Kale","price":2.99,"unit":"bunch","stockQty":20}'
```

- [ ] GET /vendor/products returns array
- [ ] POST creates draft (`isPublished: false`)
- [ ] Vendor B PATCH Vendor A product → 403
