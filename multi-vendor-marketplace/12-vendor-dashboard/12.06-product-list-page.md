# 12.6 — Product list page

## Why this step matters

The product table is the vendor's home screen for inventory — first thing they see after the store gate. It must handle loading, empty state, and API errors gracefully. Every create/edit flow returns here; get fetch and display right before forms.

---

**Where the project is now:** `/vendor/products` is stub; `api/products.js` missing; Ch 11 list endpoint works via curl only.

**After this step:** `/vendor/products` shows **vendor inventory table** from `GET /api/vendor/products` with loading, empty, and error states.

---

## Page: ProductList.jsx

### UI contract

| Element | Spec |
|---|---|
| Route | `/vendor/products` |
| Title | "Your products" |
| Primary action | "Add product" → `/vendor/products/new` |
| Data source | `GET /api/vendor/products` only |

### Table columns

| Column | Field | Notes |
|---|---|---|
| Name | `name` | Link to edit optional |
| Price | `price` + `unit` | e.g. "£2.99 / bunch" |
| Stock | `stockQty` | Integer |
| Perishable | `isPerishable` | Badge or icon |
| Status | `isPublished` | Draft / Published |
| Actions | — | Edit → `/vendor/products/:id/edit` |

### UX states

| State | Display |
|---|---|
| loading | Spinner — not empty message |
| empty (200 []) | "No products yet" + CTA |
| error | "Could not load" + retry |
| populated | Full table |

---

## Mount flow (spec)

```
useEffect:
  1. setLoading true
  2. listVendorProducts()
  3. 200 → setProducts
  4. error → setError banner
  5. setLoading false
```

Refetch when navigating back from create/edit.

---

## API contract — `listVendorProducts()`

```
GET /api/vendor/products
Authorization: Bearer <vendor-token>
```

| Status | Behaviour |
|---|---|
| 200 | Parse `products` array or raw array |
| 401 | Interceptor or login |
| 404 | Link open-shop — should not happen past gate |

Items need: `_id`, `name`, `price`, `unit`, `stockQty`, `isPerishable`, `isPublished`.

## Troubleshooting

| Symptom | Likely cause |
|---|---|
| Empty but curl has rows | Wrong response parsing |
| All vendors' products | Wrong endpoint — must be `/vendor/products` |
| Infinite loading | Missing `finally` |
| Empty during load | No loading state |
