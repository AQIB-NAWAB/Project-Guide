# 12.7 — Create product page

## Why this step matters

Adding stock is the core vendor action. This form calls `POST /products` with grocery fields — unit, weight, stock, perishable. Validation errors from the API must map to fields (see 12.4). New products start as drafts until publish.

---

**Where the project is now:** Product list loads from API; `/vendor/products/new` stub; `createProduct` not in `api/products.js`.

**After this step:** `/vendor/products/new` submits form → `POST /api/products` → navigates to list with new **draft** row.

---

## Page: CreateProduct.jsx

### UI contract

| Element | Spec |
|---|---|
| Route | `/vendor/products/new` |
| Title | "Add product" |
| Helper | "New products start as drafts until you publish." |
| CTA | "Save product" |
| Cancel | Link → `/vendor/products` (no save) |

### Form fields

| Field | Control | Client validation |
|---|---|---|
| `name` | text | required |
| `description` | textarea | optional |
| `price` | number | required; > 0 |
| `unit` | select | required — match model enum |
| `weightOrQuantity` | text/number | per ERD |
| `stockQty` | number | required; >= 0 |
| `isPerishable` | checkbox | boolean |

**Do not expose** `isPublished` — server defaults `false` (draft). Publish in 12.9.

### Unit options

Match schema enum: `each`, `kg`, `g`, `bunch`, `pack`, `litre` — hardcode matching server.

---

## API contract — `createProduct(data)`

```
POST /api/products
Authorization: Bearer <vendor-token>
Body: { name, description?, price, unit, stockQty, isPerishable?, weightOrQuantity? }
```

| Status | Behaviour |
|---|---|
| 201 | Navigate `/vendor/products` |
| 400 | Map `errors` per 12.4 |
| 404 | No store — link open-shop |

**Must not send:** `storeId`, `vendorId`, `isPublished`.

---

## Submit flow (spec)

```
onSubmit:
  1. Client validate
  2. setSubmitting true
  3. createProduct(payload)
  4. 201 → navigate('/vendor/products')
  5. 400 → field errors
  6. finally setSubmitting false
```

Wait for 201 — no optimistic row (12.4).

---

## List integration

After navigate: list refetches → new row shows **Draft** with correct price/unit.

---

## Troubleshooting

| Symptom | Likely cause |
|---|---|
| 201 but row missing | List not refetching on return |
| 400 on valid unit | Enum mismatch client vs server |
| Double product | No submit guard |
| Product wrong storeId | Server bug — fix Ch 11 |

---

## ✅ Do / ❌ Don't

**✅ Do:** Navigate on success; map 400 errors; match `unit` enum; disable submit in flight.

**❌ Don't:** Send `storeId`; auto-publish; use `alert()` for errors.
