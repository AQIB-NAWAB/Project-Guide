# 12.4 — Error mapping

Forms fail — vendors enter negative stock, lose Wi-Fi mid-save, or session expires. Chapter 12 specs how **HTTP status becomes UI feedback** without exposing stack traces.

---

## Status → UI matrix

| Status | Meaning | UI response |
|---|---|---|
| **400** | Validation failed | Inline field errors from API `errors` object |
| **401** | Token missing/expired | Redirect login; preserve return URL optional |
| **403** | Wrong role or ownership | Banner: "You don't have permission" — rare on own products |
| **404** | Product or store not found | "Product not found" + link back to list |
| **409** | Business conflict | Banner — uncommon on products |
| **5xx** | Server error | Generic retry message |
| **Network** | fetch failed | "Could not reach server" + retry |

---

## 400 field errors

API shape (align with Ch 6/11):

```
{ "errors": { "price": "Must be positive", "stockQty": "Required" } }
```

Client maps keys to input `name` attributes — show under field.

**✅ Do:** Clear field error when user edits that field.

**❌ Don't:** Show raw JSON error blob to vendor.

---

## 403 on edit page

Should not happen for own product — if seen:

- Stale id in URL — another vendor's product bookmarked  
- Bug in ownership — fix server  

Show non-technical message; log details in dev console only.

---

## Loading token expiry

If PATCH returns 401:

- Auth interceptor refreshes once (Ch 7)  
- Retry PATCH once  
- Else logout  

Reuse existing interceptor — product forms should not special-case unless interceptor missing.

---

## Optimistic UI

**Not recommended** for create/publish — wait for server 201/200 before navigate — avoids ghost rows in table.

---

## Delete confirmation (optional)

If delete button in table:

- Confirm modal: "Remove Organic Kale?"  
- DELETE → refetch list  
- 403 → should not happen for own row  

---

## Empty error state

Distinguish:

| Case | Message |
|---|---|
| API returned [] | "No products yet" |
| API failed | "Could not load products" + retry |

---

## ✅ Do / ❌ Don't

**✅ Do:** Disable submit during request — prevent double POST.

**✅ Do:** Keep error copy human — grocery vendors not engineers.

**❌ Don't:** Alert() for errors — use inline/banner.

**❌ Don't:** Swallow 400 — silent failure loses trust.
