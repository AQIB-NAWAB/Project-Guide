# 12.2 — Table plus forms

Enterprise vendors use ERP dashboards. **FreshMarket MVP** uses a **table + separate create/edit pages** — enough to prove full-stack competency without component library complexity.

---

## Pattern

```
/products          → table (list)
/products/new      → create form
/products/:id/edit → edit form + publish toggle
```

Alternative: modal forms — acceptable stretch; course specs **dedicated routes** for clarity and deep-linking.

---

## Table responsibilities

| Column (examples) | Source |
|---|---|
| Name | product.name |
| Price | formatted price + unit |
| Stock | stockQty |
| Perishable | badge yes/no |
| Status | Draft / Published from isPublished |
| Actions | Edit link; optional Delete |

Empty state: "No products yet — Add your first item."

---

## Form responsibilities

Shared grocery fields from Ch 11:

- name, description (optional)  
- price  
- unit (select)  
- weightOrQuantity  
- stockQty  
- isPerishable (checkbox)  

Create: POST — default unpublished.  
Edit: load single product from list cache or GET by id if you add route — MVP: pass state from table or refetch list and find id.

**Stretch:** `GET /products/:id` vendor-scoped — not required if list has full objects.

---

## Publish toggle

| UI | API |
|---|---|
| Switch or checkbox on edit page | PATCH `{ isPublished: true/false }` |
| Badge on table updates after save | Refetch list on navigate back |

Separate "Publish" page not needed — toggle on edit is enough.

---

## Editorial style

- Clean typography — no dense admin theme  
- Generous whitespace — grocery products deserve readable names  
- Status badges: subtle Draft (muted) vs Published (accent)  

---

## Navigation

Vendor layout nav **Products** → `/vendor/products`.

Breadcrumbs optional: Products → Edit Kale.

---

## Loading and empty states

| State | UX |
|---|---|
| Loading list | Table skeleton or spinner |
| Empty inventory | CTA to create |
| Saving form | Disabled submit |
| Load error | Retry banner |

---

## ✅ Do / ❌ Don't

**✅ Do:** Keep table read-only — edits on edit page (simpler than inline edit).

**✅ Do:** Show unit next to price in table — "£2.50 / bunch".

**❌ Don't:** Require DataGrid/MUI for gate — native table OK.

**❌ Don't:** Build customer-facing product card here — Ch 15.
