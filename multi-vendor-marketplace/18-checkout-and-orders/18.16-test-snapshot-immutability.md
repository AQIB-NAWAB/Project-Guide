# 18.16 — Test snapshot immutability

**Why this step matters:** This is the **proof** snapshots work — change product price after checkout, reload orders, old price must remain. Do not skip; it catches populate-on-read mistakes that pass every other test.

**Where the project is now:** Checkout, lists, stock, and cart clear implemented.

**After this step:** You have **proof** that changing a product price does not rewrite historical order lines.

---

## Test script — run in order

1. Note product **P** current price `P1` from shop or Compass.
2. As customer, add **P** to cart; checkout (single-store cart is fine).
3. `GET /orders/my` — record `priceAtPurchase` on line → must equal `P1`.
4. As vendor owner of **P**, `PATCH /products/:id` price to `P2` (different value).
5. `GET /orders/my` again — **same order**, `priceAtPurchase` still `P1`.
6. `GET /vendor/orders` — vendor view also shows `P1`.
7. Shop browse shows new price `P2` for new customers.

---

## Compass verification

Open Order document in Compass. Inspect embedded `orderItems` — numeric fields unchanged after product PATCH.

---

## What failure looks like

| Symptom | Bug |
|---|---|
| Order line shows P2 after PATCH | Populate live product on read, or no snapshot at write |
| `total` changed on order header | Recomputed from live data |
| lineTotal ≠ priceAtPurchase × quantity | Snapshot math wrong at checkout |
