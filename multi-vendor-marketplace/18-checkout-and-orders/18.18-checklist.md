## Definition of Done

### Checkout

- [ ] `POST /checkout` requires customer auth
- [ ] Empty cart returns 400
- [ ] Two-store cart creates **two** Order documents
- [ ] Each order has single `storeId` — no mixed-store orders
- [ ] `orderItems` embedded with `nameAtPurchase`, `priceAtPurchase`, `unitAtPurchase`, `quantity`, `lineTotal`
- [ ] Order `total` equals sum of line totals
- [ ] Stock decremented atomically with `$gte` guard
- [ ] Insufficient stock fails entire checkout — no partial orders
- [ ] Cart empty after successful checkout
- [ ] Cart unchanged after failed checkout

### List routes

- [ ] `GET /orders/my` returns customer's orders only
- [ ] `GET /vendor/orders` returns vendor store's orders only
- [ ] Cross-vendor `GET /vendor/orders` does not leak other stores' orders
- [ ] List responses use snapshot prices — not populated live product price

### Immutability (18.16)

- [ ] Product price PATCH after order does not change `priceAtPurchase` on existing order
- [ ] Test documented in README or learning log

### Prior gates

- [ ] Chapter 17 cart UI works
- [ ] Chapter 16 cart API works
- [ ] Order model snapshot fields from Ch 5
