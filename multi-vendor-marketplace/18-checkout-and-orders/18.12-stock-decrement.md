# 18.12 — Stock decrement

**Why this step matters:** Atomic decrement prevents **overselling the last item** — conditional `$inc` is production-grade grocery logic, not a nice-to-have. Fail-all if any product cannot decrement.

**Where the project is now:** Orders insert with snapshots; stock unchanged; cart still full.

**After this step:** Checkout atomically decrements `stockQty` per product; **entire checkout fails** if any product insufficient.

---

## Integration point

Inside `checkout(userId)` **before** order insert (or inside same transaction):

For each distinct `productId` in cart (aggregate quantities if duplicate lines):

```
findOneAndUpdate(
  { _id: productId, stockQty: { $gte: qty } },
  { $inc: { stockQty: -qty } }
)
```

If any returns null → throw `InsufficientStockError` → **no orders inserted**.

---

## Fail-all semantics

| Outcome | Cart | Orders | Stock |
|---|---|---|---|
| All decrements OK | cleared in 18.13 | created | reduced |
| Any decrement fails | unchanged | none created | unchanged partial — transaction rollback required |

Without transactions on standalone MongoDB: decrement in loop; on failure, **manually re-increment** products already decremented in that request — document in learning log.

---

## HTTP response on stock failure

**409 Conflict** or **400** with body:

```json
{ "message": "Insufficient stock", "productId": "..." }
```

Client can show which product failed (Chapter 19 checkout UI).

---

## Concurrency test — optional but recommended

1. Set product `stockQty` to 1 in Compass.
2. Two parallel curl checkout requests with that product qty 1.
3. **One succeeds, one fails** — total stock 0, one order only.

---

## Unpublish / delete product

Checkout validation (18.10) should reject unpublished products before decrement. Deleted product → 400, no stock change.

---

## ✅ Do / ❌ Don't

**✅ Do:** Aggregate quantity per productId before single decrement.

**✅ Do:** Wrap decrement + insert + cart clear in transaction when available.

**❌ Don't:** Decrement after cart clear — harder to retry.

**❌ Don't:** Allow negative stockQty in schema validator.
