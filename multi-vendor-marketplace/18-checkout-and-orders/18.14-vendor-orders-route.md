# 18.14 — Vendor orders route

**Why this step matters:** Split orders only matter if vendors can **see their half** — this route scopes by store ownership from Chapter 8 so Vendor B never receives Vendor A's fulfilment list.

**Where the project is now:** Customers get orders on checkout; vendors cannot list them yet.

**After this step:** `GET /vendor/orders` returns orders for **authenticated vendor's store only**, with safe customer info populated.

---

## Route contract

```
GET /api/vendor/orders
Authorization: Bearer <vendor-token>
```

**Middleware:** `requireAuth`, `requireRole('vendor')`.

---

## listOrdersForVendor(userId) — spec

1. Resolve vendor's store: `Store.findOne({ ownerId: userId })` — or your ownership field from Ch 5.
2. If no store → `404` or empty array — document choice.
3. `Order.find({ storeId: store._id }).sort({ createdAt: -1 })`.
4. Populate `customerId` with **display-safe fields** — e.g. `name` or `email` from Profile — never password.

**Do not populate `orderItems.productId` for pricing** — use snapshots.

---

## Response shape — illustrative

```json
{
  "orders": [
    {
      "_id": "...",
      "status": "pending",
      "total": 305,
      "createdAt": "...",
      "customer": { "name": "..." },
      "orderItems": [
        { "nameAtPurchase": "Basmati Rice", "priceAtPurchase": 120, "unitAtPurchase": "kg", "quantity": 2, "lineTotal": 240 }
      ]
    }
  ]
}
```

---

## Isolation test — mandatory

1. Checkout as customer with items from Vendor A and B.
2. `GET /vendor/orders` as Vendor A → sees **only Store A order**.
3. Vendor B token → **only Store B order**.
4. Vendor A cannot see customer lines belonging to B's store.

---

## ✅ Do / ❌ Don't

**✅ Do:** Scope by `storeId` from ownership lookup — not client query param.

**✅ Do:** Return newest orders first.

**❌ Don't:** Accept `storeId` query from client without ownership check.

**❌ Don't:** Expose full customer document.
