# 18.11 — Snapshot order lines

**Why this step matters:** This implements the fix for the trap in 18.2 — copy name, price, unit at commit time so order history never changes when vendors edit the catalogue. Skipping this is the most common checkout bug.

**Where the project is now:** Checkout creates orders; lines may still reference live product fields.

**After this step:** Every `orderItems[]` entry copies **nameAtPurchase**, **priceAtPurchase**, **unitAtPurchase**, **quantity**, **lineTotal** at commit time.

---

## Copy contract — per cart line

At checkout, for each line read product document **once**:

| Snapshot field | Source |
|---|---|
| `productId` | `line.productId` |
| `nameAtPurchase` | `product.name` |
| `priceAtPurchase` | `product.price` |
| `unitAtPurchase` | `product.unit` |
| `quantity` | `line.quantity` |
| `lineTotal` | `priceAtPurchase * quantity` |

Order `total` = sum of `lineTotal` in that order.

**You implement** in `orders.service.js` checkout function — no populate on read for pricing later.

---

## Immutability rule

After `Order.create` / `insertMany`, **no code path** updates snapshot fields on existing orders. Vendor product PATCH does not touch orders.

Status updates (`pending` → `packed`) change `order.status` only — not line prices.

---

## Display rule for Chapter 19

Order history UI reads `priceAtPurchase` and `unitAtPurchase` from embedded lines — never `product.price`.

---

## Edge cases

| Case | Handling |
|---|---|
| Product name has unicode | Copy as-is |
| Price is decimal | Store number; avoid float drift in display |
| Product deleted later | Order still shows snapshot; productId link may 404 |

---

## ✅ Do / ❌ Don't

**✅ Do:** Copy all snapshot fields in same transaction as order insert.

**✅ Do:** Unit-test or curl one line's field values against product at checkout time.

**❌ Don't:** Store only `productId` and quantity.

**❌ Don't:** Recompute lineTotal from live product on GET.

---

## Verify checklist

- [ ] Compass order document shows snapshot fields on each line
- [ ] `total` matches sum of line totals
- [ ] Grocery unit visible in stored line (`kg`, etc.)
