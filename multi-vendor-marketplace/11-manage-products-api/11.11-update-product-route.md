# 11.11 — Update product route

## Why this step matters

Prices change, stock runs low, typos get fixed. `PATCH /products/:id` needs ownership assert on every call — the route Chapter 8's IDOR test foreshadowed. The publish toggle in Chapter 12 also uses this endpoint.

---

**After this step:** `PATCH /api/products/:id` updates own product → **200**. Cross-vendor → **403**. Not found → **404**.

---

## Handler flow

```
PATCH /api/products/:id
  1. requireAuth
  2. requireRole('vendor')
  3. validateObjectId
  4. validateUpdateProduct(body)
  5. assertProductOwner(id, userId)
  6. updateProduct(id, userId, body)
  7. 200 + updated product
```

---

## Mutable fields

Typical PATCH set:

- name, description, price  
- unit, weightOrQuantity  
- stockQty, isPerishable  
- **isPublished** — publish toggle API side for Ch 12  

**Immutable:** `storeId`, `_id` — strip from body.

---

## Service: updateProduct

Ownership verified before update. Partial `$set` semantics.

If product deleted between assert and update — handle 404.

---

## Publish toggle via PATCH

Chapter 12 UI sends:

```json
{ "isPublished": true }
```

No separate `/publish` route required for MVP.

---

## Error matrix

| Condition | Status |
|---|---|
| Owner, valid | 200 |
| Vendor B | 403 |
| Bad id | 404 |
| Invalid body | 400 |
