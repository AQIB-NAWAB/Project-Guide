# 11.12 — Delete product route

## Why this step matters

Vendors discontinue items — hard delete in this course. `DELETE /products/:id` completes the CRUD set and reuses the same ownership chain. Optional in the UI (Ch 12) but required in the API for a complete inventory lifecycle.

---

**After this step:** `DELETE /api/products/:id` removes own product → **204** or **200**. Cross-vendor → **403**.

---

## Handler flow

```
DELETE /api/products/:id
  1. requireAuth
  2. requireRole('vendor')
  3. validateObjectId
  4. assertProductOwner(id, userId)
  5. deleteProduct(id, userId)
  6. 204 No Content (or 200 { deleted: true })
```

Pick **204** for REST convention — document in README.

---

## Service: deleteProduct

Hard delete `Product.findByIdAndDelete` — course default.

**Stretch:** soft delete `isDeleted` flag — note in learning log if chosen instead.

---

## Cart / order implications

If product in active carts — MVP may still delete — orphan cart handling later. Document known limitation in learning log.

Orders use snapshots — historical orders unaffected.
