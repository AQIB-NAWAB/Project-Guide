## Route registration order (spec)

| Order | Method | Path |
|---|---|---|
| 1 | GET | `/vendor/products` |
| 2 | POST | `/` |
| 3 | PATCH | `/:id` |
| 4 | DELETE | `/:id` |

`/vendor/products` on router mounted at `/api/products` **or** separate mount at `/api/vendor/products` — pick one:

| Mount | GET list path |
|---|---|
| `app.use('/api/products', router)` | `GET /api/products/vendor/products` — awkward |
| `app.use('/api', router)` with paths `/products`, `/vendor/products` | cleaner |

**Recommended:** Router defines `GET /vendor/products` and mount at `/api` OR split list to `vendorProducts.routes.js` mounted at `/api/vendor`.

Simplest course path: mount products router at `/api` with routes:

- `POST /products`  
- `GET /vendor/products`  
- `PATCH /products/:id`  
- `DELETE /products/:id`  

Document exact paths in README.
