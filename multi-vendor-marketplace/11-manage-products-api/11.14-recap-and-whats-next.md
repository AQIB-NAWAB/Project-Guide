# 11.14 — Recap and what's next

Chapter 11 completes **vendor product CRUD on the API** — create on own store, scoped list, update, delete, with cross-vendor mutation blocked at 403. The shop has shelves; the browser dashboard comes next.

---

## Narrative recap

**Products module:** routes, service, schema wired to existing model.

**POST /products:** storeId from vendor's store; grocery fields validated.

**GET /vendor/products:** inventory scoped to JWT — drafts included.

**PATCH /products/:id:** partial update including isPublished.

**DELETE /products/:id:** hard delete with ownership.

**Isolation:** Vendor B cannot PATCH/DELETE Vendor A's product.
