# 11.9 — Create product route

## Why this step matters

Stocking the shelf starts here. `POST /products` must resolve `storeId` from the JWT — never trust the client body — and reject vendors without a store. This is the highest-risk IDOR surface: a wrong `storeId` assignment puts kale in another vendor's stall.

---

**After this step:** `POST /api/products` creates product on vendor's store → **201**. No store → error. Customer → **403**. Invalid body → **400**.

---

## Handler flow

```
POST /api/products
  1. requireAuth
  2. requireRole('vendor')
  3. validateCreateProduct(body)
  4. createProduct(userId, validBody)
  5. 201 + product JSON
```

---

## Service: createProduct

| Step | Action |
|---|---|
| 1 | `store = getStoreByVendorId(vendorId)` |
| 2 | If !store → NotFoundError (404) or ForbiddenError — document: **404 "no store"** recommended |
| 3 | Build product doc: validated fields + `storeId: store._id` |
| 4 | Default `isPublished: false` if field exists |
| 5 | Save and return |

Strip `storeId` from client body if present.

---

## Request body (example spec)

| Field | Rule |
|---|---|
| name | required |
| price | required, positive number |
| unit | required enum per model |
| weightOrQuantity | per ERD |
| stockQty | required, >= 0 |
| isPerishable | boolean |
| description | optional |
| isPublished | optional — default false |

Match your Product schema exactly.

---

## Error matrix

| Condition | Status |
|---|---|
| No token | 401 |
| Customer | 403 |
| Vendor, no store | 404 |
| Invalid body | 400 |
| Valid | 201 |

## curl verify

```bash
curl -s -X POST http://localhost:5000/api/products \
  -H "Authorization: Bearer TOKEN_A" \
  -H "Content-Type: application/json" \
  -d '{"name":"Organic Kale","price":2.5,"unit":"bunch","stockQty":20,"isPerishable":true}'
```

Note `PRODUCT_ID` and confirm `storeId` in Compass matches vendor A's store.
