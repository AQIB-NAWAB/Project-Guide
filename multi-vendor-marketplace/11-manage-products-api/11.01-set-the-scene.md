# 11.1 — Set the scene

**Where the project is now:** Vendors have shops — API and UI from Chapters 9–10. A store document exists with `vendorId`. The Product model from Chapter 5 waits in `modules/products/` with grocery fields, but there is no **vendor product API** — no way to stock the shelves programmatically.

**Where this chapter goes:** Vendors **CRUD their own products** through protected routes — create, list mine, update, delete — with **store ownership** enforced and **cross-vendor PATCH → 403**.

This is a **build chapter**. Sub-chapters 11.7 onward add routes and service logic. The gate is curl-proof vendor product lifecycle plus isolation test.

## Bridge from last chapter

Chapters 9–10 gave vendors a **shop** — API and browser forms, with an onboarding gate blocking dashboard routes until a store exists. The Product model from Chapter 5 sits in MongoDB, but there is no way to **stock the shelves** yet. ## Think of it like stocking a stall

The vendor signed the lease (Ch 10) and has an empty stall with a nameplate. Chapter 11 is **filling the shelves via the stockroom door** — API only, no shop window yet. Each crate (product) is tagged with which stall it belongs to. Another vendor cannot relabel your crates because the warehouse checks the ownership chain on every move.

## Common beginner questions

**Q: Why not let customers see products in this chapter?**  
Two different audiences, two different routes. Vendor inventory is authenticated and scoped. Public catalogue is unauthenticated read — Chapter 14. Mixing them early leaks drafts.

**Q: Can the client send `storeId` in the POST body?**  
**No.** Resolve the store from the JWT. Client-supplied `storeId` is an IDOR vector — Vendor B could pass Vendor A's store id.

**Q: What if a vendor has no store and tries to create a product?**  
Reject with 404 or 403 — pick one, document it in `AUTHORIZATION.md`. No store means no shelves.

**Q: Is delete a hard delete?**  
Yes in this course. Soft delete is a valid production choice — note the tradeoff in your learning log.

**Q: Do I need the dashboard UI to test this chapter?**  
No. curl is enough — same as Chapter 9. Chapter 12 builds the screens.

## Why after store chapters

Ownership chain:

```
User (vendor) → Store → Product
```

`POST /products` must resolve **vendor's store** from JWT — reject if no store (404 or 403 — document choice). Cannot create product without shop from Ch 10.
