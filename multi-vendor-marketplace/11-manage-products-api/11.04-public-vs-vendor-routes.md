# 11.4 — Public vs vendor routes

Route **naming and mount order** prevent accidental exposure of vendor inventory on public paths. This sub-chapter specs how Chapter 11 routes coexist with Chapter 14 stubs in `app.js`.

---

## Namespace strategy

| Namespace | Mount | Chapter |
|---|---|---|
| Vendor writes | `/api/products` POST, PATCH, DELETE | 11 |
| Vendor reads | `/api/vendor/products` GET | 11 |
| Public reads | `/api/products` GET | 14 |

**Collision note:** `POST /api/products` and future `GET /api/products` share path — different verbs — OK in REST.

Register routers so **vendor-specific paths** (`/vendor/products`) are not captured by `/:id` param routes incorrectly.

---

## Router organization options

| Option | Structure |
|---|---|
| A — single products router | All verbs in `products.routes.js` |
| B — split routers | `vendorProducts.routes.js` + `publicProducts.routes.js` in Ch 14 |

**Course default:** Option A for Ch 11 — add public GET handlers in same module Ch 14 with clear comments.

---

## Middleware stacks

| Route | Middleware |
|---|---|
| POST /products | requireAuth → requireRole('vendor') → validate → handler |
| GET /vendor/products | requireAuth → requireRole('vendor') → handler |
| PATCH /products/:id | requireAuth → requireRole('vendor') → assertProductOwner → handler |
| DELETE /products/:id | same as PATCH |

Customer calling vendor routes → **403**.

---

## Ownership check strategies

| Strategy | How |
|---|---|
| **Store chain** | Load product → compare `product.storeId` → store.vendorId === userId |
| **Denormalized vendorId** | If product has vendorId field — compare directly |

Prefer **store chain** — matches ERD; one source of truth.

Helper name examples: `assertProductOwner`, `assertStoreOwnsProduct`.

---

## Body tampering vectors

| Field | Rule |
|---|---|
| `storeId` | Strip from POST/PATCH body — set on create from vendor's store |
| `vendorId` | Never on product — reject if sent |
| `price` | Vendor can set — validate positive number |

---

## Response shapes

**Vendor list item:** full product document for own inventory.

**Public item (Ch 14):** may omit internal fields — not defined here.

## AUTHORIZATION.md rows

| Action | Role | Check |
|---|---|---|
| POST /products | vendor | auth + resolve store |
| GET /vendor/products | vendor | auth + store scope |
| PATCH /products/:id | vendor | auth + product ownership |
| DELETE /products/:id | vendor | auth + product ownership |

---

## ✅ Do / ❌ Don't

**✅ Do:** Mount `/vendor/products` **before** `/products/:id` if on same router — avoid "vendor" parsed as id.

**✅ Do:** Document verb split in README.

**❌ Don't:** Add `GET /products` public list in Ch 11.

**❌ Don't:** Let vendor pass another store's id in POST body.
