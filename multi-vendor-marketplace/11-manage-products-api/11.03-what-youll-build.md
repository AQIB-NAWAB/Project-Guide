---

## POST /products — spec highlights

- Resolve vendor's store via `getStoreByVendorId` — no store → 404 or 403  
- Set `storeId` server-side — reject `storeId` in body from client  
- Validate grocery + price fields  
- Default `isPublished: false` if applicable  

---

## GET /vendor/products — spec highlights

- Query `{ storeId: vendorStoreId }`  
- Optional query params later: `?published=true` — not required for gate  
- Sort by `createdAt` desc — reasonable default  

---

## PATCH / DELETE — spec highlights

- `assertProductOwner(productId, vendorId)` or store-chain check  
- Vendor B on Vendor A product → **403**  
- Invalid id → 404
