# 5.9 — Scaffold models folder

**Why this step matters:** Model files belong inside feature modules — `users/`, `stores/`, `products/` — not a global `models/` junk drawer. This step creates the exact paths Chapter 5 sub-chapters fill one schema at a time.

**Where the project is now:** Empty module folders from Chapter 2; no `.model.js` files yet.

**After this step:** Each feature module contains a placeholder or first model file path confirmed; folder structure matches course convention for all six collections.

---

## Target module layout

```
server/src/modules/
├── users/
│   ├── user.model.js           ← 5.10
│   └── profile.model.js        ← 5.10
├── stores/
│   └── store.model.js          ← 5.11
├── products/
│   └── product.model.js        ← 5.12
├── cart/
│   └── cartItem.model.js       ← 5.13
├── orders/
│   └── order.model.js          ← 5.13
└── auth/                       ← routes in Ch 6 — no model here
```

**Naming convention:**

- File: `kebab-case.model.js` or `camelCase.model.js` — pick one, stay consistent
- Model name passed to `mongoose.model()`: PascalCase singular — `'User'`, `'Store'`, `'Product'`, `'CartItem'`, `'Order'`

## Model registration rule

Each model file:

1. Imports mongoose
2. Defines schema
3. Exports **single** default model

**Do not** export schema and model from separate files unless you have reason — one file per model for this course.

**Do not** call `mongoose.model('User', schema)` in multiple files — OverwriteModelError.

---

## Cross-module imports (preview)

Later chapters import across modules:

```
stores.service.js  →  import Store from './store.model.js'
                   →  import Product from '../products/product.model.js'
auth.service.js    →  import User from '../users/user.model.js'
```

Keep model paths stable — `./store.model.js` inside same module folder.
