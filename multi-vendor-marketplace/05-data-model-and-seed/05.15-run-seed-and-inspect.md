# 5.15 — Run seed and inspect

**Why this step matters:** Running seed and opening Compass is the Week 1 **"data is real"** moment. Inspecting every collection catches ref typos and missing grocery fields before you build auth against broken documents.

**Where the project is now:** Seed script committed; may not have been run yet.

**After this step:** Database populated; you have **read every collection** in Compass and confirmed fields match schema spec.

---

## Run seed

```bash
cd server
npm run seed
```

**Expected terminal output (example):**

```
MongoDB connected
Cleared collections (if using clear strategy)
Inserted 4 users, 4 profiles, 2 stores, 16 products
Seed complete
Disconnected
```

Adjust to your log messages — must show success and counts.

---

## Compass inspection checklist

Connect to same database as `MONGODB_URI`. Open each collection:

### users

- [ ] 4 documents
- [ ] Emails match vendor-a/b, customer-a/b@test.local
- [ ] roles correct — 2 vendor, 2 customer
- [ ] passwordHash present — not plain `password` field
- [ ] isVerified true for seed users (Ch 6 login tests)

### profiles

- [ ] 4 documents
- [ ] Each userId matches a user _id
- [ ] displayName populated

### stores

- [ ] 2 documents
- [ ] Each userId points to vendor user only
- [ ] Slugs unique
- [ ] No customer userId on any store

### products

- [ ] 10–20 total across both stores
- [ ] Every product has storeId pointing to valid store
- [ ] unit enum values valid
- [ ] stockQty > 0 on published items
- [ ] Mix of isPublished true/false
- [ ] isPerishable mix
- [ ] **No** vendorName or storeName duplicated on product

### cartitems

- [ ] Empty — expected

### orders

- [ ] Empty — expected

## Cross-document ownership audit

Manual trace in Compass:

1. Pick Product P → note storeId → open Store S → note userId → open User U  
2. Confirm U.role is `vendor`  
3. Repeat for product from other store — different vendor user  

**Failure:** Product.storeId points to store owned by wrong vendor — fix seed logic.

## Troubleshooting

| Symptom | Fix |
|---|---|
| E11000 duplicate key | Idempotency bug — clear or upsert |
| ValidationError | Seed missing required field — check model vs seed |
| 0 products | storeId not set after store insert |
| Wrong database | seed URI ≠ server URI — unify config |
| Connection refused | MongoDB not running |

## ✅ Do / ❌ Don't

**✅ Do:** Read raw JSON — don't trust counts alone.

**✅ Do:** Note collection names (users vs user) — Mongoose pluralization.

**❌ Don't:** Manually edit Compass docs without updating seed — drift breaks teammates.

**❌ Don't:** Seed production Atlas with clear-all strategy without confirmation.
