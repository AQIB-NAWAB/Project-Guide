# 5.5 — Grocery domain fields

A generic `{ name, price, description }` product schema could serve any marketplace. FreshMarket is **grocery** — bananas sell by the bunch, flour by the kilogram, milk expires. Your Product model must reflect how local shops actually sell food.

This sub-chapter specifies **domain fields** mandated for this course. Implement them in 5.12; reference them in seed data and vendor dashboard forms (Chapter 12).

## Why this matters for absolute beginners

Without `unit`, your UI cannot show "£2.49 / bunch" vs "£2.49 / kg" — a grocery bug that generic tutorials never mention. Without `stockQty`, checkout cannot fail when shelves are empty. These fields connect **database shape** to **customer trust** — receipts and shelves tell the same story.

## Common beginner questions

**Q: Can I use string price for display formatting?**  
A: Store **Number** in the database; format in UI. Math and checkout need numbers.

**Q: What if my local shop sells by weight at counter?**  
A: MVP cart uses integer unit counts — weighted items are stretch. Still model `unit` enum for kg/pack.

**Q: Is `isPublished` necessary before catalogue API?**  
A: Yes — vendors draft products before they go live; public catalogue filters published items (Chapter 14).

## Why domain fields matter

| Generic e-commerce | Grocery reality |
|---|---|
| "Add to cart" × 1 | Customer buys 2 bunches or 500g |
| Price is enough | Price **per unit** — £/kg, £/each |
| Stock is optional | Out-of-stock blocks checkout |
| All products equal | Perishables need fulfilment urgency |

Interviewers notice when your portfolio marketplace understands its domain.

## Product schema — grocery field specification

| Field | Type | Required | Example | Notes |
|---|---|---|---|---|
| `name` | String | yes | "Organic Bananas" | Search index in Ch 20 |
| `description` | String | no | "Fairtrade, Ecuador" | Editorial tone |
| `price` | Number | yes | 2.49 | **Per unit** — see `unit` |
| `unit` | String enum | yes | `each`, `bunch`, `kg`, `g`, `L`, `ml`, `pack` | Display as "£2.49 / bunch" |
| `weightOrQuantity` | Number | no | 500 | Meaning depends on unit — 500g or 6-pack |
| `stockQty` | Number | yes | 40 | Integer; checkout decrements (Ch 18) |
| `isPerishable` | Boolean | yes | true | UI badge; fulfilment priority hint |
| `category` | String | yes | `produce`, `dairy`, `bakery`, `pantry`, `frozen` | Filter param in Ch 14 |
| `isPublished` | Boolean | yes | false | Draft vs live catalogue |
| `imageUrl` | String | no | Cloudinary URL | Chapter 13 — null until upload |
| `storeId` | ObjectId ref | yes | — | Ownership chain |

**Enum for `unit`:** Pick a fixed list in schema — prevents "Bunch" vs "bunch" chaos. Document your enum in `docs/erd.md`.

---

## Interpreting `weightOrQuantity`

Not every product needs it. Use consistently:

| Product | unit | weightOrQuantity | Display |
|---|---|---|---|
| Bananas | bunch | null or 1 | £2.49 / bunch |
| Cheddar | kg | null | £8.99 / kg — customer selects weight at cart (stretch) |
| Rice 500g | pack | 500 | £1.89 / pack (500g) |
| Milk 1L | L | 1 | £1.20 / L |

For MVP cart, quantity in CartItem is **count of units** — not grams. Advanced: weighted items — out of course scope unless stretch goal.

## Stock and checkout (preview)

Chapter 18 checkout must:

1. Verify `stockQty >= cartQuantity` for each line
2. Atomically `$inc: { stockQty: -quantity }` with guard `stockQty >= quantity`
3. Fail entire checkout if any line insufficient

Schema must have `stockQty` as Number defaulting to 0 — not undefined.

## Perishability — business use

`isPerishable: true` does not change API logic in MVP. It enables:

- Vendor dashboard badge (Ch 12)
- Future delivery window rules
- Portfolio narrative in demo video

Seed script: mix true/false across products.

## Category and filters

Chapter 14 public catalogue accepts `?category=produce`. Categories must match enum or controlled vocabulary in seed data.

Suggested seed mix per store:

- 2 produce (1 perishable)
- 2 dairy (perishable)
- 2 pantry (non-perishable)
- 1 bakery

## Fields explicitly out of scope

| Field | Why omitted |
|---|---|
| `nutritionalInfo` | Scope — optional stretch |
| `allergens[]` | Real grocery needs it; not MVP |
| `expiryDate` | Per-product expiry — complex; use isPerishable flag |
| `sku` / barcode | B2B inventory — not local market MVP |
| `discountPrice` | Promotions — out of scope |

## Comparison — Product vs Order line embed

| Field | Live Product | Embedded orderItem |
|---|---|---|
| name | `name` | `nameAtPurchase` |
| price | `price` | `priceAtPurchase` |
| unit | `unit` | `unitAtPurchase` |
| stockQty | yes | no — historical |
| isPerishable | yes | optional copy if needed |

Never update order embed when product price changes.

## Seed data quality bar

Each seeded product must have:

- [ ] Valid `unit` from enum
- [ ] `stockQty` > 0 for catalogue demos
- [ ] At least one `isPublished: true` per store
- [ ] Realistic price for unit (not £999/kg)

## ✅ Do / ❌ Don't

**✅ Do:** Document unit enum in README or ERD for frontend form dropdown.

**✅ Do:** Default `isPublished: false` — vendor explicitly publishes (Ch 12).

**❌ Don't:** Store price as string — Number type for math and checkout.

**❌ Don't:** Omit `storeId` — breaks entire ownership model.
