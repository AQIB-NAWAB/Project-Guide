# 5.12 — Product model

**Why this step matters:** Products carry FreshMarket's grocery identity — unit, stock, perishability — not generic e-commerce fields. This schema is what customers browse and vendors edit; catalogue and checkout chapters assume these shapes exist.

**Where the project is now:** User, Profile, and Store models exist. No Product schema yet.

**After this step:** `product.model.js` defines grocery catalogue items with `storeId` ref and domain fields from 5.5.

---

## Product schema specification

| Field | Type | Required | Default | Notes |
|---|---|---|---|---|
| `name` | String | yes | — | Searchable — text index Ch 20 |
| `description` | String | no | — | |
| `price` | Number | yes | — | Per `unit`; min 0 |
| `unit` | String enum | yes | — | See 5.5 enum list |
| `weightOrQuantity` | Number | no | — | Pack size hint |
| `stockQty` | Number | yes | 0 | Integer ≥ 0 |
| `isPerishable` | Boolean | yes | false | |
| `category` | String | yes | — | Controlled vocabulary |
| `isPublished` | Boolean | yes | false | Draft until vendor publishes |
| `imageUrl` | String | no | null | Cloudinary URL — Ch 13 |
| `storeId` | ObjectId ref Store | yes | — | Ownership |
| timestamps | auto | yes | — | |

**Indexes:**

- `{ storeId: 1 }` — vendor product list
- `{ storeId: 1, isPublished: 1 }` — catalogue queries (Ch 14)
- Optional: `{ name: 'text', category: 'text' }` — defer to Ch 20 or note in ERD

## Enum — unit (mandated minimum)

```
each | bunch | kg | g | L | ml | pack
```

Document exact enum in model and `docs/erd.md`.

---

## Enum — category (suggested)

```
produce | dairy | bakery | pantry | frozen | beverages | household
```

Adjust to your product name — keep list finite for filter dropdown.

---

## Relationship

```
Store (_id) ←—— Product.storeId
```

**No** `userId` on Product — ownership via Store.userId chain only.

---

## Published vs draft semantics

| isPublished | Visible in public GET /products (Ch 14) |
|---|---|
| false | Hidden — vendor dashboard only |
| true | Catalogue if stockQty > 0 |

Seed: at least 3 published products per store for Week 2 demo prep.

---

## Stock rules (schema + app)

- Schema: `stockQty` Number, min 0 validator optional
- App (Ch 16–18): reject add-to-cart and checkout when insufficient

Do not use negative stock to mean "unlimited".
