## Decisions you can defend in a viva

1. **Why User/Profile split?** — Auth rotation vs display data separation.  
2. **Why storeId on Product not userId?** — One store per vendor; products belong to shop entity.  
3. **Why storeId on CartItem?** — Checkout grouping without extra Product lookup.  
4. **Why embed orderItems?** — Snapshots immutable; no separate orderlines collection.  
5. **Why not vendorName on Product?** — Denormalisation trap — populate instead.  
6. **Why empty cart/orders in seed?** — Created by feature chapters; models exist first.

## Next chapter — Authentication API (Chapter 6)

Chapter 6 adds **server-side auth only**:

- `POST /api/auth/register` — hash password, create User + Profile  
- `POST /api/auth/login` — bcrypt compare, issue JWT  
- `POST /api/auth/refresh` — refresh token rotation  
- `POST /api/auth/verify-otp` — block unverified login  
- `requireAuth` middleware — 401 on bad/missing token  

**Reuses from Chapter 5:**

- User and Profile models  
- Seed users get real password hashes if not already  
- Test emails from seed for curl flows  

**New env vars (preview):** `JWT_SECRET`, `JWT_ACCESS_EXPIRY`, `JWT_REFRESH_EXPIRY`
