# 5.6 — Prime your thinking

Before you write `user.model.js`, read how Mongoose expects schemas, refs, and model registration to work. Twenty minutes here saves hours of "populate returned null" debugging.

## Why this matters for absolute beginners

Mongoose errors are opaque when fundamentals are shaky — `OverwriteModelError`, populate returning null, validation failing silently on seed. The reads below connect **ERD arrows** to `ref: 'Store'` and embedded `orderItems` arrays. You are not learning a second database; you are learning the **glue** between Chapter 3 diagrams and Chapter 5 files.

## Common beginner questions

**Q: Schema vs model — what's the one-line difference?**  
A: Schema is the blueprint; model is the compiled constructor you query with `User.create()`.

**Q: Why unique index on email?**  
A: Duplicate emails break login — "which user is this password for?"

**Q: pre-save password hash in Ch 5 or Ch 6?**  
A: Hashing lands in auth service Chapter 6 — schemas may store `passwordHash` field now; seed may use placeholder until bcrypt exists.

**Q: Collection name `users` vs file `user.model.js`?**  
A: Mongoose pluralizes by default — know what Compass shows vs what you export.

## Concepts to extract from reading

### 1. Schema vs Model

- **Schema** — blueprint (fields, types, rules)
- **Model** — compiled constructor bound to collection name

Pattern for this course:

```
user.model.js  →  defines schema + exports mongoose.model('User', userSchema)
```

Other files `import User from './user.model.js'` — never recompile model.

### 2. ObjectId refs

```javascript
storeId: { type: Schema.Types.ObjectId, ref: 'Store', required: true }
```

Populate only works when:

- `ref` string matches registered model name exactly
- Referenced document exists
- You call `.populate('storeId')` on query

### 3. Subdocuments vs refs

| Pattern | Use when |
|---|---|
| **Ref** | Entity has own lifecycle (Product, Store) |
| **Embed subdocument array** | Snapshot lines on Order — no independent Product |

Order `orderItems[]` is embedded — not a separate collection.

### 4. Timestamps

```javascript
{ timestamps: true }
```

Adds `createdAt`, `updatedAt` — use on all top-level schemas this chapter.

### 5. Indexes

Declare in schema or ERD:

- `User.email` — unique
- `Store.userId` — unique (one store per vendor)
- `Product.storeId` — index for vendor queries
- `CartItem.userId` — index for cart fetch

Read: [Mongoose indexes](https://mongoosejs.com/docs/indexes.html)

## Common mistakes (learn now, avoid in 5.10–5.13)

| Mistake | Symptom |
|---|---|
| Model defined twice | OverwriteModelError |
| Wrong ref name | populate returns null id only |
| Forgot `required: true` | Seed saves incomplete docs silently |
| Mixed plural collection names | Mongoose defaults to lowercase plural — know your collection names in Compass |
| Schema typos in enum | ValidationError on seed |
