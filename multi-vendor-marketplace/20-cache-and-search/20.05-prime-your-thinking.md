## In plain English

| Pattern | Simple meaning |
|---|---|
| **Cache-aside** | Application code decides when to read/write cache — you control it in middleware or service |
| **Read-through** | Cache library fetches from DB automatically — not what this course uses |
| **Thundering herd** | TTL expires → 100 users all miss cache at once → 100 MongoDB queries simultaneously |

---

## Common beginner mistakes (read before coding)

1. **Caching before measuring** — 20.7/20.10 want before/after numbers; measure baseline first.
2. **One global cache key** — filters and pagination need distinct keys or you serve wrong pages.
3. **Forgetting invalidation on DELETE** — product gone from DB but still in cache.
4. **`KEYS catalogue:*` in production** — blocks Redis; use `SCAN` in invalidation helper.
5. **Caching authenticated responses on public route** — catalogue is public; cart is not.
