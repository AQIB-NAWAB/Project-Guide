# 20.13 — Explain query

**Why this step matters:** "We added search" is weak in a viva — `explain()` proof that MongoDB uses your index, not a full collection scan, shows you verified performance instead of assuming.

**After this step:** You have proof that search uses **text index** — not collection scan.

---

## Run explain

In script, mongosh, or temporary route — **dev only**:

```javascript
const explain = await Product.find(
  { $text: { $search: 'rice' } },
  { score: { $meta: 'textScore' } }
).explain('executionStats')
```

Inspect `executionStats.executionStages` or `queryPlanner.winningPlan`.

---

## Pass criteria

| Result | Gate |
|---|---|
| `TEXT` or `IXSCAN` on text index | ✓ Pass |
| `COLLSCAN` on large collection | ✗ Fix index |

Small seed may still be fast on COLLSCAN — **index must exist** in Compass indexes tab regardless.
