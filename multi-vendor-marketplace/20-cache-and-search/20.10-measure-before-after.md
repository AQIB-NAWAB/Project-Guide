# 20.10 — Measure before/after

**Why this step matters:** Interviewers ask "did caching help?" — honest before/after numbers in README turn a tutorial exercise into evidence. Even modest gains on a small seed prove you measured, not guessed.

**After this step:** README documents **response time** for catalogue GET — cold vs cache HIT.

---

## Measurement contract

Same URL, same conditions:

```bash
# Miss — new key or FLUSHDB in dev only
time curl -s "http://localhost:5000/api/products?page=1&limit=20" -o /dev/null

# Hit — repeat immediately
time curl -s "http://localhost:5000/api/products?page=1&limit=20" -o /dev/null
```

Or log `Date.now()` delta in middleware — one line per request in dev.
