# 20.12 — Text index

**Why this step matters:** Regex search on every product row slows down as the catalogue grows. A text index is the MongoDB-native way to make `?search=rice` scale — Week 4 demo expects search on production.

**After this step:** Product collection has **text index** on `name` and `category`; `GET /products?search=` uses `$text` query.

---

## Index creation — spec

Run once via script or startup migration flag:

```javascript
// Illustrative — you implement
Product.collection.createIndex(
  { name: 'text', category: 'text' },
  { name: 'product_text_search' }
)
```

Document in README how to create index on fresh deploy.

---

## Search query contract

When `search` query param present and non-empty:

```javascript
Product.find(
  { $text: { $search: searchTerm }, isPublished: true, ...otherFilters },
  { score: { $meta: 'textScore' } }
).sort({ score: { $meta: 'textScore' } })
```

Combine with existing pagination — `page`, `limit` from Ch 14.

**Sanitize:** Reject or strip characters that break `$search` — basic trim minimum.

---

## Cache interaction

Search responses cache with key including `search=` param — invalidation same as list.

---

## Fallback without search term

No `search` param → existing list query without `$text`.

---

## ✅ Do / ❌ Don't

**✅ Do:** Index both name and category — "Dairy" finds category.

**✅ Do:** Only search published products.

**❌ Don't:** Case-sensitive regex on unindexed fields for gate — use text index.
