# 20.8 — Redis connection

**Why this step matters:** Every cache and queue feature in Chapters 20–21 shares one Redis connection module — wire it once correctly here and both chapters inherit the same `REDIS_URL` discipline.

**After this step:** `server/src/shared/redis.js` connects to Redis from `REDIS_URL`; fails gracefully or logs warning per 20.14 strategy.

---

## Env contract

```
# server/.env.example
REDIS_URL=redis://127.0.0.1:6379
```

Add to `config.js` validation — **optional** for dev if fallback required, or **required** if you want fail-fast. Course gate requires **fallback when down** (20.14) — optional URL with `getRedisClient()` returning null is valid pattern.

---

## redis.js exports — spec

| Export | Purpose |
|---|---|
| `connectRedis()` | Call on boot after MongoDB or parallel |
| `getRedisClient()` | Returns client or null if unavailable |
| `disconnectRedis()` | Graceful shutdown optional |

Use `node-redis` v4+ or `ioredis` — document package in README.

---

## Boot wiring

`server.js` — attempt Redis connect; log success or warning. **Do not block HTTP** if Redis optional for fallback design.

---

## Commit

```bash
git add server/src/shared/redis.js server/.env.example server/package.json
git commit -m "feat(server): add redis connection module"
```

---

## ✅ Do / ❌ Don't

**✅ Do:** Update `.env.example` — not real credentials.

**✅ Do:** Single client instance — connection pool.

**❌ Don't:** Put Redis password in client `.env`.
