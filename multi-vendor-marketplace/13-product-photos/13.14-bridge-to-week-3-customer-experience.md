# 13.14 — Bridge to Week 3 (customer experience)

You finished **Week 2 — vendor experience**. Take a breath. This sub-chapter exists because the shift to Week 3 is the biggest **role change** in the course — not because Week 3 is harder, but because you stop thinking like a shop owner and start thinking like a shopper.

---

## What you just finished (Week 2 recap)

As a **vendor**, you built:

- Login and registration (Ch 6–7)  
- Proof that vendors cannot touch each other's data (Ch 8)  
- Opening a shop in the browser (Ch 9–10)  
- Adding products with grocery fields and photos (Ch 11–13)  

**Week 2 demo:** A vendor can log in, open a shop, add a product with a photo, and see it in their dashboard.

If that demo is not true in your browser yet, **do not start Chapter 14**. Week 3 builds on seeded products in MongoDB that vendors created — empty shops make Week 3 feel broken when the problem is Week 2.

---

## What changes in Week 3

| Week 2 mindset | Week 3 mindset |
|---|---|
| "I am the seller" | "I am the buyer" |
| Routes like `/vendor/products` | Routes like `/products` (public catalogue) |
| Forms that **create** data | Pages that **browse** and **buy** |
| One store at a time | **Many stores** in one cart |
| curl with vendor token | curl + browser as **customer** |

You are **not** throwing away vendor work. Customers will browse the **same products** vendors published — new API routes sit **beside** vendor routes, not instead of them.

---

## In plain English

| Term | Simple meaning |
|---|---|
| **Public catalogue** | The shop window — products any customer can see without being a vendor |
| **Vendor-scoped route** | API that only shows **your** inventory when **you** are logged in as vendor |
| **Customer** | User with role `customer` — buys from many vendors |
| **Multi-vendor cart** | One shopping cart with items from Shop A and Shop B — checkout splits into two orders later |

---

## Think of it like a real market hall

Week 2 you built **stalls** (stores) and **shelves** (products). Week 3 you build the **aisles** where shoppers walk, pick items from different stalls, and pay once at the end — the system splits money and orders per stall behind the scenes (Chapter 18).

## Common beginner questions

**Q: Do I delete vendor pages?**  
No. Vendor dashboard stays. You **add** customer pages (`/shop`, `/cart`, etc.).

**Q: Why another chapter before the shop UI (Chapter 14)?**  
Same pattern as always: **API first, UI second** (Ch 6→7, Ch 11→12). Chapter 14 builds `GET /products` so Chapter 15's React grid has real data — not fake arrays in components.

**Q: I'm confused about auth — do customers log in to browse?**  
Your public catalogue can allow anonymous browse or require login — course gate usually allows browse, requires login for cart/checkout. Chapter 14 spec defines public read; cart (Ch 16) requires customer auth.

**Q: Week 3 feels sudden.**  
Re-read this page whenever that happens. The course intentionally pauses here so the role switch is explicit — not a hidden jump.
