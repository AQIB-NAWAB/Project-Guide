# 13.9 — Upload route

## Why this step matters

JSON routes cannot accept files — multipart needs multer and a dedicated endpoint. This route streams bytes to Cloudinary, runs ownership checks, and saves only the URL in MongoDB. It completes the server half of the photo pipeline.

---

**Where the project is now:** Cloudinary helper exists; no HTTP endpoint accepts files.

**After this step:** Authenticated vendors POST multipart image to own products; MongoDB updated with URL; old image not deleted yet (13.10).

---

## Install multer

From `server/`:

```bash
npm install multer
```

---

## Multer middleware spec

Create **`server/src/modules/products/upload.middleware.js`**:

| Setting | Value |
|---|---|
| Storage | `multer.memoryStorage()` |
| Field name | `image` (client must match) |
| Max file size | 5 MB |
| Allowed MIME | `image/jpeg`, `image/png`, `image/webp` |

Export middleware e.g. `uploadProductImage = upload.single('image')`.

**Error handling:**

| Multer error | Map to |
|---|---|
| `LIMIT_FILE_SIZE` | 400 + message |
| Unexpected field | 400 |
| No file | 400 — handled in route after multer |

---

## Route contract

| Field | Value |
|---|---|
| Method | POST |
| Path | `/products/:id/image` (under your API prefix) |
| Middleware chain | `requireAuth` → `requireRole('vendor')` → `uploadProductImage` → handler |

**Handler sequence (spec):**

1. Load product by `:id`  
2. Assert vendor owns product's store — reuse ownership from Chapter 11  
3. If `!req.file` → 400  
4. Call `uploadProductImage(req.file.buffer, { productId, mimetype })`  
5. Update product `imageUrl`, `imagePublicId` — **do not delete old yet**  
6. Return updated product JSON (exclude internal fields)

---

## Response shape

**Success 200:**

```json
{
  "id": "674a...",
  "name": "Organic Tomatoes",
  "imageUrl": "https://res.cloudinary.com/...",
  "imagePublicId": "freshmarket/products/674a-...",
  "storeId": "...",
  "isPublished": true
}
```

Match your existing product DTO conventions — same fields as GET vendor product where possible.

---

## Error matrix

| Condition | Status |
|---|---|
| Not authenticated | 401 |
| Not vendor role | 403 |
| Product not found | 404 |
| Not product owner | 403 |
| No file / bad type / too large | 400 |
| Cloudinary error | 502 or 500 — generic client message |

---

## Wire in router

Mount on existing `products.routes.js` — **before** any `/:id` PATCH that might conflict if ordering matters. Express matches in order — document route registration order.

Do not accept image in JSON body on PATCH — keep upload isolated.

## ✅ Do / ❌ Don't

**✅ Do:** Run ownership check before Cloudinary call — saves failed uploads on 403.

**✅ Do:** Use same error JSON shape as other product routes.

**❌ Don't:** Accept `image` as base64 string in JSON on this route.

**❌ Don't:** Skip file type validation server-side — client checks are not enough.
