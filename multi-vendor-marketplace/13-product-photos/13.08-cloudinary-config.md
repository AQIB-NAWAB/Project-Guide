# 13.8 — Cloudinary config

## Why this step matters

Upload credentials belong in server env — never the React bundle. This step wires Cloudinary the same way Chapter 4 wired MongoDB: validate at boot, document in `.env.example`, fail fast if missing. The shared helper keeps route handlers thin.

---

**Where the project is now:** Product CRUD works; no upload infrastructure; `imageUrl` empty on documents.

**After this step:** Cloudinary credentials load from env via `config.js`; a shared upload helper wraps the SDK — ready for multer in 13.9.

---

## Environment variables

Add to `server/.env` (local) and `server/.env.example` (committed):

| Variable | Example placeholder |
|---|---|
| `CLOUDINARY_CLOUD_NAME` | `your-cloud-name` |
| `CLOUDINARY_API_KEY` | `123456789012345` |
| `CLOUDINARY_API_SECRET` | `your-api-secret` |

**Update `shared/config.js`:**

- Read all three vars  
- If any missing or empty → fail at startup (same pattern as `MONGODB_URI`)  
- Export e.g. `{ cloudinary: { cloudName, apiKey, apiSecret } }`

From `server/`:

## Upload helper contract

Create **`server/src/shared/cloudinary.js`** (path your choice — stay consistent):

| Export | Input | Output | Behaviour |
|---|---|---|---|
| `configureCloudinary()` | — | void | Call once at module load or first use — `cloudinary.config({ cloud_name, api_key, api_secret })` |
| `uploadProductImage(buffer, options)` | Buffer, `{ productId, mimetype }` | `{ url, publicId }` | Upload to folder `freshmarket/products/`; return `secure_url` and `public_id` |
| `deleteProductImage(publicId)` | string | void or boolean | Call destroy; swallow/log non-fatal errors in replace flow |

**You implement** — course specifies exports and return shape, not full SDK wrapper code.

**Options spec for upload:**

| Option | Value |
|---|---|
| `resource_type` | `'image'` |
| `folder` or `public_id` prefix | `freshmarket/products/{productId}-...` |
| Format | preserve or force jpg — document choice |

Return mapping:

```
secure_url  →  product.imageUrl
public_id   →  product.imagePublicId
```

## ✅ Do / ❌ Don't

**✅ Do:** Use `secure_url` not `url` if HTTP variant returned.

**✅ Do:** Configure SDK once — avoid re-config on every request.

**❌ Don't:** Log API secret on startup.

**❌ Don't:** Import cloudinary helper in client code.
