# 13.11 — Upload UI component

## Why this step matters

Vendors choose photos in the browser — file input, preview, upload progress. This component calls the multipart route from 13.9 and updates the edit page. Thumbnails in the product list make Week 2's demo feel like a real shop, not a spreadsheet.

---

**Where the project is now:** API accepts multipart upload; vendor dashboard has no file picker.

**After this step:** Vendor selects image on edit (or post-create) page; preview updates; product list shows thumbnails.

---

## UX placement options

| Pattern | Flow | Gate |
|---|---|---|
| Edit page only | Create product → redirect to edit → upload | ✅ |
| Create + upload | Two-step wizard on create page | ✅ |
| Inline on create | Create then auto-upload before redirect | ✅ |

Pick one; document in README. Simplest: **upload on EditProduct** after product exists (needs `:id` for route).

---

## Component spec

Create **`client/src/components/ImageUpload.jsx`** (or equivalent):

| Prop | Type | Purpose |
|---|---|---|
| `productId` | string | POST target |
| `currentUrl` | string? | Show existing thumbnail |
| `onSuccess` | `(product) => void` | Parent refreshes state |

**UI elements:**

- Hidden or styled file input — accept `image/jpeg,image/png,image/webp`  
- Optional preview of selected file before upload  
- Upload button or auto-upload on select  
- Loading state during request  
- Error message area  

**You implement** styling — match vendor dashboard editorial table aesthetic from Chapter 12.

|---|---|

## EditProduct integration

- Pass `product.id` and `product.imageUrl` to ImageUpload  
- On success, update local product state or refetch product  
- Show success toast/message optional  

CreateProduct: if upload only on edit, show hint — *"Save product, then add photo on edit page."*

---

## Client validation (UX only)

| Check | Message |
|---|---|
| File > 5 MB | Before upload — matches server limit |
| Wrong type | Before upload |
| No file selected | Disable upload button |

Server remains source of truth — never skip server validation.

---

## Error mapping

| Status | UI |
|---|---|
| 400 | Inline — file type/size |
| 403 | "You cannot edit this product" |
| 401 | Redirect login via interceptor |
| 502 | Generic retry message |
