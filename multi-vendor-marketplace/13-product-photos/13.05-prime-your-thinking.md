# 13.5 — Prime your thinking

Before you wire multer and Cloudinary, read how **server-side uploads** work in the Cloudinary Node SDK. This sub-chapter points you at the right docs — not a tutorial replacement.

---

## Mandatory reads

Read **before** sub-chapter 13.8:

1. **Cloudinary — Node.js image and video upload**  
   Search: `Cloudinary node upload stream buffer`  
   Focus: uploading from a buffer (multer memory storage output).

2. **Cloudinary — Upload API reference**  
   Focus: `folder`, `public_id`, `resource_type: 'image'`, response fields `secure_url`, `public_id`.

3. **Cloudinary — Delete resources**  
   Focus: `cloudinary.uploader.destroy(public_id)` — needed for 13.10.

4. **Multer documentation — single file**  
   Focus: `upload.single('fieldname')`, `limits.fileSize`, `fileFilter`.

Skim only if short on time: Cloudinary transformation URLs — useful for Chapter 15 thumbnails, not required for gate.

## Multer + Cloudinary mental model

```
HTTP multipart  →  multer  →  req.file.buffer
                              req.file.mimetype
                              req.file.size
                                    ↓
                          cloudinary.uploader.upload_stream(...)
                                    ↓
                          { secure_url, public_id }
                                    ↓
                          products.service.updateImage(...)
```

Read multer's `memoryStorage` limits — large files consume RAM per request.

## Error cases to plan for (spec, not code)

| Failure | User sees | Server logs |
|---|---|---|
| File too large | 400 "Image must be under 5MB" | size + product id |
| Wrong MIME | 400 "Only JPEG, PNG, WebP" | mimetype |
| Cloudinary timeout | 502 generic | full error server-side |
| 403 ownership | 403 same as PATCH product | attempt audit optional |
