# 15.12 — Checklist and learning log

**Why this step matters:** Chapter 16 assumes customers can pick products from **two stores** in the UI. Clearing this gate honestly prevents cart demos that fail because browse was never finished.

Gate for Chapter 15. Learning log: **`learning-log/15-shop-the-marketplace.md`**.

---

## Definition of Done

### Routes & navigation

- [ ] `/shop` renders Shop page
- [ ] `/shop/:productId` renders ProductDetail
- [ ] Nav link to shop from home or global header
- [ ] No collision with vendor `/vendor/*` routes

### Shop page

- [ ] Fetches `GET /products` with page, limit, optional category, storeId
- [ ] Renders grid of ProductCard components
- [ ] Loading and error states handled
- [ ] Empty state when no products match filters
- [ ] Pagination controls (Prev/Next or equivalent) using `totalPages`
- [ ] Filter change resets to page 1

### ProductCard

- [ ] Shows imageUrl or placeholder
- [ ] Shows name, price/unit, store.name
- [ ] Links to product detail

### Filters

- [ ] Category filter refetches API with `category` param
- [ ] Store filter refetches API with `storeId` param
- [ ] Clear filters restores full catalogue

### Product detail

- [ ] Fetches `GET /products/:id`
- [ ] Shows image, name, price, unit, store, description if available
- [ ] 404 handling for missing product
- [ ] Add to cart button present — stub until Ch 17 (logs or TODO comment with productId + storeId)

### Multi-vendor browse (gate)

- [ ] Shop grid shows products from **≥2 stores** (seed permitting)
- [ ] Verified in browser — not API-only

### API discipline

- [ ] No duplicate publish/stock filtering in client
- [ ] No per-card detail API calls on grid
- [ ] Uses `client/src/api/products.js` helpers

### Prior gates

- [ ] Chapter 14 public catalogue API complete
- [ ] Chapter 13 images display from Cloudinary URLs
