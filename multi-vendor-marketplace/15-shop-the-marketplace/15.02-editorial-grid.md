# 15.2 — Editorial grid

The shop page is a **grid of cards** — not a dense data table. Each card is a scannable unit of trust: photo, name, price, who sells it.

This sub-chapter defines the **card contract** you implement in 15.6 and consume from Chapter 14 API shape.

---

## In plain English

| Term | Simple meaning |
|---|---|
| **Editorial grid** | A clean photo-first layout — like a food magazine, not an Excel export |
| **Product card** | One box in the grid — everything a customer needs to decide "do I want to learn more?" |
| **Line-clamp** | CSS that truncates long product names to two lines so cards stay equal height |
| **Placeholder image** | Grey box or icon when `imageUrl` is missing — layout must not collapse |

**Beginner mistake:** Building a table with columns `id`, `storeId`, `price` because that is what MongoDB stores. Customers need **photos and plain language**, not database columns.

---

## Product card — required elements

| Element | Source field | Notes |
|---|---|---|
| Photo | `imageUrl` | Placeholder if missing — grey or icon |
| Product name | `name` | Line-clamp 2 lines max |
| Price | `price` + `unit` | e.g. `$3.50 / kg` |
| Store name | `store.name` | Secondary text — multi-vendor identity |
| Link | `id` | Whole card or title links to `/shop/:id` |

**Optional for MVP:** category badge, stock indicator, perishable icon.

---

## Layout spec

| Viewport | Columns |
|---|---|
| Mobile | 1–2 |
| Tablet | 2–3 |
| Desktop | 3–4 |

Use CSS grid or flex — no UI library mandated. Consistent gap between cards.

**Image aspect ratio:** square or 4:3 with `object-fit: cover` — avoids layout jump when Cloudinary images vary.

---

## Empty and loading states

| State | UI |
|---|---|
| Loading | Skeleton cards or spinner — not blank page |
| Empty filter | "No products match" + clear filters action |
| API error | Message + retry — reuse Ch 2 error pattern |
| No imageUrl | Placeholder — do not break card layout |

---

## What not to put on card

| Exclude | Why |
|---|---|
| stockQty exact (optional) | Can show "Low stock" later |
| store owner email | PII |
| vendor internal ids | No customer value |
| Add to cart on card | Detail page first — reduces mis-clicks; stretch OK |

Gate minimum: navigate to detail; add-to-cart on detail as stub.

---

## Typography hierarchy

1. **Product name** — primary weight  
2. **Price / unit** — secondary, tabular nums if available  
3. **Store name** — tertiary, muted color  

Editorial = restraint — not 12 badges per card.

---

## Accessibility

- `alt` on images — product name or empty alt if decorative with visible title  
- Card link keyboard focusable  
- Filter controls labeled  

---

## ✅ Do / ❌ Don't

**✅ Do:** Reuse one `ProductCard` component on shop grid.

**✅ Do:** Handle missing `store` gracefully — "Unknown store" string.

**❌ Don't:** Fetch `/products/:id` for every card on grid — list payload enough.

**❌ Don't:** Hardcode product array — always from API.
