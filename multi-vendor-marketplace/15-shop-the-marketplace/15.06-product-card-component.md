# 15.6 — Product card component

**Why this step matters:** The product card is the **atom** of the marketplace UI — reused on every row of the grid. Nailing photo, price/unit, and store name once keeps the multi-vendor identity visible on every tile.

**Where the project is now:** Shop page shell exists; no grid rendering.

**After this step:** `ProductCard` displays photo, name, price/unit, store — links to detail route.

---

## Component contract

**File:** `client/src/components/ProductCard.jsx`

| Prop | Type | Required |
|---|---|---|
| `product` | object | yes |

**Expected product shape** (from Ch 14 API):

```javascript
{
  id: string,
  name: string,
  price: number,
  unit: string,
  imageUrl?: string,
  store?: { id: string, name: string }
}
```

---

## Render spec

| Region | Content |
|---|---|
| Image area | `<img src={imageUrl} />` or placeholder |
| Title | `name` — link to `/shop/${id}` |
| Price line | formatPrice(price, unit) — you implement formatter |
| Store line | `store?.name ?? 'Unknown store'` |

Whole-card clickable **or** title-only link — pick one; whole-card better UX.

---

## Price formatting

| Example | Display |
|---|---|
| price 3.5, unit kg | `$3.50 / kg` |
| price 2, unit each | `$2.00 each` |

Locale — your choice; consistent with vendor dashboard if prices shown there.

---

## Image fallback

When `!imageUrl`:

- Div with background color + "No photo" text, or  
- Generic produce icon  

Fixed height on image container — grid alignment.

---

## CSS notes

- `object-fit: cover` on img  
- `text-overflow: ellipsis` on long names  
- Hover state on card — subtle shadow or border
