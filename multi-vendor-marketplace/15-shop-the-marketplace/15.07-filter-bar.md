# 15.7 — Filter bar

**Why this step matters:** Filters teach the **refetch pattern** — UI state drives query params, API returns a new page of truth. Client-side filtering would break pagination and hide products on other pages.

**Where the project is now:** ProductCard exists; shop shows static or unfiltered list.

**After this step:** Category and store filters update API query params; shop refetches on change.

---

## Component spec

**File:** `client/src/components/CatalogueFilters.jsx` (or inline in Shop.jsx)

| Prop | Type | Purpose |
|---|---|---|
| `category` | string | Current value |
| `storeId` | string | Current value |
| `onCategoryChange` | fn | Parent updates state |
| `onStoreChange` | fn | Parent updates state |

## API param mapping

| UI value | Query param |
|---|---|
| category '' | omit `category` |
| category 'fruit' | `?category=fruit` |
| storeId '' | omit `storeId` |
| storeId set | `?storeId=...` |

Build params object — strip empty strings before fetch.
