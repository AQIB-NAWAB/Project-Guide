# 15.3 — What you'll build

By chapter end, customers open **Shop**, filter and paginate the catalogue, open **product detail**, and see items from multiple vendors — all driven by Chapter 14 API.

Blueprint for 15.4–15.10. Specs only.

---

## The customer journey this chapter completes

```
Home / nav  →  /shop (grid + filters)  →  /shop/:id (detail + stub button)
                     ↑                           ↑
              GET /products                 GET /products/:id
```

No new server routes. Every screen **consumes** Chapter 14. If the API returns wrong data, fix the API — do not patch around it in React with hardcoded arrays.

**Multi-vendor proof for gate:** Grid must show products from **at least two different `store.name` values** at once — that is what makes FreshMarket a marketplace, not a single-shop app.

## API client functions

### `getProducts(params)`

| Param | Maps to query |
|---|---|
| `page` | `page` |
| `limit` | `limit` |
| `category` | `category` |
| `storeId` | `storeId` |

**Returns:** `{ items, page, limit, total, totalPages }` — typed or JSDoc your choice.

### `getProduct(id)`

**Returns:** single product object — same shape as list item + description if present.

Errors: 404 → user message on detail page.

## Shop page behaviour spec

| Feature | Spec |
|---|---|
| Initial load | `GET /products?page=1&limit=20` |
| Filter change | Reset to page 1; refetch with params |
| Pagination | Page buttons or Prev/Next using `totalPages` |
| URL sync | **Stretch:** `?category=` in searchParams |
| Loading | Show loading state per fetch |

---

## Filter bar spec

| Control | Options source |
|---|---|
| Category | Hardcoded list matching seed OR derive from first full fetch (stretch) |
| Store | Fetch stores list **stretch** OR hardcode seed store ids + names for MVP |

**MVP acceptable:** categories from fixed array in constants file matching seed.

Changing filter updates query and refetches — no client-side filter of full catalogue (API paginates).
