# 15.5 — Scaffold customer pages

**Why this step matters:** Routes and API helpers are the **skeleton** of the shop — without `/shop` and `getProducts()` in place, every later component has nowhere to mount. One small scaffold step prevents file-path confusion for the rest of the chapter.

**Where the project is now:** No customer-facing product pages.

**After this step:** Routes `/shop` and `/shop/:productId` render placeholder pages; nav links to shop.

---

## Create files

| File | Purpose |
|---|---|
| `client/src/pages/Shop.jsx` | Shop grid container |
| `client/src/pages/ProductDetail.jsx` | Detail container |
| `client/src/api/products.js` | `getProducts`, `getProduct` stubs calling API |

---

## Route registration spec

In `App.jsx` (or routes module):

| Path | Element |
|---|---|
| `/shop` | `<Shop />` |
| `/shop/:productId` | `<ProductDetail />` |

**Not** nested under vendor layout — customer-facing top-level routes.

Optional: wrap in `CustomerLayout` later — not required for gate.
