## Demo script — all ten steps

Narrate at step 5: *"One checkout, two orders — each store fulfils separately."*

| Step | Actor | Action | Expected |
|---|---|---|---|
| 1 | Customer | Browse `/shop` | Products from **2+ stores** |
| 2 | Customer | Add items from Store A and B | Cart badge updates |
| 3 | Customer | Open `/cart` | **Two store groups** + subtotals |
| 4 | Customer | `/checkout` → Place order | `201`; redirect |
| 5 | Customer | `/orders/confirmation` | **Two orders**; snapshot lines |
| 6 | Customer | `/orders` | **Two cards** in history |
| 7 | Vendor A | `/vendor/orders` | **Only** A's order |
| 8 | Vendor B | `/vendor/orders` | **Only** B's order |
| 9 | Vendor A | Mark **packed** | Badge updates on vendor list |
| 10 | Customer | Refresh `/orders` | A = `packed`; B = `pending` |

Steps 7–8 are the **isolation headline** — separate windows, not role-switching in one tab.

## Common demo failures

| Failure | Likely cause | Fix |
|---|---|---|
| One order only | Single-store cart | Add B's products; Ch 18 split |
| Vendor sees all orders | Missing store scope | Ch 18.14 |
| Wrong history prices | Live product price in UI | Snapshot fields |
| Empty confirmation | Lost nav state | Pass `orders` in state (19.6) |
| Status not on customer side | No refresh | Step 10 requires refresh |
| Checkout 409 | Low stock | Reduce qty or re-seed |
