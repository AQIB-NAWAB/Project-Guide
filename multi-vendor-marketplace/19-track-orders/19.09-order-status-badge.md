# 19.9 — Order status badge

**Why this step matters:** One shared badge component keeps status labels consistent across confirmation, customer history, and vendor inbox — without it, the same order shows "Pending" in one place and "pending" in another.

**After this step:** Orders display **status** consistently — `pending` → `packed` → `dispatched` (or your enum).

---

## Component contract

`OrderStatusBadge.jsx` — prop `status: string`.

| Status | Visual |
|---|---|
| `pending` | Neutral — awaiting fulfilment |
| `packed` | In progress |
| `dispatched` | Complete / shipped |

Map unknown status to raw string — don't crash.

---

## Schema alignment

Use exact enum values from Order model (Ch 5). Document in README:

```
pending | packed | dispatched
```

Optional: `cancelled` — out of scope unless implemented.

---

## Where used

- Order confirmation
- My orders list
- Vendor orders list

Single component — one mapping table.

---

## Accessibility

Badge text readable — not color-only. `aria-label` includes status name.

**✅ Do:** Default new orders to `pending` from API.
