## Verify API first — curl matrix

Run with tokens from seed accounts (vendor A, vendor B, customer C):

```bash
# 1 — Customer checkout (cart must have items from two stores)
curl -s -X POST http://localhost:5000/api/checkout \
  -H "Authorization: Bearer <customer-token>" \
  -H "Content-Type: application/json" \
  -d '{}'

# 2 — Customer order history
curl -s http://localhost:5000/api/orders/my \
  -H "Authorization: Bearer <customer-token>"

# 3 — Vendor A fulfilment list
curl -s http://localhost:5000/api/vendor/orders \
  -H "Authorization: Bearer <vendor-a-token>"
```

| Test | Expected |
|---|---|
| Checkout | `201` + `{ orders: [...] }` — array length = distinct stores in cart |
| Customer list | Only C's orders; line prices match snapshot, not live product price |
| Vendor list | Only orders for vendor A's store; includes customer display fields |

Save one redacted JSON response per route in learning log — UI will mirror these shapes.

## Expected order JSON shape (UI contract)

Customer and vendor pages read **embedded** line items — no product lookup for price:

```json
{
  "_id": "...",
  "storeId": "...",
  "status": "pending",
  "total": 305,
  "orderItems": [
    {
      "nameAtPurchase": "Organic milk 1L",
      "priceAtPurchase": 120,
      "unitAtPurchase": "each",
      "quantity": 2
    }
  ],
  "createdAt": "..."
}
```

Vendor list may add `customerId` or populated customer name — match your API from Ch 18.
