# 19.5 — Checkout page

**Why this step matters:** This is the button that turns a cart into orders — the climax of Week 3. Without a review screen, customers cannot see what they are buying before committing, and you cannot demo commerce in a browser.

**Where the project is now:** Cart page groups items by store (Ch 17); `POST /checkout` works via curl (Ch 18). The checkout CTA may still be a stub — no review-and-submit screen yet.

**After this step:** Customer reviews cart at `/checkout`, places order via `POST /checkout`, then navigates to confirmation with order data in location state.

This is the **review gate** before money moves: same grouped layout as `/cart`, read-only lines, submit CTA instead of edit actions.

## Step 1 — Route and API helper

Register `/checkout` inside customer protected route group. Add `checkout()` → `POST /checkout` with Bearer via interceptor. Returns `{ orders: [...] }`. **❌ Don't** send prices or line items — server snapshots from cart.

---

## Step 2 — Page load contract

1. On mount, call `getCart()` — reuse `StoreCartGroup` / `CartItemRow` in **read-only mode**.
2. Loading → loading UI (not empty-cart message).
3. Empty cart → redirect to `/cart` or `/shop` with brief message.
4. Else render per-store sections + **grand total** (sum of store subtotals).

| Data | Display |
|---|---|
| Group by `storeId` | One section per store |
| `storeName` | Section heading |
| Line name, unit, price, qty | Read-only rows |
| Per-store subtotal | Sum of line totals |

---

## Step 3 — Submit contract

1. **Place order** triggers `checkout()`; set `isSubmitting` — disable until response settles.
2. Success `201` → navigate to `/orders/confirmation` with `{ orders: [...] }` in state.
3. Error → inline API message; stay on checkout.
4. On success, refresh cart badge so header count clears.

---

## POST /checkout — API contract

```
POST /api/checkout
Authorization: Bearer <customer-token>
Content-Type: application/json

{}
```

| Response | UI action |
|---|---|
| `201` + `{ orders: [...] }` | Navigate to confirmation with body |
| `400` empty cart | Redirect or message; link to shop |
| `400` / `409` stock | Inline error; highlight `productId` if returned |
| `401` | Redirect login |
| `5xx` | Message + retry |

Each order: `_id`, store name, `status` (`pending`), `total`, `orderItems[]` snapshots, `createdAt`.

/cart → [Proceed to checkout] → /checkout → [Place order] → /orders/confirmation

## Troubleshooting

| Observation | Likely cause | Action |
|---|---|---|
| 401 on submit | Token missing/expired | Check interceptor; re-login |
| 201 but one order | Single-store cart | Add second vendor's products |
| Grand total wrong | Wrong subtotal formula | Match cart page math |
| Confirmation empty | No state passed | Fix success handler |
| CORS error | URL mismatch | Align `VITE_API_URL` |
