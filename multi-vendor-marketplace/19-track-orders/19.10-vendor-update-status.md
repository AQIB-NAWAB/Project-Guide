# 19.10 — Vendor update status

**Why this step matters:** Static order lists are read-only theatre — vendors must advance fulfilment and customers must see progress. This step also rehearses ownership checks you will enqueue notifications for in Chapter 21.

**Where the project is now:** Vendor orders page is read-only (19.8). New orders show `pending` — no browser or API path to advance status unless PATCH exists from Ch 18.

**After this step:** Vendor advances status via `PATCH /vendor/orders/:orderId` — customer sees updated badge on refresh.

Status updates are **vendor-only, store-scoped writes**. Customer reads through history — no polling required for Week 3 gate.

## Step 1 — PATCH API contract

```
PATCH /api/vendor/orders/:orderId
Authorization: Bearer <vendor-token>
Content-Type: application/json

{ "status": "packed" }
```

### Server rules — spec

1. Load order — `404` if missing.
2. Resolve vendor store from `ownerId` — `storeId` must match; else `403`.
3. Validate enum + **transition** — else `400`.
4. Persist; return updated order.

| From | To |
|---|---|
| `pending` | `packed` |
| `packed` | `dispatched` |
| `dispatched` | *(terminal)* |

Reject backwards transitions unless documented. **❌ Don't** accept client `storeId` for auth. **❌ Don't** let customer role call this route.

| Code | Meaning |
|---|---|
| `200` | Updated order |
| `400` | Invalid status or transition |
| `403` | Another vendor's order |
| `404` | Order not found |

---

## Step 2 — UI contract

Per row, **one next action** by current status:

| Status | Control | Body |
|---|---|---|
| `pending` | "Mark packed" | `{ status: "packed" }` |
| `packed` | "Mark dispatched" | `{ status: "dispatched" }` |
| `dispatched` | — | No action |

1. Click → disable while PATCH in flight.
2. Success → update local state or refetch list.
3. `400`/`403` → show API message.

Customer `/orders` shows update on **refresh** — demo step 10 in 19.11.

## Troubleshooting

| Observation | Likely cause | Action |
|---|---|---|
| 403 on own order | Store ownership mismatch | Check `store.ownerId` |
| 400 invalid transition | Skipped step | Advance one state at a time |
| UI stale after success | No state update | Refetch or merge response |
| Customer never sees update | No refresh | Hard refresh `/orders` |
| Dispatched still has action | UI not keyed on status | Hide button when terminal |
