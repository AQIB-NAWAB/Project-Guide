# 9.5 — Prime your thinking

Before scaffolding routes, read how other APIs expose **"my resource"** vs **"resource by id"** — your `GET /stores/me` and `PATCH /stores/:id` design benefits from five minutes of context.

---

## Mandatory topics (read before 9.8)

| Topic | Why it matters here |
|---|---|
| **Slug vs MongoDB `_id` in URLs** | PATCH uses `:id` — know ObjectId format and validation |
| **REST nested vs flat routes** | `/vendors/me/store` vs `/stores/me` — you chose flat |
| **409 Conflict semantics** | Duplicate store create |
| **BOLA/IDOR on PATCH** | Chapter 8 — ownership on `:id` param |

## Suggested reads

Pick **at least two** — skim for decisions, not memorization.

1. **MDN HTTP response codes** — 201, 404, 409, 403 sections  
   *https://developer.mozilla.org/en-US/docs/Web/HTTP/Status*

2. **REST "me" or "current user" resource patterns** — how GitHub, Stripe expose `/me` endpoints (conceptual search: "REST API current user resource")

3. **MongoDB ObjectId** — valid 24-hex string; invalid id → 404 not 500  
   *Mongoose docs: CastError handling*

4. **OWASP BOLA** — refresh from Chapter 8 before PATCH goes live  
   *https://owasp.org/API-Security/editions/2023/en/0xa1-broken-object-level-authorization/*

5. **Express Router param routes** — order matters: `/me` before `/:id` if you add `GET /:id` later

## Slug vs id — decision recap

| URL style | Example | Chapter 9 |
|---|---|---|
| ObjectId | `PATCH /api/stores/664a1b2c3d4e5f6789012345` | ✓ primary |
| Slug | `PATCH /api/stores/mayas-market` | optional future |
| Me alias | `GET /api/stores/me` | ✓ required |

**Route ordering warning:** If you later add `GET /stores/:id`, register **`GET /stores/me` first** — otherwise Express treats `"me"` as an id parameter.

---

## `/stores/me` vs query param

Some APIs use `GET /stores?vendorId=...` — **reject for authenticated vendor context:**

| Approach | Problem |
|---|---|
| `?vendorId=` from client | Client can tamper — must ignore and use JWT |
| `GET /stores/me` | Token defines scope — no tamper |

---

## Idempotency note (stretch)

`POST /stores` is **not** idempotent — calling twice creates conflict second time. Optional header `Idempotency-Key` is production polish — not required for gate.

Document in learning log if you researched it — shows senior awareness.
