# 9.11 тАФ Update store route

## Why this step matters

Store settings тАФ name, description, address тАФ change over time. `PATCH /stores/:id` reuses Chapter 8's ownership assert so only the leaseholder edits their stall. Chapter 10's settings page calls this route on save.

---

**Where the project is now:** Vendor can create and fetch own store.

**After this step:** `PATCH /api/stores/:id` updates mutable fields when caller owns the store тАФ **200** + updated document. Wrong vendor тЖТ **403**. Invalid id or missing store тЖТ **404**.

---

## Handler flow (spec)

```
PATCH /api/stores/:id
  1. requireAuth
  2. requireRole('vendor')
  3. validateObjectId(id) тЖТ 404 if invalid format
  4. validateUpdateStore(req.body) тЖТ 400 if invalid
  5. assertStoreOwner(id, userId) тЖТ 403 if not owner
  6. updateStore(id, userId, validBody)
  7. 200 + updated store JSON
```

Reuse **`assertStoreOwner`** from Chapter 8 тАФ do not rewrite ownership query inline.

---

## Service: `updateStore(storeId, vendorId, data)`

| Step | Action |
|---|---|
| 1 | Find by `_id` тАФ if missing, NotFoundError (404) |
| 2 | Ownership already checked in middleware OR re-verify `store.vendorId` equals vendorId |
| 3 | Apply allowed fields only тАФ `$set` partial update |
| 4 | Return updated document |

**Immutable:** `vendorId`, `_id` тАФ strip from PATCH body in validation.

---

## Mutable fields

Per 9.4 тАФ typically:

- `name`  
- `description`  
- `address`  
- `logoUrl`  

Empty PATCH body `{}` тЖТ 400 or 200 no-op тАФ pick one; recommend **400 "no fields to update"** for clarity.

---

## Error matrix

| Condition | Status |
|---|---|
| Invalid ObjectId format | 404 |
| Store not found | 404 |
| Store owned by another vendor | 403 |
| Invalid field values | 400 |
| Owner, valid patch | 200 |

**403 vs 404:** Some APIs return 404 for non-owner to hide existence тАФ this course uses **403** per Chapter 8 IDOR test convention. Stay consistent with `AUTHORIZATION.md`.

## Partial update semantics

PATCH is **partial** тАФ omitting `name` leaves existing name unchanged. Do not require full resource body like PUT.
