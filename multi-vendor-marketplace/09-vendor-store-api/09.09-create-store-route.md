# 9.9 — Create store route

## Why this step matters

Opening a shop is the vendor's first irreversible action on the platform. `POST /stores` must enforce one-store-per-vendor, bind `vendorId` from the JWT (never the body), and reject customers. Chapter 10's onboarding form calls this endpoint.

---

**Where the project is now:** Stores module scaffolded; `POST /` may return stub or validation-only.

**After this step:** Authenticated vendor can `POST /api/stores` with valid body → **201** + store document. Second create → **409**. Customer or anonymous → **401/403**.

---

## Handler flow (spec)

```
POST /api/stores
  1. requireAuth → userId
  2. requireRole('vendor')
  3. validateCreateStore(req.body) → 400 if invalid
  4. createStore(userId, validBody)
  5. 201 + JSON store
```

`userId` from JWT — never `req.body.vendorId`.

---

## Service: `createStore(vendorId, data)`

| Step | Action |
|---|---|
| 1 | Query Store by `vendorId` |
| 2 | If exists → throw ConflictError (409) |
| 3 | Create document with `vendorId` + validated fields |
| 4 | Return saved document |

Catch MongoDB duplicate key error (E11000) as backup → 409 if race between two POSTs.

---

## Request body contract

Minimum fields per 9.4 — example:

| Field | Rule |
|---|---|
| `name` | required, trimmed, max length |
| `description` | optional |
| `address` | per your ERD |
| `logoUrl` | optional URL |

Reject empty `name` after trim → 400.

---

## Response 201

JSON body: full store object including `_id`, `vendorId`, `createdAt`.

`Location` header optional: `/api/stores/{id}` — not required for gate.

---

## Error matrix

| Condition | Status |
|---|---|
| No token | 401 |
| Customer role | 403 |
| Invalid body | 400 |
| Vendor already has store | 409 |
| Valid first create | 201 |

## ✅ Do / ❌ Don't

**✅ Do:** Set `vendorId` in service at insert only.

**✅ Do:** Return 409 with clear message for duplicate.

**❌ Don't:** Allow customer role to create store — even if they send vendor role in body (body ignored).

**❌ Don't:** Return 500 on duplicate key — map to 409.
