# 1.6 — Course outline

This is your map for four weeks. Every chapter below is a **folder** of sub-chapters you complete **in order**. Each row tells you what milestone you reach when the checklist at the end of that chapter is fully ticked.

Do not treat this as a table of contents to skim once. Return here when you finish a week and ask: **"Can I demo what this week promised?"**

---

## How to read this outline

| Column | Meaning |
|---|---|
| **#** | Chapter number — matches folder prefix (`06-authentication-api/`) |
| **Chapter** | Feature area name — what you'll say you built |
| **Week** | Phase grouping — demo target at week end |
| **Done when…** | One-line gate summary — full checklist lives in the chapter |
| **Pair** | Backend chapter → frontend chapter that follows immediately |

**Sub-chapter count:** Most feature chapters have **20–25 sub-chapters**. Introduction and concept chapters run shorter. See `COURSE-OUTLINE.md` in the course root for the full file-by-file breakdown of every sub-chapter.

### The commerce arc — one continuous build

Week 3 is where the **marketplace identity** of the product appears. Chapters 14–19 should feel like one story:

```
Catalogue (see products)
  → Cart (hold choices)
    → Checkout (split + snapshot)
      → Orders (both sides see truth)
```

**End of Week 3 question:** Cart with items from two stores → checkout → two orders in DB → change a price → old order unchanged in UI? If yes, Week 4.

## Complexity ladder — what gets harder when

The course climbs deliberately (Rule 20). You don't meet Redis in Week 1 or deployment in Week 2.

| Rung | Topics | Chapters |
|---|---|---|
| **Stand project on disk** | Repo shape, client/server, first HTTP | 02 |
| **Config & persistence** | Env, MongoDB, models, seed | 03–05 |
| **Identity** | Hash, JWT, refresh, OTP | 06–07 |
| **Authorization** | Role, ownership, IDOR | 08 |
| **Vendor CRUD** | Store, products, images, UI | 09–13 |
| **Catalogue performance** | Pagination, populate, N+1 | 14–15 |
| **Commerce** | Cart, split checkout, snapshots | 16–19 |
| **Scale & ship** | Cache, search, async jobs, HTTPS | 20–22 |

Skipping a rung to jump ahead produces chapters that reference files that don't exist. **Work in order.**

## Sub-chapter arc — what every feature chapter repeats

When you open Chapter 06 (or any feature chapter), expect this rhythm:

1. **02.01–03** — Set scene, why it matters, goals  
2. **02.04–07** — Learn approaches, decide, deep-dive, reads  
3. **02.08** — Quiz placeholder (skip)  
4. **02.09–16** — Build: recap → scaffold → implement steps → verify  
5. **02.17–19** — Practice (optional), recap, **checklist gate**

The build cluster is the **longest** block. If a chapter feels like reading without typing, you're in the wrong file — jump forward to the "where the project is now" sub-chapter or check you didn't skip a build step.

## Competencies — what you'll prove you can do

By Chapter 22, you should be able to **build and explain** each of these (tested in viva):

1. Model vendors, stores, and products with correct ownership  
2. Hash passwords; issue role-aware JWTs; explain the threat model  
3. Enforce vendor data isolation — demonstrate IDOR failure  
4. Build multi-vendor cart and split checkout with price snapshots  
5. Avoid N+1 on catalogue; explain populate vs looping queries  
6. Store images in object storage, not MongoDB documents  
7. Cache catalogue reads; invalidate when products change  
8. Enqueue async jobs after checkout; separate worker from API  
9. Deploy with secrets in environment variables, not Git  

Each maps to specific chapters — see `COURSE-OUTLINE.md` competency table.
