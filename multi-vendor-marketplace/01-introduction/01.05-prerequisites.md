# 1.5 — Before you start

This table is the **floor** the course stands on — not a gauntlet, not a reading list of everything you'll learn anyway. It's the few things you should already be comfortable with so Chapter 2 doesn't stop on hour one to explain what `async/await` means.

**Two rules govern this list:**

1. **Pitch it to the level.** This is a **medium** course — it assumes you can already build CRUD. It does not assume you've built a multi-vendor system, deployed with Redis, or enforced row-level ownership.
2. **Never list what the course teaches.** If an early chapter covers it, it's not a prerequisite — it's the climb.

If a row says **must have**, you're expected to start with it. **Revise if rusty** means skim for an hour if you haven't used it in months — don't take a three-day detour. **Helpful, taught anyway** means don't panic if it's new; the course will touch it when needed.

---

## If you feel closer to beginner than "medium"

This course is labeled **medium** because it assumes you can already build CRUD. That does **not** mean you must know JWT, Redis, or multi-vendor architecture on day one — those are taught here.

**You are in a good place if you can:**
- Write a simple Express route that returns JSON
- Build a React component that loads data with `useEffect`
- Use Git to commit without putting secrets in the repo

**Take a short detour first if:**
- JavaScript `async/await` is completely unfamiliar
- You have never run a server and a React app at the same time
- HTTP status codes (200, 404, 401) mean nothing to you yet

The course is written so **every chapter** includes **Bridge from last chapter**, **In plain English** jargon tables, **Think of it like…** analogies, and **Common beginner questions** — use them whenever a sub-chapter feels dense. Chapter 1.2 explains the reading order in detail.

## Knowledge self-check — honest questions

Before Chapter 2, you should answer **yes** without googling the syntax:

1. Can you create an Express route that returns JSON and a 404 for a missing path?
2. Can you write a React component that fetches data in `useEffect` and displays it?
3. Can you explain the difference between `401 Unauthorized` and `403 Forbidden`?
4. Can you make a Git commit and confirm a file is listed in `.gitignore`?

If **1 or 2** is no, pause this course and build one small CRUD app (todo or notes) first. Medium assumes that baseline.

If **3** is no, search "HTTP 401 vs 403" — ten minutes — then continue.

If **4** is no, search "gitignore tutorial" — fifteen minutes — then continue. Secrets in Git is a one-way mistake.

## Recommended refresh — if rusty

If it's been months since you touched MongoDB or Express, one afternoon of refresh beats a week of friction:

**Express (1–2 hours)** — official "Hello world" + one CRUD route with in-memory array. Remind yourself: `app.get`, `app.post`, `req.body`, middleware.

**React (1–2 hours)** — Vite default template + fetch from a public API (jsonplaceholder.typicode.com). Remind yourself: state, effect, map over array.

**MongoDB (1 hour)** — MongoDB University free "MongoDB Basics" first module **or** create one collection in Atlas, insert three documents in Compass, run one find query.

Stop when comfortable — not when certified.

## Accessibility & workflow notes

**OS:** Linux, macOS, and Windows all work. Commands in the course use bash-style examples; translate `export FOO=bar` to Windows `set` or PowerShell as needed.

**Monorepo:** One Git repo with `server/` and `client/`. One terminal tab each when developing.

**AI tools:** Allowed. If you use Cursor or Copilot, still **read** what you commit — the viva is on you.

**Time:** ~60 hours/week × 4 weeks is the intended pace. Scope is by **chapters finished**, not hours logged. Slow is fine; skipping gates is not.

---

## If you're under-prepared — two paths

**Path A — Shorter gap (1–3 days):** Build a minimal **single-user todo API + React list** — Express, MongoDB or in-memory, five routes, one page. Then start Chapter 2.

**Path B — Larger gap (1–2 weeks):** Complete one structured Node + React CRUD tutorial end-to-end, then return here. A medium marketplace on weak CRUD foundations frustrates more than it teaches.

---

## If you're over-prepared — one warning

Senior developers sometimes skip chapters because "they know auth." **Don't skip the gates.** This course's auth chapter isn't only login — it's OTP, refresh token hashing, role in JWT, and the isolation tests that follow. Skim the learn-it sections if bored, but **complete the build steps and tick the checklist.** The viva asks about *this* project's choices, not auth in general.
