# 16.14 — Recap and what's next

**Why this step matters:** Chapter 17 wires React to these endpoints — if you cannot explain POST body fields and user scoping, the UI will send wrong payloads or skip `storeId`.

You entered Chapter 16 with a browse-only marketplace. You leave with a **persisted, user-scoped cart API** — multi-vendor lines with `storeId`, stock checks, full CRUD — ready for the cart UI in Chapter 17.

---

## What you built

**POST /cart:** Validates product, storeId match, stock; upserts line with userId from JWT.

**GET /cart:** Populated products and stores — no cross-user leakage.

**PATCH /cart/:itemId:** Quantity updates with ownership + stock rules.

**DELETE /cart/:itemId:** Remove own lines only.

**Isolation:** User B cannot access User A's itemIds — verified in 16.13.

**Multi-vendor:** One cart holds lines from multiple storeIds — checkout split enabled for Ch 18.

## Decisions for viva

1. **Why storeId on line?** — Checkout grouping without re-query.  
2. **Why validate storeId on POST?** — Client tamper prevention.  
3. **Why 404 on wrong user item?** — IDOR prevention.  
4. **When stock decrements?** — Checkout Ch 18, not cart add.  
5. **Guest cart?** — Out of MVP; login required.
