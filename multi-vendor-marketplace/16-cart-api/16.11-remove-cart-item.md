# 16.11 — Remove cart item

**Why this step matters:** DELETE completes CRUD — customers must remove mistakes before checkout. Same ownership rule as PATCH: only the cart owner can delete their line.

**Where the project is now:** Cart lines can be added and quantity updated; no delete.

**After this step:** `DELETE /cart/:itemId` removes one line from current user's cart.

---

## DELETE /cart/:itemId contract

| Field | Value |
|---|---|
| Auth | Required |
| Param | itemId — CartItem id |
| Success | 200 with message or 204 No Content |

---

## Handler sequence

1. `findOneAndDelete({ _id: itemId, userId: currentUser })`  
2. If null → 404  
3. Else → 204 or 200 `{ message: "Removed" }`  

Same user scoping as PATCH — never delete by `_id` alone.

---

## Response options

| Style | When |
|---|---|
| 204 No Content | Minimal REST |
| 200 + remaining cart | UX friendly for SPA — optional |

Pick one; document in README. Chapter 17 UI may refetch GET /cart after delete either way.
