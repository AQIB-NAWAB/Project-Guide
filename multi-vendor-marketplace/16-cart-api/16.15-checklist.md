---

## Definition of Done

### Module & auth

- [ ] `cart.routes.js`, `cart.service.js`, `cart.schema.js` exist
- [ ] Router mounted on app with API prefix consistent with project
- [ ] **All** cart routes require authentication — 401 without token
- [ ] userId taken from JWT only — never from request body

### POST /cart

- [ ] Accepts `productId`, `storeId`, `quantity`
- [ ] Validates product exists and is published / in stock
- [ ] Validates `product.storeId` matches request `storeId` — mismatch → 400
- [ ] Rejects quantity > stockQty and stockQty === 0
- [ ] Creates CartItem with userId, productId, storeId, quantity

### GET /cart

- [ ] Returns only current user's items
- [ ] Populates product (name, price, unit, imageUrl) and store (name)
- [ ] Empty cart → 200 with `items: []`
- [ ] No N+1 populate loop

### PATCH /cart/:itemId

- [ ] Updates quantity for own item only
- [ ] Other user's itemId → 404 or 403
- [ ] Stock re-validated on increase
- [ ] quantity < 1 rejected — use DELETE

### DELETE /cart/:itemId

- [ ] Removes own item only
- [ ] Cross-user delete fails
- [ ] Item removed from subsequent GET

### Multi-vendor & isolation (16.13)

- [ ] One customer cart holds items from **≥2 storeIds** (tested)
- [ ] User A cannot GET User B's cart contents via itemId manipulation
- [ ] User B PATCH/DELETE on A's itemId fails
- [ ] Compass shows correct userId on all cart documents

### Stock

- [ ] Shared stock validation on POST and PATCH
- [ ] Stock **not** decremented on add to cart

### Docs

- [ ] README documents cart endpoints with curl examples
- [ ] Login required for cart noted

### Prior gates

- [ ] Chapter 15 shop UI complete
- [ ] Chapter 14 catalogue API complete
- [ ] Auth API complete
