# 16.10 — Update cart item

**Why this step matters:** Quantity changes are the most common cart mutation — PATCH must stay **user-scoped** and re-check stock so customers cannot PATCH another user's itemId or order more than exists.

**Where the project is now:** GET and POST work; quantity fixed after add unless re-POST.

**After this step:** `PATCH /cart/:itemId` updates quantity for **own** line only — stock validated.

---

## PATCH /cart/:itemId contract

| Field | Value |
|---|---|
| Auth | Required |
| Body | `{ "quantity": number }` |
| Param | `itemId` — CartItem `_id` |

---

## Handler sequence

1. Validate quantity integer ≥ 1  
2. Find CartItem `{ _id: itemId, userId: currentUser }` — 404 if not found (includes other user's item)  
3. Load product for stock check  
4. If quantity > product.stockQty → 400  
5. Update quantity; save  
6. Return updated item or full cart  

**Quantity 0:** reject with 400 — use DELETE to remove.

---

## User scoping — critical

Query must include **both** `_id` and `userId`:

```javascript
// findOneAndUpdate(
//   { _id: itemId, userId },
//   { quantity },
//   { new: true }
// )
```

If not found → 404 — do not reveal whether item exists for another user.

---

## Response

**200** — updated line DTO or full cart — match POST/GET convention.

---

## curl tests

```bash
# Update own item
curl -X PATCH http://localhost:5000/api/cart/ITEM_ID \
  -H "Authorization: Bearer CUSTOMER_A_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"quantity":3}'

# Customer B tries A's item id
curl -i -X PATCH http://localhost:5000/api/cart/A_ITEM_ID \
  -H "Authorization: Bearer CUSTOMER_B_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"quantity":1}'
```

Second expects 404 or 403 — not success.

## ✅ Do / ❌ Don't

**✅ Do:** Re-validate stock on every PATCH increase.

**✅ Do:** Return 404 for cross-user itemId — same as not found.

**❌ Don't:** Allow PATCH to change productId or storeId — delete + re-add instead.

**❌ Don't:** Accept quantity as string without parsing.
