# 16.8 — Add to cart route

**Why this step matters:** POST /cart is where **`storeId` enters the system** — validated against the product so multi-vendor grouping works at checkout. Getting add wrong breaks every later chapter.

**Where the project is now:** Cart router mounted; POST handler empty or stub.

**After this step:** `POST /cart` creates or increments line with productId, storeId, quantity — validated against product.

---

## POST /cart handler sequence

1. Validate body (schema)  
2. `userId = req.user.id`  
3. Load product by productId — 404 if missing  
4. Check `product.isPublished && product.stockQty > 0` — else 400/404  
5. Assert `product.storeId.toString() === body.storeId` — else 400 store mismatch  
6. Check `quantity <= product.stockQty` — else 400 insufficient stock  
7. Upsert cart line:
   - **Option A:** existing line same user+product → increment quantity (cap at stock)  
   - **Option B:** always new line — document if chosen  
8. Return item or cart JSON  

**Recommended:** upsert same user+product — one line per product per user.

## Response

**201 Created** new line or **200 OK** updated line:

```json
{
  "id": "...",
  "productId": "...",
  "storeId": "...",
  "quantity": 2
}
```

Or return full cart — stay consistent with GET shape from 16.9.
