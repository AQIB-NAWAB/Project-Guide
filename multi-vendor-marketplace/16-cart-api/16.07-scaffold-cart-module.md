# 16.7 — Scaffold cart module

**Why this step matters:** A dedicated `cart/` module with auth on the router sets the **security boundary** once — every cart route inherits `requireAuth` before you write handlers that touch user data.

**Where the project is now:** CartItem model exists; no HTTP cart API.

**After this step:** Module files created; router mounted on app with auth; handlers stubbed or empty.

---

## Create files

| File | Purpose |
|---|---|
| `cart.routes.js` | Express router — route definitions |
| `cart.service.js` | addItem, getCart, updateItem, removeItem |
| `cart.schema.js` | Zod/Joi validation for POST/PATCH bodies |

Model file may already exist as `cart.model.js` — do not duplicate.

---

## Router mount spec

In `app.js`:

```javascript
// import cartRoutes from './modules/cart/cart.routes.js'
// app.use('/api/cart', requireAuth, cartRoutes)
// or requireAuth per route inside router
```

**All cart endpoints require authentication.**

Role: `customer` primarily — allow `vendor` JWT too if vendors can shop (recommended).

## Route skeleton

| Method | Path | Handler |
|---|---|---|
| POST | `/` | addToCart |
| GET | `/` | getCart |
| PATCH | `/:itemId` | updateCartItem |
| DELETE | `/:itemId` | removeCartItem |

Export router default.

## Validation schema — POST

| Field | Rules |
|---|---|
| productId | valid ObjectId string |
| storeId | valid ObjectId string |
| quantity | integer ≥ 1, default 1 |
