# 22.10 — Deploy API

**Why this step matters:** The API is the brain — auth, checkout, cache, and job enqueue all live here. Until `/health` returns 200 on HTTPS with `db: connected`, nothing else in production can work.

**After this step:** Express API reachable at **HTTPS** production URL; `/health` returns 200.

---

## Deploy contract — you implement per host

| Step | Spec |
|---|---|
| Push code | Git deploy or image build |
| Install deps | `npm ci --production` in `server/` |
| Set env | All keys from 22.8 |
| Start command | `node src/server.js` or `npm start` |
| Process manager | PM2, systemd, or PaaS native restart |

**Listen on `process.env.PORT`** — host sets dynamically on PaaS.

---

## Post-deploy curl

```bash
curl -s https://api.yourdomain.com/health
```

**Expected:** `{ "status": "ok", "db": "connected" }`

---

## MongoDB Atlas

- Production cluster or dedicated DB user
- Network access allows deploy IP
- Connection string in host env only

## Logs

Tail host logs on first deploy — fix `MONGODB_URI` typos, missing JWT secret, CORS not blocking health (health usually unauthenticated).
