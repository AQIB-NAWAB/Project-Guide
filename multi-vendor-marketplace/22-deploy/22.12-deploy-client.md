# 22.12 — Deploy client

**Why this step matters:** Mentors evaluate the **shop URL**, not your git repo. This step puts the React build where browsers can load it — with SPA fallback so `/shop` refresh does not 404.

**After this step:** Production client URL loads React app and fetches production API.

---

## Deploy contract

| Host type | Action |
|---|---|
| Vercel / Netlify | Connect repo; root `client/`; build `npm run build`; output `dist` |
| Same VPS | Copy `dist/` to `/var/www/app`; Nginx static root |

Set build env `VITE_API_URL` in host UI — same as 22.9.

---

## SPA fallback

Configure redirects:

- Netlify: `_redirects` → `/* /index.html 200`
- Vercel: `vercel.json` rewrites

Without this, `/shop` refresh returns 404.

---

## Smoke in browser

1. Open production client URL.
2. Home or shop loads.
3. Network tab — API calls go to **production** API domain.
4. No mixed content errors.

---

## ✅ Do / ❌ Don't

**✅ Do:** Test deep link `/shop` refresh.

**❌ Don't:** Deploy stale `dist/` from wrong API URL.
