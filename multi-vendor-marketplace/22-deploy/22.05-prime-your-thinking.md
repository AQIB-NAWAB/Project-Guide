# 22.5 — Prime your thinking

**Mandatory read:** Your chosen host's deploy docs + [Let's Encrypt](https://letsencrypt.org/docs/) basics if self-hosting TLS.

**Time box:** 60 minutes.

Deploy fails when you skim platform docs. This sub-chapter forces **host-specific literacy** before you paste env vars in 22.8.

---

## Topics

| Topic | Why for FreshMarket |
|---|---|
| Process managers | PM2, systemd — API and worker survive SSH disconnect |
| Build vs start | `npm run build` client once; `npm start` API; `npm run worker` worker |
| Environment injection | PaaS env panel vs `.env` file on VPS |
| Reverse proxy | Nginx/Caddy terminates TLS → forwards to `localhost:5000` |
| Second service / process | Worker is not optional — where host docs describe multiple processes |

---

## Host-specific questions to answer

While reading **your** platform's docs, find answers for:

1. How does your host set `PORT`? (PaaS injects it — API must listen on `process.env.PORT`)
2. Where do you run `npm run build` for client — locally or CI?
3. What happens on process crash — auto restart?
4. How do you deploy a **second** service for the worker on your PaaS?
5. Where do you set build-time `VITE_API_URL` for Vercel/Netlify?

Write answers in learning log draft — you will need them in 22.8–22.12.

## Reflect questions

1. What is the difference between **development** (`npm run dev`) and **production** (`npm start`) for the API?
2. Why must `VITE_API_URL` be set **before** `npm run build`, not after?
3. What three processes ran locally at end of Chapter 21?

If #3 is not "API, worker, client" — re-read 22.01 bridge.

## In plain English

| Term | Simple meaning |
|---|---|
| **Reverse proxy** | Public-facing server (nginx) that receives HTTPS and forwards to your Node app on an internal port |
| **Build** | Compile React to static files — happens once per deploy |
| **Start command** | What the host runs to keep API alive — `node src/server.js` or `npm start` |

---

## Common beginner mistakes

1. **Deploying without production build** — uploading `client/src` instead of `client/dist`.
2. **API listens on hardcoded 5000** — PaaS assigns `PORT` dynamically.
3. **Worker not deployed** — checkout 201 works, no notifications.
4. **CORS unchanged** — API allows localhost only.
5. **Atlas IP not whitelisted** — health shows `db: disconnected` on production only.
