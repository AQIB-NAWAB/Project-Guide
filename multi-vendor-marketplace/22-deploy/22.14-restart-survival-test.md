# 22.14 — Restart survival test

**Why this step matters:** Apps that die on reboot are hobbies, not deployments. Proving API and worker recover within 60 seconds shows you configured process managers — not just SSH sessions.

**Where the project is now:** Smoke tests passed (22.13) — API, client, worker on HTTPS; worker processed checkout.

**After this step:** App recovers after restart/kill/redeploy — API health within 60s **and** worker resumes jobs.

---

Two server processes in production: API (HTTP, enqueue) and worker (BullMQ, Ch 21). API up + worker dead = 22.13 row 6 failure. Client static survives reboot — test API + worker.

**Gate:** Restart **both** → `/health` OK → checkout → worker log within ~30s.

---

## Test matrix — you implement per host

| Action | API | Worker |
|---|---|---|
| PaaS redeploy API | Health 200 ≤60s | Unaffected if separate service |
| PaaS redeploy worker | Unaffected | Startup log; jobs resume |
| `pm2 restart <api>` | Health 200 ≤60s | Running unless restarted too |
| `pm2 restart <worker>` | Unaffected | Startup log; consumers reattach |
| VPS reboot | Auto-start if configured | Auto-start if configured |
| Kill worker PID | Health still 200 | Logs resume after restart |

API-only restart ≠ gate. Worker restart row mandatory.

---

## Verification sequence — spec

1. Baseline: `curl /health` → 200, `db: connected`  
2. Restart API — poll every 5s, 200 within 60s  
3. Quick auth in browser  
4. Restart worker — startup log with queue name (Ch 21)  
5. Checkout on test account  
6. Worker log within ~30s  

Fail step 2: crash loop / missing env. Fail step 6: worker not consuming. No Redis flush needed for pass.

---

## Auto-start — VPS contract

| Component | Spec |
|---|---|
| API | `pm2 save` or systemd `enable` + `Restart=on-failure` |
| Worker | Second PM2 app or systemd unit — same env file |
| PaaS | Worker as declared service — not SSH one-off |

---

## Platform notes

| Platform | API restart | Worker restart |
|---|---|---|
| Render / Railway | Restart web service | Restart worker service |
| PM2 | `pm2 restart <api>` | `pm2 restart <worker>` — both in save |
| systemd | `restart` + `enable` | Second unit, both enabled |

Free tier may cold-start >60s — retry once. Atlas/Redis survive independently — test your processes only.

---

## Failure modes

| Symptom | Likely cause |
|---|---|
| Health never returns | Crash loop; check API logs |
| No worker after reboot | Not in `pm2 save` / not enabled |
| 201, no log post-restart | Wrong `REDIS_URL` after redeploy |
| Only API restarted | Incomplete gate |
