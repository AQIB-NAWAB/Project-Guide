# 22.11 — Domain and HTTPS

**Why this step matters:** HTTP-only demos fail the gate and break secure cookies from Chapter 7. HTTPS is what makes your URL pasteable in a portfolio without mentors seeing browser security warnings.

**After this step:** Custom domain or platform subdomain serves API and client over **TLS** — padlock in browser.

---

## DNS contract

| Record | Points to |
|---|---|
| `api.yourdomain.com` | API host load balancer / VPS IP |
| `app.yourdomain.com` or `www` | Client static host |

PaaS often provides `*.onrender.com` — custom domain optional but professional.

---

## TLS sources

| Setup | TLS |
|---|---|
| Vercel / Netlify / Render | Automatic |
| VPS + Caddy | Auto Let's Encrypt |
| VPS + Nginx + certbot | Manual renew cron |

**Gate:** Browser shows valid HTTPS — not self-signed unless you document exception (mentor may reject).

---

## Certificate renewal

Note auto-renew on PaaS vs certbot timer on VPS — one sentence in README.

---

## API behind proxy

If Nginx terminates TLS:

```
proxy_pass http://127.0.0.1:5000;
proxy_set_header X-Forwarded-Proto https;
```

Trust proxy in Express if needed for secure cookies: `app.set('trust proxy', 1)`.

---

## ✅ Do / ❌ Don't

**✅ Do:** Wait for DNS propagation — up to 48h, usually minutes.

**❌ Don't:** Ship HTTP-only production URL for gate.
