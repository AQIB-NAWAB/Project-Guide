# 22.9 ‚Äî Build client for production

**Why this step matters:** A live client that still calls `localhost:5000` is the most common deploy failure ‚Äî Vite bakes the API URL at build time, so this step locks the browser to your production API forever (until you rebuild).

**Where the project is now:** Production env checklist complete (22.8); production API URL decided; client still runs on Vite dev with local `VITE_API_URL`.

**After this step:** `client/dist/` built with **production** `VITE_API_URL` baked in; bundle grep shows no localhost; artifact ready for 22.12.

---

Vite replaces `import.meta.env.VITE_API_URL` at **compile time** ‚Äî frozen in minified JS. Host env changes after upload do nothing. URL change ‚Üí rebuild and redeploy. Worker never reads `VITE_*` ‚Äî worker proof is 22.13 logs.

---

## Build contract ‚Äî spec

| Input | Requirement |
|---|---|
| `VITE_API_URL` | Full HTTPS origin ‚Äî `https://api.yourdomain.com` |
| Scheme | `https://` when client is HTTPS |
| Trailing slash | Match `api.js` convention |
| API live? | Build OK before API deploy if URL is final |

```bash
cd client
VITE_API_URL=https://api.yourdomain.com npm run build
```

Exit code `0`. `api.js` must use `import.meta.env.VITE_API_URL` ‚Äî not hardcoded localhost.

---

## Verify API URL in bundle ‚Äî mandatory grep

```bash
grep -r "yourdomain" dist/assets/*.js | head
```

| Result | Meaning |
|---|---|
| Production HTTPS URL found | Pass |
| `localhost:5000` found | Fail ‚Äî rebuild |
| No matches | Search API subdomain ‚Äî URL may split across chunks |

Optional: `npm run preview` ‚Äî Network tab shows production API host.

---

## SPA routing ‚Äî static host requirement

`BrowserRouter` needs fallback to `index.html` on unknown paths.

| Host | Config ‚Äî you implement |
|---|---|
| Netlify | `_redirects` or `netlify.toml` |
| Vercel | `vercel.json` rewrites |
| Nginx on VPS | `try_files $uri /index.html` |

Without fallback, `/shop` refresh ‚Üí 404. Include config in 22.12 artifact.

## Platform build env

| Platform | Where `VITE_API_URL` lives |
|---|---|
| Vercel / Netlify | Env vars ‚Üí **Production** build scope |
| GitHub Actions | Secret or `env:` before `npm run build` |
| Local upload | Prefix on shell command above |

`.gitignore` `dist/` by default ‚Äî never hand-edit minified JS. CORS errors ‚Üí fix server `CLIENT_ORIGIN` (22.8), not rebuild alone.

---

## Failure modes

| Symptom | Likely cause |
|---|---|
| Live client calls localhost | Built without production `VITE_API_URL` |
| CORS on live client | `CLIENT_ORIGIN` not updated on API |
| Mixed content blocked | `VITE_API_URL` uses `http://` |
| `/shop` refresh 404 | Missing SPA fallback |
| Old URL after fix | Stale `dist/` without rebuild |
