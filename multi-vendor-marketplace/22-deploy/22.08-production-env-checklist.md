# 22.8 ‚Äî Production env checklist

**Why this step matters:** Missing `REDIS_URL` on the worker is the #1 silent production failure ‚Äî checkout returns 201 but notifications never fire. Completing this checklist before deploy prevents hunting blank env vars at 2am.

**Where the project is now:** API, worker, and client run locally with `.env` files; deploy platform chosen in 22.4; no production host configured yet.

**After this step:** Every required variable documented in `.env.example`; production host env configured for **both API and worker**; README checklist matches code.

---

Deploy fails silently when one secret is blank ‚Äî auth 500s in prod, or checkout succeeds but notifications never fire because the worker has no `REDIS_URL`. **Contract:** every `process.env` read in `config.js` appears in `server/.env.example` **and** in the host panel for **each** server process.

**Audit:** Grep `server/src` for `process.env` (should only be in `config.js`). Diff `config.js` exports against `server/.env.example` ‚Äî zero gaps. Document key list in README ‚Äî no real values.

## Three processes ‚Äî env ownership

| Process | Start command | Env source |
|---|---|---|
| API | `npm start` / `node src/server.js` | Host env panel |
| Worker | `npm run worker` / `node src/worker.js` | **Same keys as API** |
| Client | Build-time only | Build env or CI secret |

The worker is **not optional** in production. Chapter 21 enqueues at checkout and status updates. API-only deploy ‚Üí jobs pile up. Split `REDIS_URL` between API and worker ‚Üí API enqueues to Redis A, worker listens on Redis B.

## server/.env.example

Cross-check against `shared/config.js`. Add rows your repo gained in Chapters 6‚Äì21.

| Variable | Purpose | Production note |
|---|---|---|
| `MONGODB_URI` | MongoDB | Atlas ‚Äî not `127.0.0.1` |
| `PORT` | HTTP listen | PaaS injects ‚Äî API only |
| `NODE_ENV` | Runtime mode | `production` on API **and** worker |
| `JWT_SECRET` | Sign tokens | New value ‚Äî not dev string |
| `JWT_ACCESS_EXPIRY` | Access TTL | e.g. `15m` ‚Äî Ch 6 |
| `JWT_REFRESH_EXPIRY` | Refresh TTL | e.g. `7d` |
| `CLOUDINARY_CLOUD_NAME` | Images | Ch 13 |
| `CLOUDINARY_API_KEY` | Images | Server-only |
| `CLOUDINARY_API_SECRET` | Images | Server-only |
| `REDIS_URL` | Cache + BullMQ | Worker **requires** this |
| `CLIENT_ORIGIN` | CORS | Production client HTTPS URL |

Document **your** CORS key name if `config.js` differs.

## Host contract and platform notes

| Requirement | Spec |
|---|---|
| API | All server keys in deploy panel |
| Worker | **Identical** env ‚Äî second PaaS service or PM2 app |
| Client | `VITE_API_URL` at build time only |
| Atlas | Deploy host IP allowed (22.4) |

| Platform | API | Worker | Client |
|---|---|---|---|
| Render / Railway | Web service env | Second service, `npm run worker` | `VITE_API_URL` in build |
| VPS + PM2 | `.env` or systemd file | PM2 second app | Local build ‚Üí `dist/` |
| GitHub Actions | Workflow secrets | Separate worker deploy job | Secret before build |

Cookie auth (Ch 7): `Secure` on HTTPS; `SameSite` per subdomain; CORS `credentials: true` if client sends cookies.

## Failure modes

| Symptom | Likely cause |
|---|---|
| API boots, worker crashes | Missing `REDIS_URL` or `MONGODB_URI` on worker |
| Health OK, no notifications | Worker not deployed or wrong `REDIS_URL` |
| Auth 500 | `JWT_SECRET` unset |
| CORS in browser | `CLIENT_ORIGIN` still localhost |
| API enqueues, worker silent | Split-brain Redis URLs |
