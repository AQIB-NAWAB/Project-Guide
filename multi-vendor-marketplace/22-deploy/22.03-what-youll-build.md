# 22.3 — What you'll build

Deploy is not one button — it is a **checklist of deliverables** that prove your localhost app lives on the internet. Read this before choosing a host in 22.4.

---

## Deliverables

| Deliverable | Spec |
|---|---|
| Production API URL | HTTPS, `/health` returns 200 + `db: connected` |
| Production client URL | HTTPS, loads shop, points at production API |
| **Production worker** | Second process — `npm run worker` on host — notifications work |
| Env checklist | All vars in `.env.example` + host configured for API **and** worker |
| Smoke tests | Health, login, browse, checkout on live URL — worker log after checkout |
| Restart test | Reboot or redeploy — API + worker recover |
| README Deploy section | Steps another dev could follow |

---

## Three things leave your laptop

```
┌─────────────────────────────────────────────────────────┐
│  1. API server     Express + MongoDB + Redis + enqueue  │
│  2. Worker         BullMQ processors — NOT optional     │
│  3. Client static  React build from client/dist/        │
└─────────────────────────────────────────────────────────┘
```

Beginners often deploy 1 and 3 and wonder why checkout "works" but no `[NOTIFY]` logs appear. **The worker is deliverable #2.**

---

## Typical topology

```
Browser → HTTPS CDN/static host (client)
       → HTTPS API host (Express + PM2 or PaaS)
            → MongoDB Atlas
            → Redis (managed or same VPS)
            → Cloudinary
       → Worker host (same Redis + MongoDB as API)
```

**You pick** one path in 22.4 — document yours in README.

---

## Client build contract

```bash
cd client
VITE_API_URL=https://api.yourdomain.com npm run build
```

Output `client/dist/` deployed to static host or served by API — pick one.

The API URL is **frozen at build time** — not read from a runtime file on Vercel/Netlify.

## Common beginner questions

**Q: Is deploy a coding chapter?**  
Mostly **configuration and verification** — few new features. The code is done; ops begins.

**Q: Free tier enough?**  
Usually yes for course gate. Note cold-start delays in learning log if health check is slow after idle.

**Q: One service or three?**  
Minimum: API service + worker service + static client. Can be two on VPS (API+worker via PM2, nginx for client).
