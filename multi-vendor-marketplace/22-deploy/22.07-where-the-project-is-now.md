## Local verification matrix

```bash
# Health + DB
curl -s http://localhost:5000/health

# Auth smoke (replace credentials)
curl -s -X POST http://localhost:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"vendor@seed.test","password":"..."}'

# Catalogue (cache HIT/MISS in server logs)
curl -s "http://localhost:5000/api/products?page=1&limit=5"

# Client production build
cd client && npm run build && npm run preview
```

| Check | Expected |
|---|---|
| `/health` | `status: ok`, `db: connected` |
| `npm run build` | Exit 0 — no env undefined errors |
| Preview | Shop loads; API calls use `VITE_API_URL` |

Fix build errors **before** 22.9 — production uses same build.

## Environment inventory — copy from local to host

Every key in `server/.env.example` must have a production value on the host:

| Variable | Production note |
|---|---|
| `MONGODB_URI` | Production cluster — not dev database |
| `REDIS_URL` | Same Redis for cache + BullMQ, or two URLs if you split |
| `JWT_SECRET` | New strong secret — not dev string |
| `CLOUDINARY_*` | From Ch 13 |
| `PORT` | Often set by host (Render/Railway) |
| Client `VITE_API_URL` | **HTTPS** API URL at build time |

Chapter **22.8** checklist expands this — start gathering values now.
