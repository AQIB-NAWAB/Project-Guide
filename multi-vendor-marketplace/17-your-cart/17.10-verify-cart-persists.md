# 17.10 — Verify cart persists

**Why this step matters:** Persistence is the whole point of Chapter 16 — reload, new tab, or re-login must show the same cart. If this fails, checkout in Chapter 18 has nothing reliable to read.

**Where the project is now:** Cart UI complete in a single session.

**After this step:** You have **screenshot-worthy proof** that cart survives refresh, re-login, and matches server state.

---

## Test script — run in browser

**Setup:** Log in as seed customer. Empty cart first if needed.

1. Add product from **Store A** — quantity 2.
2. Add product from **Store B** — quantity 1.
3. Open `/cart` — confirm two store groups.
4. **Hard refresh** (Ctrl+Shift+R) — cart unchanged.
5. Change quantity on one line — refresh — change persisted.
6. Remove one line — refresh — removal persisted.
7. Log out → log in as **same customer** — cart restored.
8. Log in as **different customer** — cart empty or different — not User A's items.

---

## Server cross-check

After browser tests, curl as same customer:

```bash
curl -s http://localhost:5000/api/cart \\
  -H "Authorization: Bearer <customer-token>"
```

Browser display must match JSON — no phantom lines in UI.

## Common failures

| Symptom | Likely cause |
|---|---|
| Cart empty after refresh | Client-only state; POST never ran |
| Wrong user's cart | Token mix-up; scoping bug in Ch 16 |
| Count wrong | Badge rule ≠ actual lines |
| Store name missing | Populate not on GET /cart |
