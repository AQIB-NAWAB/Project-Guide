## API contracts to verify before 17.5

Run as logged-in **customer** (not vendor):

```bash
# Add from two different stores — use real product IDs from seed
curl -s -X POST http://localhost:5000/api/cart \\
  -H "Authorization: Bearer <token>" \\
  -H "Content-Type: application/json" \\
  -d '{"productId":"<id1>","quantity":2}'

curl -s http://localhost:5000/api/cart \\
  -H "Authorization: Bearer <token>"
```

**Expected:** Response includes **two items** with different `storeId` values; populated `product` and `store` (or equivalent) for display.

If populate is missing, fix Chapter 16 before building UI — otherwise cart page shows raw ObjectIds.
