# 17.7 — Remove item

**Why this step matters:** DELETE from the UI completes the edit loop — customers fix mistakes without clearing the whole cart. Badge count and store subtotals must update after remove.

**Where the project is now:** Quantities update via PATCH; no way to delete a line from the UI.

**After this step:** Remove control per row calls `DELETE /cart/:itemId`; empty store sections disappear; empty cart shows empty state.

---

## DELETE contract

```
DELETE /api/cart/:itemId
Authorization: Bearer <token>
```

**Success:** 204 or 200 — per your Chapter 16 implementation.

**After success:** Remove row from UI or refetch full cart. Refetch is simpler and avoids drift.

---

## UI control

| Element | Spec |
|---|---|
| Trigger | "Remove" text button or trash icon — accessible label required |
| Confirm | Optional confirm dialog — not required for MVP |
| Loading | Disable remove while request pending |

When last item in a store group is removed, **remove entire store section** — do not show empty heading.

---

## Decrease-to-zero path

Align with 17.6: when − would go below 1, call **DELETE** instead of PATCH `quantity: 0` if API rejects zero.

Document behaviour in README one line.

---

## Error handling

| Status | UI |
|---|---|
| 404 | Refetch cart — item already removed |
| 401 | Redirect login |
| Network error | Toast or inline retry |
