# 17.9 — Cart icon and header badge

**Why this step matters:** Discoverability — customers need to see **something is in the cart** while browsing. The badge turns cart from a hidden URL into a normal e-commerce pattern.

**Where the project is now:** Cart works at `/cart` but customers discover it only via URL or manual navigation.

**After this step:** Header shows cart icon with **live item count** on customer-facing pages.

---

## Target files

- `client/src/components/CartIcon.jsx`
- Mount in customer layout or `App.jsx` header — wherever shop pages already navigate

---

## Count contract

Fetch `GET /cart` (or lightweight count endpoint if you added one — **not required**).

**Badge number:** Document one rule in README:

| Rule | Formula |
|---|---|
| Line count | Number of cart rows |
| Unit count | Sum of all `quantity` values |

Pick **unit count** for grocery — matches "3 items" customer language when one line is 3 kg.

---

## When to refresh count

| Event | Refresh |
|---|---|
| App load / customer login | Yes |
| After add, PATCH, DELETE | Yes |
| Route change | Optional if using global hook |
| Window focus | Optional stretch |

---

## Display contract

| Count | Badge |
|---|---|
| 0 | No badge or hidden |
| 1–99 | Integer |
| 100+ | `99+` optional |

Icon links to `/cart`. Accessible: `aria-label="Cart, N items"`.

---

## useCart hook (optional)

`client/src/hooks/useCart.js` — encapsulate fetch, count, `refreshCart()`.

Share between `CartIcon`, `Cart.jsx`, and `ProductDetail` to avoid duplicate `GET /cart` calls.

**Not mandatory** — duplicate fetches acceptable for gate if documented.

---

## Performance note

Full cart fetch for badge is fine at MVP scale. Chapter 20 does not cache cart. Do not premature-optimize.

---

## ✅ Do / ❌ Don't

**✅ Do:** Hide badge for vendor layout — vendors don't use customer cart.

**✅ Do:** Clear count on logout.

**❌ Don't:** Read cart from localStorage for badge.

**❌ Don't:** Show badge without updating after mutations.
