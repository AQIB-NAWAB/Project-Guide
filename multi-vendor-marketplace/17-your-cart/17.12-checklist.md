**Why this step matters:** Checkout assumes a **real, filled, multi-vendor cart in the browser**. This gate blocks Chapter 18 until browse → add → view → persist works end-to-end.

This is the **gate** for Chapter 17. Every box must be honestly ticked. Learning log committed in **`learning-log/17-your-cart.md`**.

---

## Definition of Done

### Cart page

- [ ] Route `/cart` exists and requires customer authentication
- [ ] `GET /cart` drives page content — not hardcoded items
- [ ] Items grouped by `storeId` with store name heading per section
- [ ] Each line shows product name, unit, unit price, quantity, line total
- [ ] Per-store subtotal displayed correctly
- [ ] Grand total (estimate) displayed
- [ ] Empty cart shows message and link to shop
- [ ] Loading and error states handled

### Quantity and remove

- [ ] + / − (or equivalent) calls `PATCH /cart/:itemId`
- [ ] Remove calls `DELETE /cart/:itemId`
- [ ] UI updates after successful mutations
- [ ] Stock or validation errors from API shown to user
- [ ] Removing last item in store section removes section header

### Add to cart

- [ ] Product detail page POSTs to `/cart` with `productId` and `quantity`
- [ ] Unauthenticated user cannot add — redirect or clear message
- [ ] Out-of-stock add rejected with visible error
- [ ] Success feedback after add (toast, message, or badge update)

### Header badge

- [ ] Cart icon visible on customer-facing layout
- [ ] Badge count matches documented rule (line count vs unit sum)
- [ ] Count updates after add, PATCH, DELETE without manual refresh
- [ ] Badge hidden or zero when cart empty
- [ ] Count clears on logout

### Multi-vendor proof

- [ ] Cart contains items from **at least two different stores** in UI
- [ ] Two store sections visible simultaneously
- [ ] curl `GET /cart` matches browser display

### Persistence

- [ ] Hard refresh keeps cart contents
- [ ] Log out and log in as same user — cart restored
- [ ] Different user sees only their own cart

### Prior chapter gates still true

- [ ] Chapter 16 cart API complete
- [ ] Chapter 15 shop browse works
- [ ] Chapter 7 auth + token refresh works

## Learning log — mandatory

Create **`learning-log/17-your-cart.md`**. Answer in **complete sentences**. Reference **your product name** and **real store names from seed**.

### Questions

1. **Grouping:** How does your cart page group items? Which API field keys the group? Paste a redacted JSON snippet of two items from different stores.

2. **Subtotals:** How do you compute per-store subtotal and grand total? Which price field from the API do you use?

3. **Mutations:** Walk through one quantity change — PATCH payload, response, how UI updates.

4. **Badge rule:** Do you count lines or sum quantities? Why did you choose that rule?

5. **Persistence test:** Describe your reload and two-user test results from 17.10.

6. **Auth UX:** What happens when a logged-out user tries to add to cart or open `/cart`?

7. **Chapter 18 preview:** When checkout splits by store, how will your cart UI have prepared the customer?
