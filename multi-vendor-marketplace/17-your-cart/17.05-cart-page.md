# 17.5 — Cart page

**Why this step matters:** The cart page is the **first place customers see multi-vendor grouping** — store sections mirror checkout split and prevent "one blob list" confusion before Chapter 18.

**Where the project is now:** Cart API works; no `/cart` route in React.

**After this step:** Customer navigates to `/cart` and sees items **grouped by store** with per-store subtotals.

---

## Target file

`client/src/pages/Cart.jsx` — register route in `App.jsx` as `/cart`.

Protected route: unauthenticated → redirect login.

---

## Page load contract

1. On mount, call `getCart()`.
2. While loading, show loading UI — not empty cart message.
3. On success, if `items.length === 0`, render empty state.
4. Else, transform items into groups keyed by `storeId`.

**Grouping pseudocode shape:**

```javascript
// items from GET /cart
// groups = Map storeId → { storeName, lines: [...] }
// render one StoreCartGroup per entry
```

**You implement** — course specifies grouping output, not state management library.

---

## StoreCartGroup component contract

| Prop / data | Display |
|---|---|
| `storeName` | Section heading |
| `lines` | Array of cart lines for that store |
| `subtotal` | Number — sum of `price * quantity` per line |

Use **product.price** (or populated field name from your API) × **quantity** for line total. Show **unit** next to price where available.

---

## CartItemRow — read-only for this step

In 17.5, rows can be **display-only** — quantity controls land in 17.6, remove in 17.7.

Minimum row content:

- Product name
- Unit + unit price
- Quantity (numeric display)
- Line total

---

## Empty state contract

Copy tone: editorial, not playful.

- Heading: e.g. "Your cart is empty"
- Body: one sentence — browse the marketplace
- CTA: link to `/shop`

---

## Error handling

| Error | UI |
|---|---|
| 401 | Redirect login |
| 5xx / network | Message + retry button |
| 200 empty | Empty state — not error |
