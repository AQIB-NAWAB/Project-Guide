# 17.6 — Quantity controls

**Why this step matters:** PATCH wiring teaches **refetch-after-mutation** — the pattern every cart and checkout screen needs so UI never shows stale quantities after server rejects an over-stock update.

**Where the project is now:** Cart page displays quantities as static numbers.

**After this step:** Customer increments or decrements quantity; each change calls `PATCH /cart/:itemId` and refreshes display.

---

## Interaction contract

| Action | Behaviour |
|---|---|
| Increase (+) | `quantity + 1` → PATCH → refetch or optimistic update |
| Decrease (−) | `quantity - 1` → if result ≥ 1, PATCH; if 0, treat as remove (17.7) |
| Direct input | Optional — validate integer ≥ 1; debounce PATCH |

**Minimum:** + and − buttons per line.

---

## PATCH contract (from Chapter 16)

```
PATCH /api/cart/:itemId
Authorization: Bearer <token>
{ "quantity": <positive integer> }
```

**Success:** 200 with updated item or full cart — match your Ch 16 response shape.

**Failure cases to surface in UI:**

| Status | Meaning | UI |
|---|---|---|
| 400 | Invalid quantity | Inline error |
| 404 | Item gone | Refetch cart |
| 409 / 400 | Insufficient stock | Message — show API error text if provided |

Disable + button while request in flight to prevent double-submit.

---

## UX during update

| Pattern | Acceptable |
|---|---|
| Optimistic UI | Yes — rollback on error |
| Spinner on row | Yes |
| Disable entire page | Avoid — blocks other lines |

Refetch `GET /cart` after successful PATCH if PATCH response does not return full grouped data.

---

## Stock ceiling

If product `stockQty` is available on cart line, cap + button at stock — optional enhancement. Server must still reject over-stock on PATCH.

---

## Grand total refresh

Store subtotals and grand total **must recalculate** after quantity change — no stale totals.

---

## ✅ Do / ❌ Don't

**✅ Do:** Use cart line `_id` as `itemId` in PATCH path — not productId.

**✅ Do:** Handle concurrent edits — refetch on focus if mentee leaves tab open.

**❌ Don't:** Update quantity only in React state without API call.

**❌ Don't:** Allow quantity 0 via PATCH — use DELETE (17.7).
