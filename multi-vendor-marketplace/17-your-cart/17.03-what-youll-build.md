# 17.3 — What you'll build

This sub-chapter is the blueprint. No new API routes — just **client pages and components** wired to Chapter 16 contracts. Read it once now; return after 17.10 and confirm your browser demo matches.

---

## Frontend-only chapter — what that means

| You will | You will not |
|---|---|
| Create `Cart.jsx`, cart components, header badge | Add Express routes |
| Call existing `GET/POST/PATCH/DELETE /cart` | Change CartItem schema |
| Wire product detail button to POST | Implement checkout |
| Refetch cart after every mutation | Store cart only in React state |

If something fails, **test the API with curl first** — same discipline as Chapter 7 after Chapter 6.

**Definition of done in one sentence:** Logged-in customer adds from two stores, opens `/cart`, sees two sections, changes quantity, refreshes — cart still there.

## Routes you'll add or extend

| Route | Page | Auth |
|---|---|---|
| `/cart` | `Cart.jsx` | Customer logged in |
| `/shop/:id` | `ProductDetail.jsx` | Public browse; add requires login |
| Header (all customer pages) | `CartIcon` | Shows count from `GET /cart` |

Redirect unauthenticated users hitting `/cart` to login — reuse protected route pattern from Chapter 7.

## API helpers contract

Extend `client/src/api/` with functions matching Chapter 16:

| Function | HTTP | Body / params |
|---|---|---|
| `getCart()` | `GET /cart` | — |
| `addToCart(productId, quantity)` | `POST /cart` | `{ productId, quantity }` |
| `updateCartItem(itemId, quantity)` | `PATCH /cart/:itemId` | `{ quantity }` |
| `removeCartItem(itemId)` | `DELETE /cart/:itemId` | — |

**You implement** — course specifies endpoints, not fetch wrapper internals.

---

## Cart page behaviour contract

| Element | Spec |
|---|---|
| Loading | Skeleton or spinner while `GET /cart` in flight |
| Empty | Message + link to `/shop` |
| Grouped sections | One per `storeId`; store name as heading |
| Line row | Product name, unit, unit price, quantity control, line total |
| Store subtotal | Sum of line totals in section |
| Grand total | Sum of store subtotals — label clearly as estimate |
| Checkout CTA | Button stub or disabled until Chapter 19 — link text "Proceed to checkout" acceptable |

---

## Header badge contract

| State | Display |
|---|---|
| Loading | Hide badge or show `…` |
| 0 items | Hide badge or show nothing |
| N items | Integer count — sum of **quantities** or count of **lines**; pick one, document in README |

Refetch count after add, PATCH, DELETE, and on route change if using hook without global state.
