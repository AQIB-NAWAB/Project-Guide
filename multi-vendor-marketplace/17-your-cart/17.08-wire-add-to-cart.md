# 17.8 — Wire add-to-cart

**Why this step matters:** This connects **browse → cart** — the Week 3 funnel starts working. Product detail must send `productId` and `storeId` (per Ch 16 contract) so multi-vendor lines land in the right buckets.

**Where the project is now:** Product detail shows product info; add button may be stubbed.

**After this step:** Customer clicks add on product detail → `POST /cart` → feedback → optional navigate to cart.

---

## Target file

`client/src/pages/ProductDetail.jsx` — add button and quantity picker (minimum quantity 1).

---

## Flow contract

```
User on /shop/:productId
  → if not logged in: redirect login or prompt
  → select quantity (default 1)
  → click Add to cart
  → POST /cart { productId, quantity }
  → on success: toast or inline "Added" + update header count
```

**You implement** — course specifies sequence, not toast library.

---

## POST contract

```
POST /api/cart
Authorization: Bearer <token>
{ "productId": "<id>", "quantity": <n> }
```

**Success:** 201 — item added or quantity merged per Ch 16 rules.

**Errors to handle:**

| Status | UI |
|---|---|
| 401 | Redirect login |
| 400 | Stock or validation — show message |
| 404 | Product not found |

---

## Quantity on detail page

Default `1`. Optional stepper before add — do not exceed `stockQty` if shown on detail.

If same product already in cart, Ch 16 may merge quantities — UI should still show success.

---

## Auth gate

| State | Behaviour |
|---|---|
| Anonymous | Disable button + "Log in to add" or redirect on click |
| Customer | Full flow |

Vendors browsing as themselves are out of scope — assume customer role for shop pages.

---

## Navigation after add

| Pattern | Acceptable |
|---|---|
| Stay on detail | Yes — show confirmation |
| Go to /cart | Yes — optional secondary link |
| Auto-redirect | Discouraged — interrupts browse |

---

## ✅ Do / ❌ Don't

**✅ Do:** Use `productId` from route params or loaded product document.

**✅ Do:** Disable add when `stockQty === 0` if stock shown.

**❌ Don't:** POST without Bearer token.

**❌ Don't:** Client-only cart push — must hit API.
