# 10.6 — Scaffold vendor layout

## Why this step matters

Vendor pages share navigation, header, and auth context — a layout component avoids copy-pasting shell markup into every page. This is the container for open-shop, settings, and (later) product routes. Get routing structure right before form logic.

---

**Where the project is now:** No vendor-specific layout or pages.

**After this step:** `VendorLayout.jsx` exists with shell (header/nav); `pages/vendor/` folder exists; router nests vendor routes; layout checks auth role — store gate logic lands in 10.9.
