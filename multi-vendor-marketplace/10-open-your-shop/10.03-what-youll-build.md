# 10.3 — What you'll build

By the end of Chapter 10, a vendor can **complete shop onboarding entirely in the browser** and edit shop details later. Storeless vendors are redirected to open-shop; vendors with a store reach the vendor shell (product pages stub OK until Ch 12).

---

## Deliverables checklist (preview)

| Deliverable | Proof |
|---|---|
| `VendorLayout` | Wraps vendor routes; runs store gate |
| `pages/vendor/OpenShop.jsx` | POST /stores form |
| `pages/vendor/StoreSettings.jsx` | GET me + PATCH form |
| Store API helpers in `api/` | getMyStore, createStore, updateStore |
| Onboarding redirect | 404 on GET me → `/vendor/open-shop` |
| Verify flow | New vendor forced through setup |
| Product routes blocked when storeless | Manual URL test |

---

## Route map (spec)

Adjust paths to match your router — stay consistent in learning log.

| Path | Component | Gate |
|---|---|---|
| `/vendor/open-shop` | OpenShop | Public within vendor auth — no store required |
| `/vendor/settings` | StoreSettings | Store required |
| `/vendor` or `/vendor/dashboard` | Placeholder or redirect | Store required |
| `/vendor/products` | Stub OK until Ch 12 | Store required |

All except open-shop nested under `VendorLayout` with shared nav shell.

## API helpers (spec)

| Function | HTTP | Used by |
|---|---|---|
| `getMyStore()` | GET /stores/me | Layout gate, settings load |
| `createStore(payload)` | POST /stores | OpenShop submit |
| `updateStore(id, payload)` | PATCH /stores/:id | StoreSettings submit |

Use authenticated client from Ch 7 — Bearer header automatic.

## Form fields (match Ch 9)

| Field | OpenShop | StoreSettings |
|---|---|---|
| name | required | editable |
| description | optional | editable |
| address | per ERD | editable |
| logoUrl | optional | editable |

Submit buttons: "Open shop" vs "Save changes". Loading and error states required.

## Error handling (preview Ch 12 depth)

| Status | UI |
|---|---|
| 400 | Inline field errors from API |
| 409 on POST | Message + link to settings/dashboard |
| 403 | Rare — role bug |
| 401 | Redirect login |
| Network error | Retry message |

## Out of scope

## ✅ Do / ❌ Don't

**✅ Do:** Reuse same input components for open-shop and settings where possible.

**✅ Do:** After POST success, refetch GET me before navigation.

**❌ Don't:** Hardcode store id in client — read from GET me.

**❌ Don't:** Skip gate because seed vendor always has store — test fresh vendor.
