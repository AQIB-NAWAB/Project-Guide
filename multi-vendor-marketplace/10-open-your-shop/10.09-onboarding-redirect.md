# 10.9 — Onboarding redirect

## Why this step matters

Without a gate, storeless vendors bookmark `/vendor/products` and hit confusing API errors later. The layout checks `GET /stores/me` on mount — **404 means redirect to open-shop**. This is product-shaped UX: force the lease signing before the stockroom.

---

**Where the project is now:** OpenShop and StoreSettings work in isolation; layout checks role but **does not enforce store existence**.

**After this step:** `VendorLayout` enforces **store gate** — 404 on `GET /stores/me` redirects to `/vendor/open-shop`; open-shop path is exempt.

---

## Gate logic (spec)

```
VendorLayout mount / route change:
  1. Not authenticated → /login
  2. role !== vendor → customer home
  3. path is /vendor/open-shop → render <Outlet /> (skip store check)
  4. getMyStore()
     - 200 → cache store; render <Outlet />
     - 404 → navigate('/vendor/open-shop', { replace: true })
     - 401 → logout / login
```

Use `replace: true` — avoids back-button loop.

---

## UI contract — gated routes

| Path | Store required? | GET me 404 |
|---|---|---|
| `/vendor/open-shop` | No | **Stay** — show form |
| `/vendor`, `/vendor/settings` | Yes | Redirect open-shop |
| `/vendor/products` | Yes | Redirect open-shop |

While GET me pending: loader or skeleton. Nav disabled until gate resolves.

---

## API contract

```
GET /api/stores/me
Authorization: Bearer <vendor-token>
```

| Status | Gate action |
|---|---|
| 200 | Cache store; render child |
| 404 | Redirect open-shop |
| 401 | Clear auth; login |

Same `getMyStore()` as OpenShop and settings — one helper, three call sites.

---

## VendorStoreContext (recommended)

| State | Use |
|---|---|
| `store` | Nav name; `_id` for PATCH |
| `loading` | Gate spinner |
| `refetch()` | After POST in OpenShop |

OpenShop success: `refetch()` then `navigate('/vendor')`.

---

## Login redirect integration

| Role | Post-login |
|---|---|
| vendor | `navigate('/vendor')` — gate sends storeless to open-shop |
| customer | `/` or `/shop` |

Do not hardcode open-shop in login — layout owns the gate.

## Troubleshooting

| Symptom | Likely cause |
|---|---|
| Infinite redirect | open-shop not excluded from gate |
| Stuck on spinner | GET me never resolves — check API URL/CORS |
| Gate passes but settings 404 | Stale context — verify 201 before refetch |
| Seed vendor sent to open-shop | GET me actually 404 for that user |

---

## ✅ Do / ❌ Don't

**✅ Do:** Test open-shop → create → dashboard without refresh; share one `getMyStore()`.

**❌ Don't:** Redirect open-shop on 404; use registration as store proxy; skip gate on settings.
