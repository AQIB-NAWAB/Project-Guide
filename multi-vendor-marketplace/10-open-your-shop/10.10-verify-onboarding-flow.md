# 10.10 — Verify onboarding flow

## Why this step matters

Individual pages can work while the journey breaks — gate bypass, wrong redirect, stale cache after create. Run the full script with a **fresh vendor account** in the browser. This is the Chapter 10 gate before product APIs assume every vendor has a store.

## Common failures

| Symptom | Fix |
|---|---|
| Stuck on open-shop after POST | Gate not refetching; navigate before 201 completes |
| Flash of products page | Gate async — add loading state |
| 401 on POST | Token not attached — check api helper |
| CORS error | Server CORS config |
| Form fields empty on settings | GET me response mapping wrong |
