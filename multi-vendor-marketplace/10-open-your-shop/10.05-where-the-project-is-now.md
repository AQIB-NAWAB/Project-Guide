### Store API contracts (reference)

| Method | Path | Role | Success | Key failure |
|---|---|---|---|---|
| `POST` | `/api/stores` | vendor | 201 | 409 duplicate, 403 customer |
| `GET` | `/api/stores/me` | vendor | 200 | 404 no store yet |
| `PATCH` | `/api/stores/:id` | vendor | 200 | 403 wrong owner |

**Body rule:** Never send `vendorId` — server sets from JWT.

### Pre-build curl gate

```bash
curl -s http://localhost:5000/api/stores/me -H "Authorization: Bearer <vendor-token>"
curl -s -X POST http://localhost:5000/api/stores \
  -H "Authorization: Bearer <vendor-token>" \
  -H "Content-Type: application/json" \
  -d '{"name":"Test Market","address":"1 High St"}'
```

- [ ] GET /stores/me → 404 for fresh vendor
- [ ] POST /stores → 201; second POST → 409
- [ ] PATCH /stores/:id → 200 for owner
