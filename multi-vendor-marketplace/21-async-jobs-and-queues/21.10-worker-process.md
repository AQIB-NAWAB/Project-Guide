# 21.10 — Worker process

**Why this step matters:** Jobs enqueued without a running worker sit in Redis forever — checkout "works" but nothing notifies. A separate worker entry is the architectural lesson of this chapter and a production requirement in Chapter 22.

**After this step:** `server/src/worker.js` is a standalone entry that registers BullMQ `Worker`(s) and processes jobs. `npm run worker` starts it.

---

## worker.js responsibilities — spec

| Step | Action |
|---|---|
| 1 | Load `.env` + validate config (same as API) |
| 2 | Connect MongoDB if processor reads Order/User |
| 3 | Register Worker on `notifications` queue |
| 4 | Route by `job.name` to correct processor |
| 5 | Log startup: `[worker] listening on queue notifications` |
| 6 | Handle `SIGINT`/`SIGTERM` — `worker.close()` optional |

**Processors** implemented in 21.12 — stub with log-only handler OK for first commit:

```javascript
// illustrative router
async function processor(job) {
  switch (job.name) {
    case JOB_NAMES.ORDER_CREATED:
      return handleOrderCreated(job.data);
    // ...
    default:
      throw new Error(`Unknown job: ${job.name}`);
  }
}
```

---

## package.json script — contract

```json
{
  "scripts": {
    "worker": "node src/worker.js"
  }
}
```

Adjust path if your `server.js` lives under `src/` — match existing `dev` script style.

ESM project: use `node --import` or `"type": "module"` consistently with Ch 2.

## MongoDB in worker?

| Processor needs | Connect DB? |
|---|---|
| `order.created` — only logs payload | Optional — payload has ids |
| Fetch customer email from User | **Yes** — `connectDB()` in worker boot |
| `product.updated` cache bust | **No** — Redis only |

**You decide** — document if worker shares `shared/db.js` with API.

## Failure visibility

| Event | Expected log |
|---|---|
| Worker starts | Queue name + Redis connected |
| Job received | Job id + name |
| Processor success | `[NOTIFY]` mock lines in 21.12 |
| Processor throw | BullMQ retry log — configure in 21.14 |

## ✅ Do / ❌ Don't

**✅ Do:** Separate file from `server.js` — gate requirement.

**✅ Do:** Register all job processors worker will handle.

**❌ Don't:** Call `app.listen()` in worker — no HTTP server.

**❌ Don't:** Forget to document two-terminal setup in README.
